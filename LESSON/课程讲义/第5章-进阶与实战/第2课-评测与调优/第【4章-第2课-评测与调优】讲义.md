## 一、课程导入（10分钟）

### 1.1 为什么需要评测与调优？

在之前的课程中，我们学习了如何搭建Agent。但搭建完成只是第一步，**真正让Agent产生价值的是持续的评测和优化**。

> **场景**：你搭建了一个房产文案生成Agent，上线后发现：
> - 有些文案质量很好，有些却很糟糕
> - 用户反馈说"有时候有用，有时候没用"
> - 你不知道该如何改进
> 
> **问题**：没有评测体系，就无法知道问题在哪里；没有优化方法，就无法持续提升。

### 1.2 评测与调优的重要性

> **观点**：智能客服Agent的任务完成率和用户满意度直接关系到客户留存和销售额。通过评估并优化Agent，可以提高其服务质量，从而促进业务发展。
> 
> **来源**：搜狐网 - 亚马逊云科技助力AI质量全面优化，Agent构建秘籍第六章!
> 
> **链接**：https://m.sohu.com/a/949559754_122025561/

**没有评测的Agent就像没有仪表盘的飞机**：
- 不知道飞得多高、多快
- 不知道油量还剩多少
- 不知道是否偏离航线
- 最终可能坠毁

**评测与调优的价值**：
1. **发现问题**：知道哪里做得不好
2. **验证效果**：确认优化是否有效
3. **指导决策**：数据驱动的产品决策
4. **持续改进**：形成正向循环

## 二、评测指标体系（20分钟）

### 2.1 核心评测指标

> **观点**：评估AI Agent的性能指标需从多个维度综合考量，核心指标包括任务完成率（成功执行目标任务的次数与总尝试次数的比例）和响应时间（从用户输入到Agent输出答案的平均延迟）。
> 
> **来源**：腾讯云开发者社区 - 如何评估AI Agent的性能指标?
> 
> **链接**：https://cloud.tencent.com/developer/ask/2186746

**核心指标框架**：

| 维度 | 指标 | 定义 | 计算方法 |
|-----|------|------|---------|
| **效果指标** | 任务完成率 | 成功完成任务的占比 | 成功次数/总次数 |
| | 准确率 | 回答/操作的正确率 | 正确次数/总次数 |
| | 用户满意度 | 用户对服务的满意程度 | NPS评分/满意度调查 |
| **效率指标** | 响应时间 | 从输入到输出的时间 | 平均延迟（秒） |
| | 吞吐量 | 单位时间处理量 | 请求数/秒 |
| | 资源消耗 | 计算资源使用情况 | CPU/内存/成本 |
| **稳定性指标** | 可用性 | 系统正常运行时间 | 正常运行时间/总时间 |
| | 错误率 | 发生错误的频率 | 错误次数/总次数 |
| | 恢复时间 | 故障后恢复所需时间 | 平均恢复时间 |

### 2.2 详细指标说明

**任务完成率（Task Completion Rate）**

```
定义：Agent成功完成用户任务的次数占总任务次数的比例

计算公式：
任务完成率 = 成功完成的任务数 / 总任务数 × 100%

示例：
- 总任务数：1000次
- 成功完成：850次
- 任务完成率：85%

评估标准：
- 优秀：> 90%
- 良好：80-90%
- 及格：70-80%
- 需改进：< 70%
```

**用户满意度（User Satisfaction）**

```
测量方法：
1. NPS（净推荐值）
   - 问题："您有多大可能向朋友推荐这个Agent？（0-10分）"
   - 计算：推荐者% - 贬损者%
   
2. CSAT（客户满意度）
   - 问题："您对这次服务满意吗？（1-5分）"
   - 计算：满意用户数 / 总用户数
   
3. 直接反馈
   - 👍/👎按钮
   - 文字反馈
   - 用户访谈
```

**响应时间（Response Time）**

```
定义：从用户输入到Agent输出完整响应的时间

测量点：
1. 首次响应时间：输出第一个字符的时间
2. 完整响应时间：输出完整答案的时间
3. 交互响应时间：多轮对话中每轮的响应时间

优化目标：
- 首次响应 < 1秒
- 完整响应 < 3秒
- 复杂任务 < 10秒
```

### 2.3 评测方法

> **观点**：Google提出五种应对方式：衡量关键指标，设计A/B测试定义KPI（目标完成率，用户满意度，成本），用LLM作为评判：构造裁判Agent根据预定义准则评估结果质量。
> 
> **来源**：今日头条 - 读完Google这份Agent白皮书，我才理解了什么是Agent
> 
> **链接**：http://m.toutiao.com/group/7595779234526757422/

**评测方法1：人工评测**

```
方法：人工标注员对Agent输出进行评分
优点：质量高，能捕捉细微差别
缺点：成本高，速度慢
适用：小规模测试，关键场景验证

实施步骤：
1. 准备测试集（100-500个典型问题）
2. 定义评分标准（1-5分，明确各分数标准）
3. 多人标注，取平均值
4. 计算一致性（Kappa系数）
```

**评测方法2：自动评测**

```
方法：使用程序自动评估
优点：快速、可重复、成本低
缺点：可能不够准确
适用：大规模测试，回归测试

常用方法：
1. 规则匹配：检查输出是否包含关键信息
2. 相似度计算：与标准答案计算BLEU/ROUGE分数
3. 模型评判：用另一个LLM评估输出质量
```

**评测方法3：用户反馈**

```
方法：收集真实用户的反馈
优点：反映真实使用情况
缺点：反馈可能不及时、不全面
适用：线上产品持续监控

收集方式：
1. 显式反馈：👍/👎按钮、评分、评论
2. 隐式反馈：使用时长、完成率、留存率
3. 主动调研：问卷、访谈、用户测试
```

## 三、A/B测试与效果评估（20分钟）

### 3.1 什么是A/B测试？

> **观点**：AI Agent进行在线A/B测试与效果评估的步骤：定义目标与指标，明确测试目标（如转化率提升、点击率优化等），选择关键指标（如用户停留时长、购买率、跳出率）。
> 
> **来源**：腾讯云开发者社区 - 问AI Agent如何进行在线A/B测试与效果评估?
> 
> **链接**：https://cloud.tencent.com/developer/ask/2186850

**A/B测试定义**：
将用户随机分成两组，一组使用A版本（对照组），一组使用B版本（实验组），对比两组的关键指标，判断哪个版本更优。

```
A/B测试流程：

1. 假设提出
   "优化提示词可以提升任务完成率"
   
2. 实验设计
   ├─ 对照组（50%用户）：使用原提示词
   └─ 实验组（50%用户）：使用新提示词
   
3. 指标定义
   ├─ 主要指标：任务完成率
   ├─ 次要指标：用户满意度、响应时间
   └─ 护栏指标：错误率（不能上升）
   
4. 实验运行
   运行2周，收集数据
   
5. 结果分析
   实验组完成率：87%
   对照组完成率：82%
   提升：5%（统计显著）
   
6. 决策
   新提示词效果更好，全量发布
```

### 3.2 A/B测试设计要点

**要点1：样本量计算**

```
问题：需要多少用户才能得出可靠结论？

影响因素：
- 基准转化率（当前水平）
- 预期提升幅度（希望提升多少）
- 统计显著性水平（通常95%）
- 统计功效（通常80%）

示例：
- 当前完成率：80%
- 希望检测到：5%的相对提升（到84%）
- 每组需要：约1000用户
- 总样本量：2000用户

工具：在线样本量计算器
```

**要点2：随机分组**

```
分组原则：
- 随机分配，确保两组用户特征相似
- 避免同一用户同时看到两个版本
- 考虑用户粘性（同一用户始终看到同一版本）

分组方法：
- 用户ID哈希取模
- 随机数生成
- 时间窗口轮换（慎用）
```

**要点3：指标选择**

```
指标层次：

北极星指标（1个）
└─ 最核心的业务指标
   例：任务完成率

主要指标（2-3个）
└─ 直接反映实验效果的指标
   例：准确率、用户满意度

次要指标（多个）
└─ 辅助判断的指标
   例：响应时间、使用频次

护栏指标（必须监控）
└─ 不能变差的指标
   例：错误率、系统稳定性
```

### 3.3 A/B测试案例分析

**案例：优化房产文案生成Agent**

```yaml
实验背景:
  问题: 文案生成质量不稳定，用户满意度只有3.5/5分
  假设: 优化提示词模板可以提升文案质量和用户满意度

实验设计:
  对照组: 使用原有提示词
  实验组: 使用优化后的提示词（增加了更多示例和约束）
  
  流量分配:
    - 对照组: 50%用户
    - 实验组: 50%用户
    
  实验时长: 2周

指标定义:
  主要指标:
    - 用户满意度: 目标从3.5提升到4.0+
    
  次要指标:
    - 文案采纳率: 用户直接使用文案的比例
    - 生成时间: 文案生成耗时
    
  护栏指标:
    - 错误率: 不能上升
    - 响应时间: 不能增加超过20%

实验结果:
  对照组:
    - 用户满意度: 3.5
    - 文案采纳率: 60%
    - 平均生成时间: 2.5秒
    
  实验组:
    - 用户满意度: 4.2 (+20%)
    - 文案采纳率: 75% (+25%)
    - 平均生成时间: 2.3秒 (-8%)
    
  统计显著性: p < 0.01（结果可信）

结论与决策:
  - 新提示词显著提升了用户满意度和文案采纳率
  - 响应时间还有所改善
  - 决策: 全量发布新提示词
```

## 五、Prompt调优与自动化（15分钟）

### 5.1 Prompt调优技巧

> **观点**：提升基础方案效果的方法：精心设计系统Prompt，在提示词中明确Agent的职责边界、何时使用工具。
> 
> **来源**：今日头条 - Agent业务价值实现与迭代逻辑:从单Agent到Multi-Agent的演进之路
> 
> **链接**：http://m.toutiao.com/group/7568316536235065856/

**调优技巧1：明确角色和边界**

```markdown
❌ 不好的Prompt：
你是一个助手，帮助用户处理各种问题。

✅ 好的Prompt：
你是"安居乐寓"的智能房产顾问，专门帮助用户解决租房和买房相关问题。

你的职责：
1. 了解用户的房产需求
2. 推荐合适的房源
3. 解答房产相关问题

你不能：
1. 处理法律纠纷
2. 进行价格谈判
3. 代替签署合同

当遇到超出能力范围的问题时，礼貌地告知用户并建议转人工处理。
```

**调优技巧2：提供示例（Few-shot）**

```markdown
示例1：
用户："我想在深圳租一套两室一厅的房子"
分析：
- 城市：深圳
- 户型：两室一厅
- 预算：未明确（需要询问）
- 区域：未明确（需要询问）
```

## 附录：常见问题解答

### Q1：评测需要多少数据才可靠？

**A**：
- 人工评测：至少100-500个样本
- A/B测试：每组至少1000用户
- 监控指标：至少运行1-2周
- 统计显著性：通常要求p < 0.05

### Q2：如何平衡评测成本和效果？

**A**：
- 自动化评测为主，人工评测为辅
- 核心场景人工评测，一般场景自动评测
- 定期全面评测，日常抽样评测
- 利用用户反馈降低评测成本

### Q3：Prompt调优有什么技巧？

**A**：
1. 一次只改一个地方
2. 保留历史版本，方便回滚
3. 用数据说话，不要凭感觉
4. 关注边界情况和失败案例
5. 让用户参与评测

### Q4：优化效果不明显怎么办？

**A**：
1. 检查是否找对了问题
2. 尝试不同的优化方向
3. 参考优秀案例和最佳实践
4. 考虑是否需要调整产品定位
5. 寻求用户和专家的反馈

### Q5：如何处理优化和业务需求的冲突？

**A**：
- 优先保证核心功能稳定
- 优化和业务需求并行推进
- 用数据证明优化的价值
- 分阶段实施，降低风险
- 保持与业务团队的沟通
