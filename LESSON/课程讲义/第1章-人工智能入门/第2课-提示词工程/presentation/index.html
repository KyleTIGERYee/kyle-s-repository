BROKE是一个融合OKR方法论的提示词框架，特别适合工作任务类场景。

2.1.1

BROKE五要素详解

要素 英文 说明 示例
Background 背景 阐述任务背景，提供充足信息 "我们是深圳长租公寓品牌，服务年轻白领"

Evolve 演变 根据反馈进行优化 "如客户对价格敏感，优先推荐性价比高的"

框架选择指南

BROKE

什么时候用BROKE？

  • ✓ 工作任务类场景
  • ✓ 需要明确交付物
  • ✓ 有明确的KPI或成功标准
  • ✓ 需要迭代优化
CO-STAR

什么时候用CO-STAR？

  • ✓ 内容创作类场景
  • ✓ 需要控制风格和语气
  • ✓ 面向特定受众
  • ✓ 营销文案、客户沟通
2.2.1

Few-Shot Prompting

Few-Shot Prompting是通过提供少量示例，让模型学习任务模式的技术。

为什么有效？

Few-Shot示例为大模型提供额外的知识，称为In-Context Learning（上下文学习），允许模型在给定上下文中学习和推理，而无需更新模型参数。

最佳实践

  • • 示例数量：1-5个效果最佳
  • • 示例质量：选择有代表性的
  • • 示例多样性：覆盖不同输入类型
  • • 格式一致性：输入输出格式统一

分隔符的使用

分隔符用于隔离指令与数据，防止指令注入，提高提示词的清晰度和安全性。

""" 请分析以下客户反馈 并提取关键信息 客户反馈： """ 我上周入住西乡店的A栋302房间 整体体验还不错 房间采光很好 但是空调有点吵 """ 提取要求： 1. 入住门店 2. 房间号 3. 正面反馈 4. 负面反馈 """
### 客户反馈 ### 我上周入住 西乡店的A栋302房间 ... ### 提取要求 ### 1. 入住门店 2. 房间号 3. 正面反馈 4. 负面反馈
--- 示例1： 输入： "我想找个西乡附近的两房" 输出： { "区域": "西乡" "户型": "两居室" } --- 示例2： 输入： "南山科技园附近，单间" 输出： { "区域": "南山科技园" "户型": "单间" } ---

2.2.3

Markdown格式控制

将Prompt Markdown化，本质上是一次"软件工程化"的思维升级。

❌ 无格式

请为客户生成一份房源对比表。 要求使用表格格式，包含房源名称、价格、面积、通勤时间、评分列。 在最后一行添加推荐结论。 使用加粗标注最优项。

✅ Markdown格式
请为客户生成一份房源对比表。 要求： 1. 使用Markdown表格格式 2. 包含以下列： 房源名称、价格、面积、 通勤时间、评分 3. 在最后一行添加"推荐结论" 4. 使用**加粗**标注最优项 示例输出格式： | 对比项 | 房源A | 房源B | |--------|-------|-------| | 价格 | 2800元 | **2500元** | | 面积 | 25㎡ | 22㎡ | **推荐结论**： 推荐房源B，性价比最高。

Coze自动优化提示词

提示词是智能体的灵魂。Coze的"自动优化提示词"功能可以帮助我们学习优秀Prompt的结构。

# 角色 你是安居乐寓的金牌租房顾问 拥有5年行业经验 熟悉深圳各区域房源情况 ## 技能 ### 技能1：需求分析 - 主动询问客户的预算、区域、户型偏好 - 记录客户的特殊需求 （如养宠物、需要停车位等） ### 技能2：房源推荐 - 根据客户需求推荐3-5套合适房源 - 每套房源说明3个匹配理由 ## 限制 - 只推荐安居乐寓旗下房源 - 不承诺具体价格，以实际查询为准 - 保持专业、友好的服务态度

逆向学习要点：观察角色定义、技能模块划分、限制条件的重要性
