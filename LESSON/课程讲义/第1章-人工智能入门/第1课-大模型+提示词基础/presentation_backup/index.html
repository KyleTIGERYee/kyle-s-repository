1.1.1 AI Basics

什么是大语言模型？

大语言模型（Large Language Model，简称LLM）是一种基于深度学习的人工智能系统，它能够理解和生成人类语言。

通俗理解

想象一个读过互联网上几乎所有文本的"超级读者"，它通过海量阅读学会了语言的模式。当你向它提问时，它基于学到的模式"预测"接下来应该说什么。

1.1.2 Core Concept

概率性：AI如何"思考"？

当你问AI问题时，它并不是在"思考"正确答案，而是在计算：

"基于我刚才看到的所有内容，下一个最可能出现的词是什么？"

这就是AI的"思考"方式——不是理解，而是概率预测。它选择概率最高的词作为下一个输出。

交互演示：预测下一个词

AI计算的概率分布：

"好" 75%
"热" 10%
"冷" 10%
"糟糕" 5%

AI选择："好"（概率最高）

1.1.2 Core Concept

非确定性的影响

由于概率机制，同样的输入可能得到不同的输出：

不同输出

刷新页面再问同样的问题，答案可能不同

可能出错

有时会产生幻觉，生成看似合理但实际错误的内容

非100%准确

从概率角度看，大模型的准确率不可能做到100%

与AI协作的本质

与AI协作不是像使用计算器那样得到确定的结果，而是像与一个有知识但有时会说错话的同事合作。你需要学会验证AI的输出，通过更好的提示词引导AI走向正确的概率分布。

工作原理

LLM通过分析输入文本的上下文，计算每个可能的下一个词的概率，然后选择概率最高的词作为输出。

这个过程会不断重复，每次生成一个词，直到生成完整的回答。

1.3.1 Token & Billing

Token与计费

什么是Token？

Token是模型处理文本的基本单位。不同的模型对token的定义略有不同，但大致遵循：

  • • 英文：1个token ≈ 0.75个单词
  • • 中文：1个token ≈ 0.5个汉字

为什么Token很重要？

  • 1. 计费单位：API调用按token数量收费
  • 2. 上下文限制：模型有最大token限制
  • 3. 性能影响：token越多，处理时间越长

实际例子

文本："你好，请介绍一下深圳的天气情况。"

你 好 ， 请 介绍 一下 深圳 的 天气 情况 。

大约11个tokens

Token计算器能帮助您根据输入文本，预估其在不同模型上对应的tokens数量。您也可以通过Tokenizer API实现tokens数量的计算

1.3.3 Temperature

温度（Temperature）

什么是Temperature？

Temperature是一个控制模型输出"随机性"的参数。

工作原理

  • • Temperature = 0：模型总是选择概率最高的词（确定性输出）
  • • Temperature = 1：模型按原始概率分布选择（平衡模式）
  • • Temperature > 1：模型更倾向于选择低概率的词（更有创造性）

实际对比

提示词："请为一款面向深夜加班族的能量饮料写一句简短、有力的广告语。"

Temperature = 0（低，更确定）

"加班到凌晨，喝它提精神！"

Temperature = 1.0（中，平衡）

"夜再深，劲儿仍足！"

Temperature = 2（高，更有创意）

"夜深人不蔫，罐启能量燃！"

1.3.3 Temperature

如何选择Temperature？

场景 推荐Temperature 原因
代码生成 0.0 - 0.3 需要准确无误
数据分析 0.1 - 0.3 需要事实准确
一般问答 0.5 - 0.7 平衡准确性和流畅性
创意写作 0.8 - 1.2 需要多样性和创意
头脑风暴 1.0 + 需要跳出常规思维

1.3.4 Other Parameters

其他重要参数

Top-p

Nucleus Sampling

• 控制模型从概率最高的多少比例的词中选择

• Top-p = 0.9 表示只从概率累计达到90%的词中选择

• 一般来说Temperature和Top-p只选择一个进行调节

Max

Max Tokens

• 限制AI回复的最大token数

• 用于控制成本和回复长度

• 防止模型生成过长的内容

1.4.2 Prompt Importance

提示词输入的重要性

既然AI是基于概率预测下一个词，那么输入的质量直接决定了输出的质量。

提示词工程的核心

通过精心设计的输入，引导AI走向正确的概率分布，从而得到更好的输出。

输入质量对比

❌ 低质量输入

"帮我写个东西"

→ AI需要反复追问才能理解需求

✅ 高质量输入

"帮我写一封请假邮件，我是市场部的员工，因为感冒发烧需要请2天病假，语气要正式但诚恳"

→ AI能直接生成符合要求的内容

1.5.1 Structured Prompt

为什么需要结构化提示词？

"帮我写个关于待出租的房间介绍"

❌ 不知道目标受众是谁

❌ 不知道风格要求

❌ 不知道需要多长

❌ 不知道重点是什么

【角色】房产顾问

【任务】为刚毕业的大学生写租房介绍

【要求】

  • • 语气亲切友好
  • • 突出性价比和安全性
  • • 控制在200字以内

角色

2

3

要求

具体约束条件

1.6 Practice 1

实战练习1：调节Temperature参数

目标

观察不同Temperature设置对输出结果的影响

步骤

  1. 1. 打开Coze或其他AI平台
  2. 2. 使用提示词："请为一款面向深夜加班族的能量饮料写一句简短、有力的广告语。"
  3. 3. 分别设置Temperature为：0、1、2
  4. 4. 记录并比较三次输出的差异

思考

  • • 哪个Temperature产生的广告语最符合你的预期？
  • • 如果Temperature=0和Temperature=2的输出差异不大，可能是什么原因？
