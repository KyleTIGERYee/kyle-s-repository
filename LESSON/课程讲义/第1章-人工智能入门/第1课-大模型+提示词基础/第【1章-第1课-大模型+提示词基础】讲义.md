## 1.1 AI基础概念

### 1.1.1 什么是大语言模型（LLM）

大语言模型（Large Language Model，简称LLM）是一种基于深度学习的人工智能系统，它能够理解和生成人类语言。

> **通俗理解**：想象一个读过互联网上几乎所有文本的"超级读者"，它通过海量阅读学会了语言的模式、语法规则和世界知识。当你向它提问时，它并不是"思考"后回答，而是基于学到的模式"预测"接下来应该说什么。

**关键特征**：
- **大规模**：训练数据包含数千亿甚至数万亿个词
- **通用性**：能处理多种任务（问答、翻译、写作、编程等）
- **生成能力**：能创造连贯、有意义的文本

**常见的大语言模型**：
- DeepSeek
- 通义千问（阿里巴巴）
- 豆包（字节跳动）
- GPT系列（OpenAI）
- Claude（Anthropic）
- Gemini（Google）


### 1.1.2 概率与非确定性

大语言模型的核心特性是**概率性**和**非确定性**。

> **核心观点**：模型不会"真正地理解你的意图"，只会计算概率。


**什么是概率性？**

当你问AI一个问题时，它并不是在"思考"正确答案，而是在计算：**"基于我刚才看到的所有内容，下一个最可能出现的词是什么？"**

举例说明：
```
输入："今天天气很"
AI计算：
- "好" 的概率：75%
- "热" 的概率：10%
- "冷" 的概率：10%
- "糟糕" 的概率：5%

AI选择："好"（概率最高）
```

**非确定性的影响**：

由于这种概率机制，同样的输入可能得到不同的输出。这就是为什么：
- 刷新页面再问同样的问题，答案可能不同
- AI有时会"胡说八道"（幻觉）
- 大模型的准确率不可能做到100%

> "技术原理本质上是做'Next Token Prediction'，即通过当前输入的文本预测下一个词出现的概率来进行训练和回答。因此，从概率的角度看，大模型的准确率不可能做到100%。"


**与AI协作的本质**：

理解这一点非常重要——与AI协作不是像使用计算器那样得到确定的结果，而是像与一个有知识但有时会说错话的同事合作。你需要：
- 学会验证AI的输出
- 通过更好的提示词引导AI走向正确的概率分布
- 接受一定程度的不确定性

### 1.1.3 预测下一个词（Next Token Prediction）

这是大语言模型最核心的工作机制。

**Token是什么？**

Token是模型处理文本的基本单位。它可以是一个字、一个词，甚至是一部分词：
- "我" → 1个token
- "人工智能" → 可能是2-3个tokens
- "unbelievable" → 可能是3个tokens（un + believ + able）

**Next Token Prediction的工作原理**：

```
用户输入："请介绍一下深圳"

AI的处理过程：
1. 将输入转换为tokens
2. 计算下一个最可能的token
3. 将生成的token加入上下文
4. 重复步骤2-3，直到生成完整的回答

生成过程示例：
"请" → "介绍" → "一下" → "深圳" → "。" → "\n" → "深圳" → "是" → "中国" → "的" → ...
```

> "Next Token Prediction。大模型的生成公式，是关于'过去'的条件概率。Context Window就是你的'前世今生'。模型根据这个前世今生，计算概率，掷骰子。"



**为什么这很重要？**

理解Next Token Prediction能帮助你：
1. **理解AI的局限性**：AI没有真正的"理解"，只是在预测
2. **编写更好的提示词**：给AI足够的上下文，它就能预测得更准确
3. **调试AI输出**：当AI回答不好时，往往是上下文不够清晰

## 1.2 Transformer架构通识

### 1.2.1 什么是Transformer

Transformer是2017年由Google提出的神经网络架构，它是现代大语言模型的基础。

> "Transformer算法是一种基于注意力机制(Attention Mechanism)的深度学习模型，最初由Vaswani等人在2017年的论文《Attention is All You Need》中提出。"

**Transformer的革命性之处**：

在Transformer之前，处理序列数据（如文本）主要使用RNN（循环神经网络），它必须按顺序处理每个词，速度很慢。

Transformer的创新：
- **并行处理**：能同时处理句子中的所有词
- **注意力机制**：让模型关注输入中最重要的部分
- **可扩展性**：可以训练更大的模型，获得更好的效果

### 1.2.2 注意力机制（Attention Mechanism）

注意力机制是Transformer的核心，它让模型能够"关注"输入中相关的部分。

**通俗理解**：

想象你在阅读一篇文章时，眼睛会自动关注重要的词语。注意力机制就是让AI拥有这种能力。

**自注意力（Self-Attention）示例**：

```
句子："猫坐在垫子上，因为它很温暖。"

问题："它"指的是什么？

人类理解：看上下文，"它"指的是"垫子"

AI的处理：
- 计算"它"与句子中每个词的"关联度"
- "垫子"的关联度：0.85
- "猫"的关联度：0.10
- "坐"的关联度：0.05

结论："它"最可能指"垫子"
```

> "注意力机制（Attention Mechanism）：Transformer模型的核心是自注意力（Self-Attention）机制，它允许模型在序列的每个位置同时考虑所有位置的信息，从而捕捉序列内部..."


**为什么注意力机制重要？**

1. **捕捉长距离依赖**：能理解句子开头和结尾之间的关系
2. **并行计算**：大幅提升训练速度
3. **可解释性**：可以看到模型"关注"了哪些词

### 1.2.3 不涉及复杂数学的理解方式

你不需要理解Transformer的数学细节。只需要记住：

- **输入**：文本被切分成tokens
- **处理**：通过多层注意力机制，每个token都与其他所有token"交流"
- **输出**：预测下一个token

想象一个会议室，里面坐满了代表每个词的人。他们互相讨论、交换信息，最后每个人都对"接下来应该说什么"有了判断。

## 1.3 大模型参数详解

### 1.3.1 Token与计费

**什么是Token？**

Token是模型处理文本的基本单位。不同的模型对token的定义略有不同，但大致遵循：
- 英文：1个token ≈ 0.75个单词
- 中文：1个token ≈ 0.5个汉字
Token计算器能帮助您根据输入文本，预估其在不同模型上对应的tokens数量。您也可以通过Tokenizer API实现tokens数量的计算
**为什么Token很重要？**

1. **计费单位**：API调用按token数量收费
2. **上下文限制**：模型有最大token限制
3. **性能影响**：token越多，处理时间越长

**实际例子**：

```
文本："你好，请介绍一下深圳的天气情况。"

Token分解（示意）：
["你", "好", "，", "请", "介绍", "一下", "深圳", "的", "天气", "情况", "。"]

大约11个tokens
```

### 1.3.2 上下文窗口（Context Window）

**什么是上下文窗口？**

上下文窗口是模型能"记住"的最大token数量。它包括：
- 你的输入（提示词）
- AI之前的回复
- 系统指令

> "Context Window就是你的'前世今生'。模型根据这个前世今生，计算概率..."


**常见模型的上下文窗口**（2026年1月最新数据）：

| 厂商 | 最强模型 | 上下文窗口 |
|-----|---------|-----------|
| DeepSeek | DeepSeek V3.2 | 128K tokens |
| 阿里巴巴 | Qwen3-Max-Thinking | 256K tokens |
| 月之暗面 | Kimi K2.5 | 256K tokens|
| OpenAI | ChatGPT-5.2 | 400K tokens |
| Anthropic | Claude Opus 4.5 | 200K tokens |
| Google | Gemini 3.0 Pro | 1M tokens |

**上下文窗口的意义**：

1. **成本控制**：窗口越大，处理成本越高
2. **注意力保持**：太长的对话会让模型"忘记"早期的内容
3. **对话轮数限制**：长对话需要截断或总结

### 1.3.3 温度（Temperature）

**什么是Temperature？**
 
Temperature是一个控制模型输出"随机性"的参数。

**工作原理**：

- **Temperature = 0**：模型总是选择概率最高的词（确定性输出）
- **Temperature = 1**：模型按原始概率分布选择（平衡模式）
- **Temperature > 1**：模型更倾向于选择低概率的词（更有创造性）

**实际对比**：

```
提示词："请为一款面向深夜加班族的能量饮料写一句简短、有力的广告语。"

Temperature = 0（低，更确定）：
"加班到凌晨，喝它提精神！"

Temperature = 1.0（中，平衡）：
"夜再深，劲儿仍足！"

Temperature = 2（高，更有创意）：
"夜深人不蔫，罐启能量燃！"
```

**如何选择Temperature？**

| 场景 | 推荐Temperature | 原因 |
|-----|----------------|-----|
| 代码生成 | 0.0 - 0.3 | 需要准确无误 |
| 数据分析 | 0.1 - 0.3 | 需要事实准确 |
| 一般问答 | 0.5 - 0.7 | 平衡准确性和流畅性 |
| 创意写作 | 0.8 - 1.2 | 需要多样性和创意 |
| 头脑风暴 | 1.0 + | 需要跳出常规思维 |

### 1.3.4 其他重要参数

**Top-p（Nucleus Sampling）**：
- 控制模型从概率最高的多少比例的词中选择
- Top-p = 0.9 表示只从概率累计达到90%的词中选择
- 一般来说**Temperature**和**Top-p只选择一个进行调节**

**Max Tokens（最大生成长度）**：
- 限制AI回复的最大token数
- 用于控制成本和回复长度

## 1.4 不确定性思维：与AI协作的本质

### 1.4.1 接受AI的不确定性

与AI协作，首先要建立正确的心态：

1. **AI不是搜索引擎**：它不会"查找"答案，而是"生成"答案
2. **AI不是计算器**：同样的输入可能得到不同输出
3. **AI会犯错**：需要人类验证重要信息

### 1.4.2 提示词输入的重要性

既然AI是基于概率预测下一个词，那么**输入的质量直接决定了输出的质量**。

**提示词工程的核心**：

通过精心设计的输入，引导AI走向正确的概率分布，从而得到更好的输出。

## 1.5 结构化提示词及框架

### 1.5.1 为什么需要结构化提示词

**非结构化提示词的问题**：

```
❌ 不好的提示词：
"帮我写个关于待出租的房间介绍"

问题：
- 不知道目标受众是谁
- 不知道风格要求
- 不知道需要多长
- 不知道重点是什么
```

**结构化提示词的优势**：

```
✅ 好的提示词：
"请扮演一位专业的房产顾问，为刚毕业的大学生写一段租房介绍。
要求：
- 语气亲切友好
- 突出性价比和安全性
- 控制在200字以内
- 包含一个行动号召"
```

### 1.5.2 基础结构化框架

**角色-任务-要求 框架**：

```
【角色】
你是[具体角色]，具备[相关能力/经验]。

【任务】
请帮我[具体任务描述]。

【要求】
1. [具体要求1]
2. [具体要求2]
3. [具体要求3]

【输出格式】
请以[格式要求]输出。
```

**示例**：

```
【角色】
你是安居乐寓的金牌销售顾问，拥有5年租房服务经验，熟悉深圳各区域的房源情况。

【任务】
为一位在南山科技园工作的年轻白领推荐合适的房源。

【要求】
1. 预算在3000-4000元之间
2. 通勤时间不超过30分钟
3. 突出房源的安全性和便利性
4. 语气专业但亲切

【输出格式】
请用以下格式输出：
- 推荐房源名称
- 推荐理由（3点）
- 行动号召
```

### 1.5.3 提示词工程六大策略（OpenAI官方）

根据OpenAI官方指南，提示词工程有六大策略：

1. **写清楚说明**
   - 提供清晰的指令和背景信息
   - 明确输出的格式和长度

2. **提供参考**
   - 给AI提供相关的背景材料
   - 要求AI基于提供的材料回答

3. **将复杂任务拆分**
   - 把大任务分解成小步骤
   - 每一步都有明确的输入输出

4. **给模型思考时间**
   - 要求AI"一步一步思考"
   - 让AI先进行"思维链"

5. **使用外部工具**
   - 结合API、数据库等外部资源
   - 让AI调用工具完成特定任务

6. **系统测试变更**
   - 对提示词的修改进行A/B测试
   - 用数据验证哪种提示词效果更好

## 1.6 实战练习

### 实战1：调节Temperature参数

**目标**：观察不同Temperature设置对输出结果的影响

**步骤**：
1. 打开Coze或其他AI平台
2. 使用相同的提示词："请为一款面向深夜加班族的能量饮料写一句简短、有力的广告语。"
3. 分别设置Temperature为：0、1、2
4. 记录并比较三次输出的差异

**思考题**：
- 哪个Temperature的输出最符合你的预期？
- 在什么场景下你会选择高Temperature？什么场景选择低Temperature？

### 实战2：编写结构化提示词

**目标**：用AI辅助编写一个结构化提示词

**步骤**：
1. 明确你的需求（例如：生成房源描述、回复客户咨询等）
2. 使用以下模板向AI求助：

```
我需要为一个[具体场景]编写提示词。

请帮我设计一个结构化提示词，包含：
1. 角色设定
2. 任务描述
3. 具体要求（至少3条）
4. 输出格式

我的场景是：[详细描述]
```

3. 根据AI的建议，完善你的提示词
4. 测试并迭代优化

**示例场景**：
- "我是安居乐寓的客服，需要回复客户关于押金退还的咨询"
- "我需要为西乡店的新房源写一段吸引人的介绍"
- "我要帮客户比较两个房源的优缺点"

## 1.7 本课小结

### 核心概念回顾

1. **大语言模型（LLM）**：基于海量文本训练的AI系统，通过预测下一个词来生成文本
2. **概率与非确定性**：AI不会"理解"，只会计算概率；同样的输入可能得到不同输出
3. **Next Token Prediction**：大模型的核心工作机制
4. **Transformer架构**：基于注意力机制的神经网络架构
5. **Token**：模型处理文本的基本单位
6. **上下文窗口**：模型能记住的最大token数量
7. **Temperature**：控制输出随机性的参数

### 关键技能

- 理解AI的工作原理和局限性
- 能够调节Temperature等参数控制输出
- 掌握基础的结构化提示词编写方法
