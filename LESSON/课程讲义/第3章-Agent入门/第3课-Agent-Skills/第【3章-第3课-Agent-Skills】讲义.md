## 一、课程导入（10分钟）

### 1.1 从Tools到Skills的演进

在前面的课程中，我们学习了如何通过**工具（Tools）**扩展Agent的能力：

- 调用天气API获取实时天气
- 查询数据库获取房源信息
- 使用地图服务查询位置

这些工具让Agent能够"做"很多事情，但有一个问题：

> **场景：房产文案撰写**
>
> 用户："帮我写一套房源的推广文案"
>
> 使用Tools的Agent：可以调用房源查询工具获取信息，但写出来的文案可能不够专业、缺乏营销技巧。

**问题**：工具告诉Agent"有什么"，但没有教Agent"怎么做"。

### 1.2 解决方案：Agent Skills

> **观点**：把某个领域的专业知识（工作流程、上下文、最佳实践），打包在一起，让通用AI变成领域专家。可以把Skill理解成一个「微型专家系统」。
>
> **来源**：今日头条 - 一键复制 10 年工作经验，Coze 2.0 Skill 实测

## 二、认知重塑：Tools vs Skills（15分钟）

### 2.1 什么是Agent Skills？

> **观点**：进入2025年，行业术语发生了深刻的变化，"Agent Skills"逐渐取代单纯的"Tools"成为核心叙事。这种转变并非仅仅是营销辞藻的更迭，而是反映了技术架构的根本性升级。
>
> **来源**：今日头条 - 2026 AI 新风口：告别 Prompt Engineering，Agent Skills 才是智能体...

**Agent Skills（智能体技能）**是封装的专业知识包，它：

- 包含领域专业知识和工作流程
- 提供最佳实践和行为模式
- 塑造Agent的思考和决策方式
- 让通用AI变成领域专家

### 2.2 Tools与Skills的本质区别

> **观点**：Tools（工具）是Agent的"手"，负责执行对外部世界的真实操作。Skills则是Agent的"专业知识"，它提供封装的专业知识，旨在塑造Agent的思考和决策能力。
>
> **来源**：CSDN博客 - 大模型智能体"内卷"时代：Agent Skills让AI从"青铜"到"王者"

```
┌─────────────────────────────────────────────────────────────────┐
│                    Tools vs Skills 对比                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Tools（工具）                        Skills（技能）             │
│   ─────────────                        ─────────────             │
│                                                                 │
│   🤲 执行操作                          🧠 掌握知识               │
│                                                                 │
│   • 调用API获取数据                    • 提供专业方法论          │
│   • 执行具体动作                       • 指导思考方式            │

│                                                                 │
│   示例：                               示例：                    │
│   • 查询房源数据库                     • 房产文案撰写技巧        │
│   • 获取天气信息                       • 客户需求分析方法        │
│   • 发送邮件                           • 销售谈判策略            │
│                                                                 │
│   关系：Tools是"手"，Skills是"脑"                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 深度对比表

| 维度 | 工具(Tools) | 技能(Skills) |
|-----|------------|-------------|
| **核心定义** | 原子化的可执行函数，侧重于API调用 | 模块化的专业领域知识包，侧重于思维模式 |
| **功能定位** | 执行外部操作 | 提供内部知识 |
| **实现方式** | 函数调用、API请求 | 指令文件、知识库、工作流 |
| **作用对象** | 外部世界（数据、服务） | Agent本身（思维、决策） |
| **典型示例** | 天气查询、数据库操作 | 文案撰写、需求分析 |
| **使用时机** | 需要获取数据或执行操作时 | 需要专业知识和方法论时 |

### 2.4 渐进式披露机制

> **观点**：Claude Skills采用了一种聪明的"渐进式披露"（Progressive Disclosure）机制：发现阶段只加载所有Skills的名称和描述（轻量级）；激活阶段当你的请求匹配某个Skill的描述时，Claude才加载完整内容；执行阶段按照Skill的指令执行任务。
>
> **来源**：CSDN博客 - 2026 最新 Claude Skills 开发指南：原理、入门及实战案例

```
┌─────────────────────────────────────────────────────────────────┐
│                 Progressive Disclosure 原理                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   发现阶段：元数据加载（~100 tokens）                            │
│   ┌──────────┐                                                  │
│   │ Name     │ ──┐                                             │
│   │ Desc     │   │ 极轻量级，快速扫描                            │
│   └──────────┘   │                                              │
│                  ▼                                              │
│   激活阶段：智能匹配                                             │
│   ┌──────────────────────────────────┐                         │
│   │ 用户请求 → 匹配Skill描述         │                         │
│   └──────────────────────────────────┘                         │
│                  │ 命中 → 加载完整内容                          │
│                  ▼                                              │
│   执行阶段：按需加载                                            │
│   ┌──────────────────────────────────┐                         │
│   │ SKILL.md + 知识库 + 脚本         │                         │
│   └──────────────────────────────────┘                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**为什么渐进式披露重要？**

1. **效率提升**：不需要加载所有Skills，保持上下文轻量
2. **按需使用**：只在需要时才加载完整内容
3. **可扩展性**：可以安装大量Skills而不影响性能
4. **成本优化**：减少不必要的token消耗

### 3.1 第一式：发现他人的优质Skills ⭐⭐⭐⭐⭐（10分钟）

**元Skill**：`find-skills`

**核心操作**：
```bash
npx skills find [关键词]
```

**使用场景**：
- 遇到新领域不知道如何下手
- 想找现成的最佳实践
- 调研行业标准工作流
- 寻找灵感参考

**操作演示**：

**步骤1：确定搜索关键词**
```bash
# 场景：你需要做房源推荐，想找相关的Skills

npx skills find real estate recommendation
```

**步骤2：分析搜索结果**
```
返回结果示例：

1. property-recommendation
   └─ 房源推荐最佳实践
   └─ 安装：npx skills install property-recommendation
   └─ 来源：clawhub.ai

2. rental-analysis
   └─ 租赁分析技能包
   └─ 安装：npx skills install rental-analysis
   └─ 来源：skills.sh

   └─ 客户匹配算法
   └─ 安装：npx skills install customer-matchmaking
```

**步骤3：查看Skill详情**
```bash
# 查看Skill的详细内容

```

**练习1：发现练习**

> **任务**：用find-skills搜索"房产文案"或"property writing"相关Skills
>
> **操作**：
> ```bash
> npx skills find 房产文案
> npx skills find property listing
> ```
>
> **产出**：记录找到的Skills，分析其结构和内容

**最佳实践**：

1. **多语言搜索**：中文搜不到时试试英文

```markdown
# 文案撰写技能
```

**修改后（房产文案）**：
```markdown
# 房产文案撰写技能

## 角色定义
你是一位专业的房产文案撰写专家，擅长将房源信息转化为吸引人的推广文案，帮助房东快速出租/出售房源。

## 撰写原则
1. 突出房源亮点：找到独特之处并放大（如地铁学区、精致装修）
2. 场景化描述：让用户能想象住在那里的生活
3. 数据支撑：用具体数字增加可信度（面积、距离、价格）
4. 情感共鸣：触动用户的情感需求（家的温馨、投资的价值）
5. 避免虚假：不用夸张虚假的描述

## 工作流程
1. 分析房源特点（位置、户型、价格、配套）
2. 确定目标客群（学生、上班族、家庭、投资客）
3. 选择文案风格（简洁专业/温馨亲切/投资导向）
4. 生成初稿（标题+开篇+细节+结尾）
5. 优化打磨（检查错误、调整语气）

## 特别注意
- 不使用"顶级"、"绝佳"等夸张词汇
- 价格标注清晰（是否为业主实收）
- 位置描述准确（到地铁站步行时间）
- 照片与描述一致
```

### 3.3 第三式：一句话创建Skill ⭐⭐（5分钟）

**元Skill**：`skill-creator`

**操作方式**：
```bash
# 方式1：使用CLI初始化
npx skills init [skill-name]

# 方式2：直接让AI帮你生成
"帮我创建一个写文案的Skill"
```

**适用场景**：
- 需求非常明确
- 快速原型验证
- 简单的一次性Skill
- 时间紧迫的临时需求

**⚠️ 不推荐原因**：

1. **需求描述往往不够清晰**
   - 直接说"帮我创建一个写文案的Skill"，AI可能生成泛泛的内容
   - 缺乏具体的场景、原则、示例

2. **自动生成的Skill缺乏深度**
   - 只有框架，没有细节
   - 缺少实际经验的沉淀

3. **可能遗漏关键内容**
   - 边界情况没有覆盖
   - 注意事项没有提及

4. **示例质量参差不齐**
   - 可能是通用示例
   - 缺乏针对性

**推荐程度**：⭐⭐☆☆☆

**什么情况下可以用？**
- 非常简单的Skill（如"回复模板"）
- 临时使用，不需要长期维护
- 快速验证思路，不追求完美

**正确示范**：
```bash
# 不推荐
"帮我创建一个写文案的Skill"

# 推荐：描述清晰
```

```markdown
# 房源推荐技能

## 角色定义
你是一位专业的房产租赁顾问，擅长根据客户需求推荐最合适的房源。你的目标是帮客户找到"家"的感觉，而不是简单的房屋匹配。

## 推荐原则
1. **需求优先**：先深入了解客户需求，再推荐房源
2. **精准匹配**：推荐数量控制在3-5套，不过多也不过少
3. **亮点突出**：每套房源都要说明推荐理由
4. **诚实透明**：缺点也要如实告知，建立信任
5. **引导决策**：帮助客户分析利弊，但不做最终决定

## 推荐流程
### 第一步：需求挖掘（5个关键问题）
1. 预算范围（是否含水电物业）
2. 地理位置（地铁沿线/区域偏好）
3. 居住人数（独居/合租/家庭）
4. 入住时间（急迫程度）
5. 特殊要求（养宠物/停车位/装修偏好）

### 第二步：初步筛选
1. 在数据库中筛选符合基础条件的房源
2. 排除明显不符合的选项
3. 保留5-8套候选房源

### 第三步：深度匹配
1. 匹配度评分（1-5分）
2. 分析每套的优劣势
3. 确定推荐顺序

### 第四步：输出推荐
推荐格式：
```
【推荐房源1】匹配度：★★★★☆
地址：xxx
租金：xxx元/月
亮点：地铁口5分钟、精装修、采光好
不足：老小区
一句话推荐：适合xxx人群
```

## 常见场景处理
### 场景1：预算模糊
"根据您的需求，我建议先看看3000-5000元/月的房源，这个区间性价比最高，您觉得可以吗？"

### 场景2：选择困难
"这两套各有特色：A套交通方便但价格高，B套价格实惠但稍远，您更看重哪方面呢？"

### 场景3：犹豫不决
"不着急决定，您可以先看看这几套，对比一下再做决定。我可以帮您预约实地看房。"

## 禁忌事项
- 不推荐明显有问题的房源（安全隐患、产权纠纷）

- 不打压房东或房源
- 不承诺无法兑现的事情
```

### 7.2 常见问题

**Q1: Skills和Prompts有什么区别？**

A: Skills是持久化的能力包，Prompts是一次性的指令。Skills可以跨对话使用，Prompts每次都要重新输入。

**Q2: 一个Agent可以安装多少个Skills？**

A: 理论上没有限制，但建议控制在5-10个，太多会影响性能和匹配准确性。

**Q3: Skills会影响AI的响应速度吗？**

A: 不会。渐进式披露机制确保只有在需要时才加载Skill内容。

**Q4: 如何判断一个Skill的质量？**

A: 看4个方面：1）角色定义是否清晰；2）工作流程是否完整；3）示例是否充分；4）用户评价如何。
