# Plan Automator

**紫灵专属自动化技能** - 将测试任务、文档编写、质量巡检等工作自动化

## What this does

- **任务规划**：分析用户需求，生成结构化的 plan 文件（模块拆分、功能点、文档模板）
- **Cron 配置**：询问用户定时任务配置（触发时间、飞书推送等），创建 OpenClaw 定时任务
- **自动执行**：定时触发后读取 plan，操作浏览器，生成文档，更新进度
- **进度追踪**：维护 plan 文件状态，推送进度摘要到飞书

## First Run Setup（Premium Wizard）

**触发条件**：首次使用或用户说"setup" / "configure" / "重新配置"

**时间预估**：约 60 秒

**配置问题**（3-8 个）：

1. **任务类型**：这次要执行什么类型的任务？
   - A. 功能测试 + 文档生成（如按钮级操作手册）
   - B. 回归测试巡检
   - C. 数据质量检查
   - D. 自定义任务

2. **测试环境**：测试环境的登录信息是？
   - URL: [默认：https://uat-mp.szajly.com/]
   - 账号：[默认：易飞虎]
   - 密码：[默认：硬编码在 plan 中]

3. **文档输出目录**：文档保存到哪个目录？
   - [默认：`<workspace>/安居乐寓按钮级操作手册_<日期>/`]

4. **Plan 文件位置**：plan 文件存放在哪里？
   - [默认：`<workspace>/plan_<日期>.md`]

5. **Cron 触发时间**：多久执行一次？
   - A. 每 30 分钟
   - B. 每小时
   - C. 每天 9:00-18:00（工作时间）
   - D. 自定义 cron 表达式

6. **飞书推送**：是否推送进度摘要到飞书？
   - A. 是，推送到当前群聊
   - B. 是，推送到指定频道
   - C. 否，只记录到日志

7. **浏览器配置**：是否需要特殊浏览器参数？
   - [默认：禁用 PNA 安全策略参数，用于 iframe 页面]

**配置完成后**：
- 保存到 `<workspace>/.skill-config/plan-automator.json`
- 生成 plan 文件模板
- 创建 cron 定时任务
- 执行一次测试运行

## Core Workflow

### 阶段一：任务规划（用户提出需求时）

```
用户说："我要测试营销模块" 或 "创建一个文档重写任务"
  ↓
技能分析需求，不清楚则追问：
- 测试范围？（哪些模块）
- 文档类型？（操作手册/测试报告/巡检记录）
- 优先级？（P0/P1/P2）
- 完成时间？
  ↓
生成 plan 文件，包含：
- 任务总览（目标、原则、流程）
- 模块拆分（功能清单、编号规则、依赖关系）
- 文档模板（根据任务类型动态生成）
- 进度追踪表格
  ↓
给用户确认 plan 内容
```

### 阶段二：Cron 配置（plan 确认后）

```
询问用户：
1. 触发时间：多久执行一次？
2. 飞书推送：是否推送进度？推送到哪里？
3. 执行策略：每次执行几个功能点？
4. 错误处理：失败后重试几次？是否跳过？
  ↓
生成 cron 提示词，调用 OpenClaw 创建定时任务
  ↓
告知用户 cron 已创建，如何修改/删除
```

### 阶段三：自动执行（cron 触发后）

```
读取 plan 文件
  ↓
检查工作目录下已完成的文档数量
  ↓
与 plan 中标记的已完成数量进行比对
  ↓
如有差别，更新 plan 文件中的状态标记和统计指标
  ↓
找到下一个"⏳ 待完成"功能点
  ↓
启动浏览器 → 登录测试环境
  ↓
实际操作功能：
- 导航到功能入口
- 点击、填写、提交
- 记录页面结构、字段、校验规则
  ↓
自动生成文档：
- 功能概述
- 页面结构（ASCII 图，自动分析 DOM 生成）
- 字段说明
- 操作流程
- 校验规则
- 异常场景
- 关联影响
  ↓
保存文档到输出目录
  ↓
更新 plan 文件：
- 标记状态为 ✅
- 更新统计表格
- 添加执行记录
  ↓
推送进度摘要到飞书（如配置）
  ↓
关闭浏览器
```

## Plan 文件结构规范

### 必需章节

```markdown
# plan_<日期>.md - [任务名称]

## 一、任务总览
- 项目目标
- 执行原则
- 任务执行流程
- 状态变更规则
- 编号规则

## 二、模块详细规划
### 模块一：[模块名称]
- 功能说明
- 预计功能数
- 优先级
- 功能清单表格（编号、名称、操作类型、复杂度、依赖项、状态、完成时间）

## 三、统计指标
- 进度追踪表格（模块、总功能数、已完成、进行中、待完成、完成率）
- 质量指标

## 四、当前任务
- 当前状态
- 待执行功能列表

## 五、定时任务执行记录
- 任务执行时间
- 执行内容
- 进度状态
- 备注
```

### 状态标记规则

| 状态 | 标记 | 变更条件 |
|------|------|----------|
| 待完成 | ⏳ | 初始状态 |
| 进行中 | 🔄 | 开始实际操作浏览器 |
| 已完成 | ✅ | 文档编写完成并保存后 |
| 阻塞 | ⏸️ | 有前置依赖未完成 |

## Error Handling

### 错误处理规则

| 错误类型 | 处理方式 |
|----------|----------|
| 页面加载失败 | 重试 3 次，仍失败则标记为"⏸️ 阻塞"，记录到执行日志，跳过执行下一个 |
| 登录失败 | 立即停止，推送错误通知到飞书 |
| 无测试数据 | 文档中说明"测试环境无 XX 数据，功能逻辑基于 XX 推断"，标记为 ✅ |
| 文档与计划不一致 | 扫描工作目录下已生成的文档，与 plan 中标记的已完成功能进行比对。如发现文档存在但 plan 中标记为待完成，则更新 plan 状态为 ✅ 已完成，并更新统计指标 |
| 网络超时 | 重试 2 次，仍失败则记录异常，继续执行 |

### 失败通知

推送条件：
- 登录失败
- 连续 3 个功能执行失败
- 浏览器无法启动

推送内容：
```
🚨 Plan Automator 错误通知

任务：[plan 文件名]
错误：[错误描述]
时间：[YYYY-MM-DD HH:mm]
建议操作：[建议处理方式]
```

## Commands

### 创建新任务

```
用户："我要测试营销模块"
技能：追问需求细节 → 生成 plan → 配置 cron → 创建定时任务
```

### 手动触发执行

```
用户："现在执行 plan_0323.md"
技能：读取 plan → 检查工作目录文档与 plan 状态一致性 → 如有差别更新 plan → 执行下一个功能 → 更新进度 → 推送摘要
```

### 查询进度

```
用户："当前进度如何？"
技能：读取 plan 统计表格 → 汇报完成率、剩余功能、预计完成时间
```

### 暂停/恢复

```
用户："暂停任务"
技能：标记 plan 为"⏸️ 暂停"，cron 继续但跳过执行

用户："恢复任务"
技能：标记 plan 为"🔄 进行中"，继续执行
```
