| 状态     | 说明      | 变更条件              |
| ------ | ------- | ----------------- |
| ⏳ 待完成  | 尚未开始    | 初始状态              |
| 🔄 进行中 | 正在操作浏览器 | 开始实际操作            |

**状态变更流程**:

1. 开始操作前：⏳ 待完成 → 🔄 进行中
2. 操作完成并编写文档后：🔄 进行中 → ✅ 已完成
3. 发现依赖未完成：🔄 进行中 → ⏸️ 阻塞

## Plan 生成规则

### 1. 分析用户需求

**输入**：用户的任务描述（如"我要测试营销模块"）

**追问要点**（如不清晰）：
- 测试范围？（哪些模块、哪些功能）
- 文档类型？（操作手册/测试报告/巡检记录）
- 优先级？（P0/P1/P2）
- 完成时间要求？
- 特殊要求？（如必须截图、必须包含 ASCII 图等）

### 2. 拆分模块

**原则**：
- 按业务领域拆分（如营销、供应商、财务）
- 按功能类型拆分（如查询、录入、审核）
- 按优先级拆分（P0 优先、P1 其次）

### 3. 编号规则

**格式**：`[模块前缀]-[序号]`

**示例**：
- MK-REC-001：营销 - 推荐活动 -001
- SP-HA-001：供应商 - 家电家具 -001
- FM-010：财务 -010

### 4. 估算工时

**复杂度评估**：
- **高**：多步骤表单、复杂配置、多页面跳转（预计 30-60 分钟）
- **中**：标准表单、简单配置（预计 15-30 分钟）
- **低**：查询、导出、查看（预计 5-15 分钟）

### 5. 识别依赖

**依赖类型**：
- **数据依赖**：需要前置功能创建数据
- **状态依赖**：需要前置功能改变状态
- **无依赖**：可独立执行
