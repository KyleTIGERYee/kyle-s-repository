# 错误码和处理方式

**用途**：标准化错误处理

---

## 错误分类

### A 类：浏览器相关（A001-A999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| A001 | 浏览器启动失败 | 检查 Chrome 是否安装，检查启动参数 | 3 次 |
| A002 | 页面加载超时 | 检查网络连接，检查 URL 是否正确 | 2 次 |
| A003 | iframe 加载失败 | 检查浏览器 PNA 参数配置 | 3 次 |
| A004 | 元素找不到 | 页面结构变更，需要更新选择器 | 1 次 |
| A005 | 点击无响应 | 元素可能被遮挡，尝试滚动或等待 | 2 次 |

### B 类：登录认证（B001-B999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| B001 | 登录失败 - 账号密码错误 | 检查 plan 文件中的账号密码 | 0 次 |
| B002 | 登录失败 - 账号被锁定 | 联系管理员解锁 | 0 次 |
| B003 | 登录超时 | 检查网络，检查测试环境是否可用 | 2 次 |
| B004 | Session 过期 | 重新登录 | 1 次 |

### C 类：数据相关（C001-C999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| C001 | 无测试数据 | 文档中说明，标记为✅ | 0 次 |
| C002 | 数据创建失败 | 检查必填字段和校验规则 | 2 次 |
| C003 | 数据重复 | 使用不同的测试数据 | 1 次 |
| C004 | 数据查询为空 | 确认筛选条件，确认数据存在 | 0 次 |

### D 类：文档生成（D001-D999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| D001 | 文档保存失败 | 检查目录权限，检查磁盘空间 | 2 次 |
| D002 | ASCII 图生成失败 | 使用简化版页面结构描述 | 0 次 |
| D003 | 文档格式错误 | 检查模板语法 | 1 次 |

### E 类：Plan 更新（E001-E999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| E001 | Plan 文件读取失败 | 检查文件路径，检查文件权限 | 2 次 |
| E002 | Plan 文件写入失败 | 检查文件是否被占用，检查权限 | 2 次 |
| E003 | 进度统计不一致 | 重新计算统计表格 | 1 次 |
| E004 | 执行记录格式错误 | 使用标准格式模板 | 1 次 |

### F 类：推送通知（F001-F999）

| 错误码 | 错误描述 | 处理方式 | 重试 |
|--------|---------|---------|------|
| F001 | 飞书推送失败 | 检查飞书配置，检查频道 ID | 2 次 |
| F002 | 推送内容为空 | 检查进度摘要生成逻辑 | 1 次 |
| F003 | 推送频率超限 | 等待 1 分钟后重试 | 1 次 |

## 错误处理流程

```
错误发生
  ↓
记录错误码和详细信息
  ↓
判断错误类型：
├─ A 类（浏览器）→ 重试 N 次 → 仍失败则标记为"⏸️ 阻塞"
├─ B 类（登录）  → 0 次重试，立即停止并推送错误通知
├─ C 类（数据）  → 根据具体错误处理（无数据则说明后继续）
├─ D 类（文档）  → 重试 N 次 → 仍失败则使用简化模板
├─ E 类（Plan）  → 重试 N 次 → 仍失败则记录日志
└─ F 类（推送）  → 重试 N 次 → 仍失败则只记录日志
  ↓
更新 plan 文件（如适用）
  ↓
推送通知（如配置且达到推送条件）
```

---

## 重试规则

| 错误严重程度 | 重试次数 | 重试间隔 | 失败后处理 |
|------------|---------|---------|-----------|
| 轻微（D/F 类） | 2 次 | 5 秒 | 记录日志，继续执行 |
| 中等（A/C/E 类） | 3 次 | 10 秒 | 标记为"⏸️ 阻塞"，跳过执行下一个 |
| 严重（B 类） | 0 次 | - | 立即停止，推送错误通知 |

## 常见错误排查

### 浏览器启动失败（A001）

**排查步骤**：
1. 检查 Chrome 是否安装：`which chrome` 或 `which "Google Chrome"`
2. 检查浏览器路径是否正确
3. 检查启动参数是否完整（特别是 PNA 相关参数）
4. 检查是否有其他进程占用浏览器

**解决方案**：
```bash
# macOS 启动 Chrome（带 PNA 禁用参数）
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=18800 \
  --disable-web-security \
  --allow-running-insecure-content \
  --disable-site-isolation-trials \
  --no-sandbox \
  --disable-dev-shm-usage
```

### 页面加载超时（A002）

**排查步骤**：
1. 检查网络连接
2. 检查测试环境是否可用（curl 测试）
3. 检查 URL 是否正确
4. 检查是否需要特殊网络配置（如 VPN、代理）

**解决方案**：
```bash
# 测试测试环境可达性
curl -I https://uat-mp.szajly.com/
```

### iframe 加载失败（A003）

**排查步骤**：
1. 检查浏览器是否配置了 PNA 禁用参数
2. 检查 console 是否有 CORS 错误
3. 检查 iframe 源地址是否可访问

**解决方案**：
确保 Chrome 启动参数包含：
```
--disable-web-security 
--allow-running-insecure-content 
--disable-site-isolation-trials
```

### 登录失败（B001）

**排查步骤**：
1. 检查账号密码是否正确（注意大小写）
2. 检查账号是否被锁定
3. 检查测试环境是否允许该账号登录

**解决方案**：
- 联系管理员确认账号状态
- 检查 plan 文件中的账号密码配置

### 无测试数据（C001）

**处理方式**：
在文档中说明：
```
> **注意**: 测试环境无 [XX] 数据，功能逻辑基于 [XX] 推断。
```
标记为 ✅ 已完成。
