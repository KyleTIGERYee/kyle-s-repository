从Tools到Skills的演进

核心问题：工具告诉Agent"有什么"，但没有教Agent"怎么做"。

传统AI

能写，但缺乏专业性

→

+ Tools

能查询房源信息了！

→

+ Skills

掌握了专业方法！

解决方案：Agent Skills

把某个领域的专业知识（工作流程、上下文、最佳实践），打包在一起，让通用AI变成领域专家。把Skill理解成一个「微型专家系统」。

认知重塑

什么是Agent Skills？

核心定义

Agent Skills（智能体技能）是封装的专业知识包。

•

包含领域专业知识和工作流程

•

提供最佳实践和行为模式

•

塑造Agent的思考和决策方式

•

让通用AI变成领域专家

时代意义

降低专业门槛

普通人也能获得专家级能力

知识复用

一次开发，多次使用

生态繁荣

催生"AI技能商店"

认知重塑

Tools vs Skills 本质区别

Tools（工具）

执行操作

API调用调用API获取数据
执行动作执行具体动作
回答"有什么"告诉Agent外部信息

Skills（技能）

掌握知识

方法论提供专业方法论
思维方式指导思考方式
回答"怎么做"教Agent专业处理

关系：Tools是"手"，Skills是"脑"

认知重塑

深度对比表

维度工具(Tools)技能(Skills)
核心定义原子化的可执行函数，侧重于API调用模块化的专业领域知识包，侧重于思维模式
功能定位执行外部操作提供内部知识
实现方式函数调用、API请求指令文件、知识库、工作流
作用对象外部世界（数据、服务）Agent本身（思维、决策）
典型示例天气查询、数据库操作文案撰写、需求分析
使用时机需要获取数据或执行操作时需要专业知识和方法论时
认知重塑

渐进式披露机制

发现阶段：元数据加载（~100 tokens）

只加载Skills的名称和描述，极轻量级，快速扫描

激活阶段：智能匹配

当用户请求匹配Skill描述时，加载完整内容

执行阶段：按需加载

加载SKILL.md + 知识库 + 脚本

认知重塑

Tools + Skills 协同工作

Skills指导"如何写"，Tools提供"写什么"

场景：房源推广文案撰写

Step 1: Agent理解需求
└─ 识别需要：获取房源信息 + 撰写推广文案

Step 2: 调用Tools获取数据
└─ 使用【房源查询Tool】获取房源详细信息
    • 户型：两室一厅
    • 面积：85㎡
    • 价格：5200元/月

Step 3: 调用Skills撰写文案
└─ 使用【房产文案Skill】提供专业方法论
    • 应用撰写原则
    • 遵循文案结构

Step 4: 生成专业结果

协同工作要点

Tools 提供原料

查询、获取、展示外部信息

Skills 提供方法

指导如何处理、思考、决策

协同产生价值

既有信息又有方法，输出高质量结果

形象比喻：房产顾问Agent

Agent

= 一个刚毕业的大学生

Tools

= 他的工具箱

• 手机（查询房源APP）

• 计算器（计算价格）

• 地图（查看位置）

Skills

= 专业知识和经验

• 房产文案撰写技巧

• 客户需求分析方法

• 销售谈判策略

结果对比

只有Tools

能查信息，但不会用

只有Skills

知道怎么做，但无法获取信息

Tools + Skills

既能获取信息，又知道如何专业处理

第五式

迭代优化Skills ⭐⭐⭐⭐

🤔 为什么需要迭代？

• 使用中发现缺失的功能

• 遇到难以处理的边界情况

• 需要补充新的知识

• 业务场景变化，需要更新

🔄 迭代方式A：快速迭代

发现问题 → 立即记录 →
用skill-creator修改 → 测试验证

🔍 迭代方式B：审查式迭代

新开对话框 → 让AI用skill-creator审查现有Skill → 检查标准规范 → 提出改进建议

📋 迭代检查清单

☐ Skill定位是否清晰？

☐ 工作流程是否完整？

☐ 示例是否充分？

☐ 边界情况是否覆盖？

☐ 语言表达是否清晰？

第六式

手动深度定制 ⭐⭐⭐（进阶）

🎯 适用场景

• 对工作流程有清晰认识

• 追求极致的定制化

• 有时间和精力深度打磨

• 需要的Skill非常专业/独特

⚠️ 为什么不推荐单独使用？

• 需要非常清楚整体工作流程

• 要求有清晰的条理认识

• 工作量较大

• "烧脑"

💡 推荐做法：组合使用

Step 1: 先用第六式建立骨架

Step 2: 用第五式让AI审查

Step 3: 用第二式补充细节

Step 4: 用第四式验证优化

常见问题

Q1: Skills和Prompts有什么区别？

Skills是持久化的能力包，Prompts是一次性的指令。Skills可以跨对话使用，Prompts每次都要重新输入。

Q2: 一个Agent可以安装多少个Skills？

理论上没有限制，但建议控制在5-10个，太多会影响性能和匹配准确性。

Q3: Skills会影响AI的响应速度吗？

不会。渐进式披露机制确保只有在需要时才加载Skill内容。

Q4: 如何判断一个Skill的质量？

看4个方面：1）角色定义是否清晰；2）工作流程是否完整；3）示例是否充分；4）用户评价如何。
