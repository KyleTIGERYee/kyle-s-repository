### 7.3 课后作业

**作业一：概念理解（必做）**

1. 解释什么是工具调用（Tool Calling），它解决了什么问题？
2. 描述Function Calling的完整执行流程（从用户输入到最终回复）
3. 对比Dify中两种工具开发模式的优缺点
4. 什么是"防弹版"代码？为什么要这样写？
5. 提交：文字说明（500字以上）

**作业二：房源接口工具复现（必做）**

1. 在Dify中创建工作流模式的房源查询工具
2. 按照讲义步骤配置：开始节点 → HTTP请求 → 代码执行 → 结束
3. 测试以下场景：
   - 正常查询：城市=深圳，预算=5000
   - 空数据查询：城市=火星
   - 错误场景：无效的Token
4. 截图并提交：工作流配置 + 测试结果

**作业三：管家接口工具开发（必做）**

1. 分别用两种模式开发管家查询工具
2. 对比两种模式的开发体验和输出效果
3. 提交：
   - 工作流模式截图
   - 自定义工具模式截图
   - 对比分析（200字）

**作业四：知识库检索工具（选做）**

1. 创建一个知识库（可以上传房产相关文档）
2. 开发知识库检索工作流
3. 实现防弹版代码执行节点
4. 测试并优化检索效果
5. 提交：工作流截图 + 测试对话记录

**作业五：错误处理优化（选做）**

1. 选择你已开发的一个工具
2. 按照讲义中的五层错误处理模型优化代码
3. 设计至少3个异常测试用例
4. 提交：优化后的代码 + 测试用例 + 测试结果
