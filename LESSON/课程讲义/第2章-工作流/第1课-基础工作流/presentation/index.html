1.1.1 核心概念

什么是工作流？

定义

工作流（Workflow）是指业务过程的部分或整体在计算机应用环境下的自动化，是对工作流程及其各操作步骤之间业务规则的抽象和描述。

"Dify 工作流是 Dify 平台最核心的功能之一，它在 AI 应用中负责流程编排——让多个模型调用、工具调用、逻辑节点、变量处理协同工作。"

通俗理解：工厂生产线

1.3.1 核心技能

什么是参数提取？

定义

参数提取是将用户自然语言转化为结构化数据的过程，让AI从文本中识别出关键信息。

用户输入

"我想在西乡租个单间，预算3000左右"

LLM 提取

识别区域、户型、预算...

结构化输出
{"area": "西乡", "layoutName": "单间", "maxPrice": "3000"}

租房场景示例

用户表达

"我想租一个单间，预算1000"

提取的参数
{layoutName: 单间, minPrice: 0, maxPrice: 1000}
用户表达

"我想在西乡租房"

提取的参数
{area: 西乡}
用户表达

"宝安区一房一厅"

提取的参数
{area: 宝安区, layoutName: 一房一厅}
1.3.2 输出格式

结构化输出

什么是结构化输出？

结构化输出让AI返回JSON格式的数据，便于后续节点处理。 

{
  "area": "西乡",
  "layoutName": "单间",
  "minPrice": "0",
  "maxPrice": "3000"
}

为什么需要结构化输出？

易于解析

程序可以直接读取JSON数据

节点间传递

数据在工作流节点间流转

格式统一

输出格式可预测、可验证

说明：结构化输出是LLM节点的核心技能，通过配置输出格式，LLM可以稳定地输出我们需要的JSON结构。

1.4.5 结果返回

输出节点

功能

返回工作流执行结果。

"Coze的workflow提供了两种输出模式：一，返回一些变量值，然后让聊天模型基于这些变量值回复用户；二，在workflow里拼接好输出..."

模式一：返回变量

  • • 工作流返回结构化数据
  • • 由AI根据数据生成回复
  • • 适合需要进一步处理的场景

模式二：直接输出

  • • 工作流直接生成完整回复
  • • 适合简单场景
  • • 响应速度更快

示例配置

# 模式一：返回变量
输出节点:
  类型: variables
  输出:
    - name: recommendations
      value: {{llm_result}}
    - name: count
      value: {{item_count}}
# 模式二：直接输出
输出节点:
  类型: text
  内容: |
    为您找到{{count}}套房源：
    {{recommendations}}

常见问题

Q: 结构化输出不稳定？

  • • 检查JSON Schema是否正确
  • • 强调"只返回JSON"
  • • 调低Temperature（0.1-0.3）

Q: 参数提取不准确？

  • • 在提示词中增加更多示例
  • • 明确说明每个参数的提取规则
  • • 处理好"未提及"的情况

Q: Coze和Dify区别？

  • • 变量引用语法不同
  • • 结构化输出配置界面略有差异
  • • 核心逻辑相同，可迁移
