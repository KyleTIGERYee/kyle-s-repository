### 2.1 意图识别核心原理

**意图识别（Intent Recognition）**是智能客服系统的"大脑"，能够自动理解用户的真实意图，并将请求路由到不同的处理分支。

> "意图识别(Intent Recognition)：自动理解用户真实意图，识别用户问题类型，提取关键实体信息，智能路由到对应处理流程。"
> 
> —— 来源：CSDN博客 - Coze智能体开发

| 意图类型 | 触发场景 | 包含参数 | 处理方式 |

## 三、RAG基础与知识库检索（25分钟）

### 3.1 什么是RAG？

**RAG**（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术。

#### 为什么需要RAG？

传统的大语言模型存在以下局限：
1. **知识截止**：模型训练数据有时间限制，无法获取最新信息
2. **幻觉问题**：模型可能会生成看似合理但实际错误的内容
3. **专有知识缺失**：不了解企业内部的专有文档和数据

> **观点**：RAG（检索增强生成）通过将非结构化文档转化为可检索的知识单元，提升AI回答的准确性和专业性。
> 
> —— 来源：CSDN博客 - 探索 RAG

#### RAG的工作原理

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   用户提问       │────▶│   检索相关文档   │────▶│   生成回答       │
│  "合同违约金     │     │  从知识库中找到  │     │  基于检索到的    │
│   怎么算？"     │     │  相关合同条款    │     │  内容生成准确回答 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
    查询向量化的           向量相似度搜索            LLM结合上下文
    用户问题               返回Top-K相关文档          生成最终回复
```

### 3.2 Dify知识库创建与配置

#### 创建知识库的操作步骤

在Dify中创建知识库非常简单，按照以下步骤操作：

**详细步骤**：

1. **进入知识库页面**：点击左侧导航栏的「知识库」
2. **创建知识库**：点击「创建知识库」按钮

4. **上传文档**：拖拽或选择要上传的文件
5. **选择分段模式**：根据文档类型选择合适的分段方式
6. **完成创建**：点击「保存并处理」，系统自动进行向量化

### 3.3 检索参数详解

#### Top-K 参数

**Top-K** 表示返回最相关的K条结果。

```
用户问题："可以养宠物吗"
     │
     ▼
┌─────────────────────────────────────┐
│          知识库检索结果              │
├─────────────────────────────────────┤
│ 1. 可以养宠物/养猫吗？  Score: 0.92  │ ← Top-1
│ 2. 我想找带阳台的房子   Score: 0.45  │ ← Top-2
│ 3. 预算1500元能租什么   Score: 0.32  │ ← Top-3
│ 4. 怎么联系管家        Score: 0.28  │
│ 5. 可以短租吗          Score: 0.25  │
└─────────────────────────────────────┘
         │
         ▼ 返回Top-3条结果
```

**设置建议**：
- **默认值**：3条
- **建议范围**：3-5条
- **过多**：会引入噪声，影响回答质量
- **过少**：可能遗漏相关信息

#### Score 阈值

**Score** 表示相似度分数，范围0-1，值越高表示越相似。

| Score值 | 相似程度 | 说明 |
|--------|---------|------|
| 0.9+ | 非常相似 | 几乎是相同问题 |
| 0.7-0.9 | 高度相似 | 问题相关性强 |
| 0.5-0.7 | 中等相似 | 有一定关联 |
| <0.5 | 低相似度 | 关联性较弱 |

**设置建议**：
- **默认值**：0.5
- **提高精准度**：可设置为0.7-0.8，但可能漏检
- **提高召回率**：可设置为0.3-0.5，但可能引入噪声

**实际应用示例**：

```yaml
配置场景：精准问答
  Top-K: 3
  Score阈值: 0.7
  效果：只返回高度相关的结果，答案更精准

配置场景：广泛检索
  Top-K: 5
  Score阈值: 0.4
  效果：返回更多可能相关的结果，覆盖面更广
```

### 3.5 RAG效果调优技巧

**提升RAG效果的三个关键点**：

1. **文档质量**
   - 确保文档格式规范，避免扫描件
   - 问题表述清晰，答案完整准确

2. **参数调优**
   - **Top-K**：根据答案复杂度调整，简单问答用3，复杂问题用5
   - **Score阈值**：根据精准度需求调整，默认0.5即可

3. **提示词设计**
   - 明确告知LLM如何使用检索结果
   - 设置"无相关信息时的处理规则"

### 4.3 实操演示：状态管理

**场景**：在安居乐寓项目中，代码节点用于
1.解析LLM输出的JSON，提取意图和参数。
2.解析搜房接口和管家接口输出的JSON，提取并且生成可直接回复用户的内容。
3.数据校验：确保搜房接口和管家接口输出的内容有具体数据。

**代码节点配置**：

```python
# Python代码示例
import json

def main(router_output, current_state_str):
    """
    State Manager: 解析LLM输出的JSON，管理搜索状态
    """
    
    # 1. 解析LLM输出
    router_data = {}
    try:
        if isinstance(router_output, dict):
            router_data = router_output
        else:
            raw_text = str(router_output)
            start_idx = raw_text.find('{')
            end_idx = raw_text.rfind('}')
            if start_idx != -1 and end_idx != -1:
                json_str = raw_text[start_idx:end_idx+1]
                router_data = json.loads(json_str)
    except Exception as e:
        router_data = {"intent": "chat", "params_update": {}}
    
    # 2. 提取数据
    intent = router_data.get("intent", "chat")
    new_params = router_data.get("params_update", {})
    
    # 3. 读取旧状态
    state = {}
    if current_state_str:
        try:
            state = json.loads(str(current_state_str))
        except:
            state = {}
    
    # 4. 智能合并
    if new_params.get("area") and new_params["area"] != state.get("area"):
        state["storeNames"] = ""
        state["address"] = ""
    
    for key, value in new_params.items():
        if value is not None and str(value) != "":
            state[key] = value
    
    # 5. 返回结果
    return {
        "intent": intent,
        "new_state_str": json.dumps(state, ensure_ascii=False),
        "area": state.get("area", ""),
        "layoutName": state.get("layoutName", ""),
        "minPrice": state.get("minPrice", ""),
        "maxPrice": state.get("maxPrice", "")
    }
```

**输入变量**：
- `router_output`: LLM节点的输出
- `current_state_str`: 当前对话状态

**输出变量**：
- `intent`: 解析后的意图
- `new_state_str`: 更新后的状态字符串
- `area`, `layoutName`, `minPrice`, `maxPrice`: 提取的参数

### 5.3 节点配置详解

#### 步骤1：输入节点

```yaml
节点名称: 开始
类型: start

配置:
  输入参数:
    - 名称: user_query
      类型: string
      描述: 用户查询内容
      必填: true
      示例: "我想在西乡租个单间"
```

#### 步骤2：LLM意图识别

```yaml
节点名称: Agent Brain
类型: llm

模型: qwen-flash
Temperature: 0.1

提示词: |
  # Role
  你是一个AI业务规划师。
  
  # 意图类型
  - search_house: 用户想找房子
  - query_kb: 用户想咨询问题
  - chat: 闲聊或其他
  
  # 用户输入
  {{sys.query}}
  
  # 输出
  以JSON格式返回意图和参数

结构化输出: 启用
```

#### 步骤3：条件分支

```yaml
节点名称: 意图分发
类型: if-else

条件:
  - case: search_house
    条件: {{llm.intent}} == "search_house"
  - case: query_kb
    条件: {{llm.intent}} == "query_kb"
  - case: chat
    条件: 其他（默认）
```

#### 步骤4：知识库检索（query_kb分支）

```yaml
节点名称: 知识检索
类型: knowledge-retrieval

配置:
  知识库: 安居乐寓FAQ
  查询内容: "{{input.user_query}}"
  召回数量: 3
```

#### 步骤5：LLM生成回答

```yaml
节点名称: 知识答复
类型: llm

模型: qwen-flash

  你安居乐寓客服。请根据知识库回答用户问题。
```

### 6.5 常见问题

**Q1：意图识别不准确怎么办？**
- 在提示词中增加更多示例（Few-shot）
- 调整Temperature参数（建议0.1-0.3）
- 使用更具体的意图描述
- 添加意图边界说明

**Q2：知识库检索效果不好怎么办？**
- 检查文档质量，确保是文本格式
- 调整相似度阈值，适当降低以召回更多结果
- 优化文档结构，添加清晰的标题
- 增加召回数量

**Q3：代码节点支持哪些Python库？**
- 支持Python标准库：json、re、datetime、random、math
- 不支持第三方库（如pandas、numpy等）

**Q4：结构化输出不稳定怎么办？**
- 检查JSON Schema是否正确
- 在提示词中强调"只返回JSON，不要有其他内容"
- 尝试调低Temperature（建议0.1-0.3）
