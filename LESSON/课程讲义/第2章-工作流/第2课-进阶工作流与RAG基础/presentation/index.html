2.1 核心原理

意图识别核心原理

什么是意图识别？

意图识别（Intent Recognition）是智能客服系统的"大脑"，能够自动理解用户的真实意图，并将请求路由到不同的处理分支。

"意图识别(Intent Recognition)：自动理解用户真实意图，识别用户问题类型，提取关键实体信息，智能路由到对应处理流程。"

—— 来源：CSDN博客 - Coze智能体开发

核心功能

理解用户意图

分析用户输入，判断真实需求

提取关键实体

从文本中提取参数信息

智能路由

将请求分发到对应处理流程

3.1 核心概念

什么是RAG？

RAG定义

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术。

"RAG（检索增强生成）通过将非结构化文档转化为可检索的知识单元，提升AI回答的准确性和专业性。"

—— 来源：CSDN博客 - 探索 RAG

为什么需要RAG？

传统的大语言模型存在以下局限：

1

知识截止

模型训练数据有时间限制，无法获取最新信息

2

幻觉问题

模型可能会生成看似合理但实际错误的内容

3

专有知识缺失

不了解企业内部的专有文档和数据

3.1 工作原理

RAG的工作原理

Step 1

查询向量化：将用户问题转换为向量

Step 2

向量相似度搜索：返回Top-K相关文档

Step 3

LLM结合上下文生成最终回复

3.3 参数详解

Top-K 参数详解

什么是Top-K？

Top-K 表示返回最相关的K条结果。

用户问题："可以养宠物吗"
1. 可以养宠物/养猫吗？ Score: 0.92
2. 我想找带阳台的房子 Score: 0.45
3. 预算1500元能租什么 Score: 0.32
4. 怎么联系管家 Score: 0.28

设置建议

默认值 3条

适合大多数场景

建议范围 3-5条

根据答案复杂度调整

过多

引入噪声，影响质量

过少

可能遗漏相关信息

3.3 参数详解

Score 阈值详解

什么是Score？

Score 表示相似度分数，范围0-1，值越高表示越相似。

Score值 相似程度 说明
0.9+ 非常相似 几乎是相同问题
0.7-0.9 高度相似 问题相关性强
0.5-0.7 中等相似 有一定关联
<0.5 低相似度 关联性较弱

设置建议

默认值 0.5

平衡精准度和召回率

提高精准度

设置为0.7-0.8

但可能漏检

提高召回率

设置为0.3-0.5

但可能引入噪声

实际应用示例

精准问答： Top-K: 3, Score: 0.7
广泛检索： Top-K: 5, Score: 0.4

3.5 效果调优

RAG效果调优技巧

1. 文档质量

  • • 确保文档格式规范
  • • 避免扫描件
  • • 问题表述清晰
  • • 答案完整准确

2. 参数调优

  • • Top-K：简单问答用3，复杂问题用5
  • • Score阈值：默认0.5即可
  • • 根据精准度需求调整

3. 提示词设计

  • • 明确告知LLM如何使用检索结果
  • • 设置"无相关信息时的处理规则"
  • • 强调回答的专业性和友好性

代码节点的作用

什么是代码节点？

代码节点允许我们在工作流中执行自定义的Python或JavaScript代码，实现更复杂的逻辑处理。

"代码节点就是每种服务流程里的一个环节。类比：在顾客点餐流程里，顾客点了一道土豆丝，这道土豆丝是凉拌还是清炒，都需要代码节点来处理。"

4.3 实操演示

状态管理示例

应用场景

在安居乐寓项目中，代码节点用于：

  • 1. 解析LLM输出的JSON，提取意图和参数
  • 2. 解析搜房接口和管家接口输出的JSON
  • 3. 数据校验：确保输出内容有具体数据

Python代码示例

import json

def main(router_output, current_state_str):
    # 1. 解析LLM输出
    router_data = {}
    try:
        if isinstance(router_output, dict):
            router_data = router_output
        else:
            raw_text = str(router_output)
            start_idx = raw_text.find('{')
            end_idx = raw_text.rfind('}')
            if start_idx != -1 and end_idx != -1:
                json_str = raw_text[start_idx:end_idx+1]
                router_data = json.loads(json_str)
    except Exception as e:
        router_data = {"intent": "chat", "params_update": {}}
    
    # 2. 提取数据
    intent = router_data.get("intent", "chat")
    new_params = router_data.get("params_update", {})
    
    # 3. 返回结果
    return {
        "intent": intent,
        "area": new_params.get("area", ""),
        "layoutName": new_params.get("layoutName", "")
    }
