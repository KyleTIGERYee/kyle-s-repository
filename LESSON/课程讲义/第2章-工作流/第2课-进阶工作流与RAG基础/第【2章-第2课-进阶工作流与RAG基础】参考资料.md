**来源**：CSDN博客 - 探索 RAG(检索增强生成)
**链接**：https://blog.csdn.net/Andrew_Chenwq/article/details/146606862

- 2024年，Coze知识库的RAG应用成为AI领域的一大亮点

### 2. Coze知识库节点详解
**来源**：CSDN博客 - 扣子平台的知识库节点
**链接**：https://blog.csdn.net/i042416/article/details/150281290

### 4. RAG工作原理详解
**来源**：CSDN博客 - AI代码开发宝库系列:Coze工作原理与应用
**链接**：https://blog.csdn.net/chenchihwen/article/details/154488586
**核心观点**：
```python
# RAG工作原理（逻辑示意）
def rag_response(user_query, knowledge_base):
    # 1. 检索相关文档
    relevant_docs = retrieve(user_query, knowledge_base)
    # 2. 构建增强提示
    augmented_prompt = build_prompt(user_query, relevant_docs)
    # 3. 生成回答
    response = llm.generate(augmented_prompt)
    return response
```
- 通过检索增强生成技术，让Bot的回答更准确、更专业
**融入建议**：用代码示例解释RAG的工作流程，帮助学员理解技术原理

## 二、HTTP请求节点与外部API调用

### 6. Coze HTTP请求节点使用指南
**来源**：CSDN博客 - 如何使用 Coze 的 HTTP 请求节点实现高效数据交互
**链接**：https://blog.csdn.net/taotaocwl/article/details/147054412
**核心观点**：
- 支持以下核心操作：
  - **数据获取**：通过GET请求从API获取实时数据（如天气、用户信息）
  - **数据提交**：使用POST/PUT请求向服务器提交表单数据或更新资源
  - **数据管理**：通过DELETE/PATCH请求删除或修改数据
- 支持配置请求头、请求体、超时时间等参数
**融入建议**：作为HTTP节点配置的主要参考资料，详细讲解各参数的作用

### 7. Coze工作流调用外部API
**来源**：CSDN文库 - coze工作流调用外部
**链接**：https://wenku.csdn.net/answer/22sgnwg8bs
**核心观点**：
- 在工作流编辑器中拖入「HTTP请求」节点，支持GET/POST/PUT/DELETE方法
- 参数配置示例：

}

## 四、意图识别节点（Intent Recognition）

### 13. 意图识别节点详解
**来源**：CSDN博客 - Coze智能体开发:理解和判断用户意图或目的
**链接**：https://blog.csdn.net/shanghaiwren/article/details/149054534
**核心观点**：
- 意图识别（Intent Recognition）指通过大模型理解和判断用户表达的意图或目的
- 扣子工作流的意图识别节点可通过内置的大语言模型，对指定内容进行意图识别和分类
- 根据识别结果将请求路由到不同的处理分支
**融入建议**：作为意图识别节点的核心参考资料，详细讲解其工作原理

### 14. 意图识别作为"交通指挥官"
**来源**：今日头条 - Day 3 :我把AI变成了"自动化工厂"!解锁Coze工作流
**链接**：http://m.toutiao.com/group/7589602172703179298/
**核心观点**：
- 意图识别节点像一个"交通指挥官"
- 用户问天气？指挥去走天气的路
- 用户问美食？指挥去走美食的路
- 有了它，AI就能同时处理多种不同类型的请求
**融入建议**：用形象的比喻帮助学员理解意图识别的作用

### 15. 意图识别与上下文工程
**来源**：今日头条 - BuildingAI 实战:零代码搭建企业级智能体应用
**链接**：http://m.toutiao.com/group/7582991635751895578/
**核心观点**：
- 意图识别(Intent Recognition)：自动理解用户真实意图，识别用户问题类型，提取关键实体信息，智能路由到对应处理流程
- 上下文工程(Context Engineering)：精细化的上下文管理，包括prompt模板库、上下文压缩、记忆管理等
**融入建议**：将意图识别与上下文工程结合讲解，展示完整的智能体架构
