## 执行环境声明 (路径安全)

**极度重要**：本技能的 Python 脚本全部存放在技能自身目录下的 `scripts/` 文件夹中。你的终端工作目录 (CWD) 通常并不在此技能目录内，因此直接执行 `python3 scripts/xxx.py` 会报错找不到文件。

在触发本技能后、执行任何脚本之前，你必须先确定本 SKILL.md 文件所在目录的**绝对路径**（下文简称 `SKILL_ROOT`）。确定方式：观察你读取本 SKILL.md 时获取的文件路径，去掉末尾的 `/SKILL.md` 即为 `SKILL_ROOT`。

后续所有脚本命令的执行格式均为（以 `search_map.py` 为例）：
```bash
python3 <SKILL_ROOT>/scripts/search_map.py "关键词"
```
请在执行命令时将 `<SKILL_ROOT>` 替换为实际的绝对路径。

## 核心检索工作流：M-R-E-R-G 协议

**核心要求：严禁试图从记忆中盲目作答，必须严格遵守以下 M-R-E-R-G 协议。**

### 1. Map (全景加权导航)
**极度警告：不要试图去读取那庞然大物般的 system_map.json**。你必须基于下置的多维加权检索引擎寻找方向：
1. 提炼用户嘴里抛出的业务诉求，萃取真正的意图词语。
2. 启动你自带的本地终端交互，执行你的减负探测脚本（注意使用绝对路径，参见上方"执行环境声明"）：
   `python3 <SKILL_ROOT>/scripts/search_map.py "意图词1 意图词2"`
3. 那台脚本将自动帮你扩写同义词阵列图，并在打分筛出排名前 10 的文章摘要及路径打印到你面前。你只需要从高分列表中挑取需要精读的 MD 。

### 2. Reflect (反思依赖)
阅读地图后，你**必须**在向用户输出答案前，先使用 `<Thought_Process>` 标签进行逻辑推导和依赖反思。
例如：你要写“深梦扬帆3.0”相关内容，你必须在地图中找出关联深梦扬帆的历史迭代文档路径，并思考：“它涉及企业客户，我是不是应该一并查看 [企业合同] 或 [财务流程] 模块的按钮手册？”

### 3. Explore (级联递归探测)
- 当你调用工具深入阅读某篇 `.md` 文件时，**务必保持对“引路词条”的极度敏锐**。
- 若在文中看到 `影响模块`、`前置依赖`、`下游关联`、`关联系统` 等字眼，你必须**中断生成**。
- 再次调用 `python3 <SKILL_ROOT>/scripts/search_map.py "被牵连的模块关键词"` 来定位关联模块的文件路径，然后阅读拓展。
- **深度限制**：为防止无限循环，自动级联探查最多允许向下追溯 **3层** 关联模块。是否需要联查，由你依据任务必要性全权决定。

### 4. Read (绝对的无损精读 / No Chunking)
- 根据推理与探测拿到的文件 `path`，使用自身工具进行完整的提取。
- **严禁切片或断章取义**。你必须完整阅读这些文件，将里边的表单字段、特殊流转状态、异常提示全量纳入上下文。

### 5. Generate (高维输出)
在查阅完整的依赖树后，合成你的最优化输出。
如果你的输出是一份 PRD 或测试用例，请在“影响范围”或“环境准备”章节中，清晰地向用户阐明：“出于何种原因需要一并评估或兼容测试下列级联的底层模块”。

## 文档生成能力 (Feature PRD Generation)

除了依据 M-R-E-R-G 探索并解答问题外，当你需要**将新的业务需求或设计落地成标准功能文档**时，还需遵循写作规范：

1. **必读工作流规约**：当你需要写新文档时，你必须先使用工具去阅读并理解 `references/workflow-doc-generation.md` 里的生成工作流，写完后拉取 `references/quality-checklist.md` 进行出厂自检。
2. **使用标准术语**：PC端、小程序、租务系统、管家、租客、店长、流水账等。
3. **排版格式约束**：界面按钮用 **粗体** (`**提交**`)，系统页面或菜单层级使用中括号标识 `【字典设置】-【租客字典】`。
4. **触发智能化系统挂载**：当你写完或被指派去“入库清理”一个 `.md` 文档时，**先通读这个洗库防呆纲要：`references/workflow-index-update.md`**。然后必须调用它里面的防护网关小脚本（`update_node.py`）将你要写回 JSON 树骨架的高精摘要和词根注入进去。大模型绝对封锁手动替换大 JSON 代码的行为。

- "请帮我梳理一份新版租务的PRD需求..." (立刻启动 Map, Reflect 寻找功能与按钮相关模块)

- ❌ 越过系统大地图 `system_map.json`，纯靠脑补回答深层的表单字段或异常状态逻辑。
- ❌ 生造文件路径：所有目标 `.md` 的路径必须从大树表 `system_map.json` 中如实摘取！
- ❌ 面对文档中明文列出的“影响模块”却视而不见，不执行 Explore(级联阅读) 的行为。对于跨模块的高维分析任务，不 Explore 则意味着失职。
