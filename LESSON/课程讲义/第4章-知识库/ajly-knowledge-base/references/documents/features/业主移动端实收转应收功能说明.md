# 业主移动端实收转应收功能说明

业主移动端实收转应收功能说明

# 概述

本功能针对业主移动端的数据展示进行优化调整，将原有的"实收"统计口径变更为"应收"统计口径，确保业主移动端展示的数据与BI报表-楼栋收入数据保持一致。通过本次调整，业主可以更准确地了解房源的应收租金和服务费情况，提升数据透明度和业务可信度。

主要调整内容包括：房间详情字段精简、数据计算逻辑统一、时间筛选范围扩展以及统计口径标准化。

# 功能模块

## 功能一：房间详情字段调整

### （一）功能说明

优化房间详情页面的数据展示，移除已收金额相关字段，简化页面信息展示。

### （二）调整内容

1. **删除字段**
   - 删除房间详情中的**租金已收**字段
   - 删除房间详情中的**服务费已收**字段

2. **保留字段**
   - 房租（应收）
   - 服务费（应收）
   - 总计

### （三）注意事项

- 本次调整仅涉及展示字段的删除，不影响后台数据计算逻辑
- 业主如需查看已收金额，可通过其他财务报表查询

## 功能二：数据计算逻辑变更

### （一）功能说明

将业主移动端的收入数据统计口径由"实收"调整为"应收"，计算逻辑与BI报表-楼栋收入数据保持一致。

### （二）计算公式

**总计 = 房租 + 服务费**

#### 房租计算逻辑

```sql
房租 = SUM(应收租金) - SUM(租金优惠券) + SUM(租金优惠券剩余)
```

#### 服务费计算逻辑

```sql
服务费 = SUM(应收服务费) - SUM(服务费优惠券) + SUM(服务费优惠券剩余)
```


## 功能三：时间筛选功能优化

### （一）功能说明

调整业主移动端的时间筛选逻辑，由默认查看"本月数据"改为默认查看"上月数据"，并支持选择过去一年的月份进行查看。

### （二）调整内容

1. **默认时间**
   - 原逻辑：默认查看"本月数据"
   - 新逻辑：默认查看"上月数据"

2. **可选时间范围**
   - 支持选择过去12个月的月份进行数据查看
   - 时间范围：当前月份往前推12个月

### （三）操作步骤

1. 进入业主移动端首页
2. 点击时间筛选器
3. 在弹出的月份选择器中，选择需要查看的月份
4. 系统自动加载所选月份的数据

### （四）注意事项

- 默认展示上月数据，方便业主查看已结算的完整月份数据
- 历史数据范围限制为12个月，超出范围的数据需通过其他渠道查询
- 月份切换后，所有统计数据（房租、服务费、总计）同步更新

## 功能四：字段名称调整

### （一）功能说明

将"总流水金额"字段名称调整为"总计"，使统计口径更加清晰明确。

### （二）调整内容

| 原字段名称 | 新字段名称 | 说明 |
| --- | --- | --- |
| 总流水金额 | 总计 | 总计 = 房租 + 服务费 |

### （三）展示位置

- 业主移动端首页数据概览区域
- 房间详情页面

## 常见问题

**Q1：为什么删除"租金已收"和"服务费已收"字段？**

A1：本次调整将统计口径由"实收"改为"应收"，已收金额不再作为核心展示指标。业主如需查看收款情况，可通过财务报表查询。

**Q2：数据与BI报表不一致怎么办？**

A2：请检查以下几点：
- 确认所选月份是否一致
- 确认统计的门店/物业地址范围是否一致
- 确认是否包含联合经营数据
- 如仍不一致，请联系技术团队排查

**Q3：为什么默认展示上月数据？**

A3：上月数据已经完整结算，数据更加准确稳定。本月数据可能存在未结算账单，不够完整。

**Q4：历史数据能查看多久？**

A4：业主移动端支持查看过去12个月的数据，超出该范围的历史数据请联系运营人员查询。
