# 集团小程序三期功能说明

集团小程序三期功能说明

# 概述

集团小程序三期是安居乐寓面向内部员工（管家、店长、经纪人等）推出的移动端业务管理工具，旨在提升客户管理效率、优化业务流程协同。本版本在二期基础上新增了客户管理、推荐客户、通知待办、组织架构等核心功能模块，实现了从客户线索获取到跟进转化的全流程线上化管理。

本功能适用于安居乐寓全体一线业务人员，包括管家、店长、经纪人、战区负责人等角色，支持客户信息录入、分配、跟进及业务待办处理等核心场景。

# 功能模块

## 一、登录与授权

### （一）操作前提

- 用户已注册安居乐寓业务系统账号
- 用户已绑定微信账号或具备微信授权条件
- 小程序已获取用户手机号授权

### （二）操作步骤

1. 打开安居乐寓集团小程序，进入**登录**页面
2. 点击**授权登录**按钮，唤起小程序授权登录
3. 授权成功后，系统自动跳转至首页
4. 如需取消登录，点击**取消登录**按钮，返回上一个页面

### （三）注意事项

- 首次登录需完成手机号授权
- 登录状态保持时间为30天，超时需重新授权
- 点击返回按钮可返回上个页面；点击主页可跳转至主页

## 二、首页

### （一）操作前提

- 用户已完成登录授权
- 用户已配置所属门店和角色权限

### （二）操作步骤

1. 登录成功后自动进入**首页**
2. 首页展示内容包括：
   - 轮播图：展示运营配置的活动Banner
   - 金刚位：快捷入口（帮我找房、商铺租赁、大学生就业、企业合作等）
   - 通知/待办入口：显示未读通知和待办数量
3. 点击金刚位图标可跳转对应功能页面
4. 点击通知/待办区域可进入消息中心

### （三）注意事项

- 金刚位配置需通过PC端后台进行设置
- 轮播图支持跳转外部H5或内部页面
- 首页数据根据用户角色和权限动态展示

## 三、通知与待办

### （一）操作前提

- 用户已登录小程序
- 用户已配置通知权限

### （二）操作步骤

1. 在首页点击**通知/待办**入口，进入消息中心
2. 消息中心分为两个Tab页：
   - **通知**：展示系统推送的通知消息
   - **待办**：展示需要处理的业务待办事项
3. 点击通知/待办卡片可进入详情页查看具体内容
4. 待办详情页可执行处理操作

### （三）功能规则

**通知规则**：
- 仅展示未读的通知，已读的通知不展示
- 通知类型包括：客户签到提醒、系统公告、审批结果等
- 收到签到通知时，首页显示悬浮窗提示，3秒后自动关闭

**待办规则**：
- 仅展示未处理的待办，已处理的待办不展示
- 待办类型包括：客户分配确认、身份确认、审批流程等
- 待办需在指定时效内处理，超时将自动升级

**悬浮窗提示规则**：
- 收到一条签到通知则悬浮窗提示一次
- 一条签到通知只悬浮窗提示一次，不管是否已读已处理
- 三秒后自动关闭悬浮窗

### （四）注意事项

- 通知和待办数据实时同步，需保持网络畅通
- 已读通知可在"我的消息"中查看历史记录
- 待办处理完成后，相关通知自动标记为已处理

## 四、待办详情与身份确认

### （一）操作前提

- 用户有待办事项需要处理
- 待办类型为"确认身份和关联顾问"

### （二）操作步骤

1. 在待办列表点击待处理事项，进入**待办详情**页面
2. 查看待办详情信息，包括客户信息、业务类型、处理要求等
3. 点击**去确认身份和关联顾问**按钮
4. 在确认页面核对身份信息，选择关联顾问
5. 点击**确认**完成处理

### （三）功能规则

**多人处理场景**：
- 当一个项目存在两人或多人且被另一人处理时，其他未处理的人点击"去确认身份和关联顾问"，则弹窗提示"该待办已被他人处理"
- 系统实时同步处理状态，避免重复处理

**身份确认规则**：
- 必须选择与该客户关联的顾问
- 确认后该客户自动归属至所选顾问名下
- 确认操作不可撤销，请谨慎操作

### （四）注意事项

- 待办需在时效内处理，超时将影响业务流转
- 确认身份前请核实客户信息，避免关联错误
- 如遇系统提示"已被他人处理"，请联系相关同事确认

## 五、客户管理

### （一）操作前提

- 用户具有客户管理权限
- 用户已关联所属门店或战区

### （二）操作步骤

1. 在首页或菜单栏点击**客户管理**，进入客户列表页
2. 客户列表支持按条件筛选：
   - 客户状态：待跟进、跟进中、已成交、已失效
   - 客户来源：小程序、推荐、自然来访等
   - 时间范围：今日、本周、本月、自定义
3. 点击客户卡片进入**客户详情**页面
4. 在客户详情页可执行以下操作：
   - 查看客户基本信息、需求信息、跟进记录
   - 点击**跟进**添加跟进记录
   - 点击**分配**将客户分配给其他顾问
   - 点击**编辑**修改客户信息

### （三）功能规则

**客户分配规则**：
- 客户可分配至同门店或同战区的其他顾问
- 分配后原跟进记录保留，新负责人可查看历史记录
- 客户成交后，业绩归属按分配记录计算

**客户状态流转**：
- 待跟进 → 跟进中：首次添加跟进记录
- 跟进中 → 已成交：完成签约
- 跟进中 → 已失效：标记为无效客户

### （四）注意事项

- 客户信息涉及隐私，请妥善保管
- 跟进记录需真实完整，便于后续交接
- 客户分配前请与接收人沟通确认

## 六、推荐客户

### （一）操作前提

- 用户具有推荐客户权限
- 用户已完善个人信息（头像、昵称、经纪人类型）

### （二）操作步骤

1. 在首页或菜单栏点击**推荐客户**，进入推荐页面
2. 填写被推荐人信息：
   - 姓名（必填）
   - 手机号（必填）
   - 意向区域（选填）
   - 意向房型（选填）
   - 租金预算（选填）
   - 意向入住时间（选填）
3. 选择关联管家（如开启关联管家功能）
4. 点击**提交**完成推荐

### （三）功能规则

**推荐人资格**：
- 非在租用户可参与推荐活动
- 当前在租客户和同住人不可作为推荐人

**被推荐人资格**：
- 被推荐人需为新用户或未在租客户
- 同一新用户被多个推荐人推荐时，以第一个扫码的为准

**奖励规则**：
- 被推荐人入住超过X天（默认30天）后自动发券
- 奖励可按签约租期配置不同档位
- 签12个月可配置送6张100元券（每次限用1张）

**数据流转**：
- 推荐信息自动推送至尘锋SCRM系统
- 传输字段包括：被推荐人电话、推荐人电话、租金预算、意向入住时间、管家名称、管家电话、意向房型、意向地区、活动名称

### （四）注意事项

- 推荐信息提交后不可修改，请仔细核对
- 被推荐人签约后，奖励自动发放至推荐人账户
- 推荐活动数据统计仅在最新发布的活动表单中展示

## 七、我的

### （一）操作前提

- 用户已登录小程序

### （二）操作步骤

1. 点击底部导航栏**我的**，进入个人中心
2. 个人中心展示内容包括：
   - 头像、昵称、经纪人类型
   - 我的服务入口
   - 设置入口
3. 点击**编辑信息**可修改个人资料
4. 点击**头像**可更换头像
5. 点击**昵称**可修改昵称
6. 点击**经纪人类型**可查看或修改类型

### （三）功能规则

**个人信息管理**：
- 头像支持上传本地图片或拍照
- 昵称限制10个字符以内
- 经纪人类型由后台配置，部分类型不可修改

**我的服务**：
- 展示当前用户可用的服务功能
- 根据角色权限动态展示不同服务入口

### （四）注意事项

- 个人信息修改后实时生效
- 经纪人类型修改需经审批，审批通过后方可生效
- 退出登录后需重新授权登录

## 八、组织架构与成员管理

### （一）操作前提

- 用户具有组织架构查看或管理权限
- 用户为店长、战区负责人或管理员角色

### （二）操作步骤

1. 在PC端或小程序端进入**组织架构**页面
2. 组织架构以树形结构展示：
   - 公司层级
   - 战区层级
   - 门店层级
   - 个人层级
3. 点击**成员**进入成员管理页面
4. 成员管理功能：
   - 查看成员列表
   - 勾选成员进行批量分配
   - 点击成员查看详情
5. 批量分配操作：
   - 勾选2个及以上成员，**批量分配**按钮启用
   - 点击批量分配，选择目标顾问
   - 确认后完成分配

### （三）功能规则

**成员分配规则**：
- 展示所有非禁用身份为销售顾问的用户
- 分配成功后该销售顾问同步关联该项目
- 批量分配默认选择当前页成员

**权限控制**：
- 仅具有组织架构管理权限的用户可操作
- 普通成员仅可查看本部门成员

### （四）注意事项

- 成员分配后，相关客户数据同步更新归属
- 批量分配前请确认目标顾问信息
- 组织架构调整需通过PC端后台操作

## 九、权限管理

### （一）操作前提

- 用户具有权限管理权限
- 用户为管理员或超级管理员角色

### （二）操作步骤

1. 进入**权限**管理页面
2. 权限管理包括以下功能模块：
   - 导出权限：新增导出功能权限控制
   - 免责声明权限：新增免责声明权限
   - 推荐客户权限：新增"推荐客户"页面权限
   - 删除权限：新增删除权限，不受关联楼盘项目限制，勾选后同步拥有分配权限
   - 状态设置权限：勾选后同步拥有分配权限
   - 通知权限：勾选了"通知"权限，就默认勾起"确认身份和关联顾问"

### （三）功能规则

**权限关联规则**：
- 删除权限与分配权限联动：勾选删除权限后自动拥有分配权限
- 状态设置权限与分配权限联动：勾选状态设置权限后自动拥有分配权限
- 通知权限与确认身份权限联动：勾选通知权限默认勾选确认身份和关联顾问权限；如取消其中一个，那"通知"权限也取消勾选

**权限生效规则**：
- 只有"通知"权限的人才能收到通知和处理待办
- 权限修改后实时生效，已登录用户需重新登录后生效

### （四）注意事项

- 权限配置需谨慎，避免越权操作
- 关键权限变更需记录操作日志
- 权限调整前请确认影响范围

## 十、微信设置与引导关注

### （一）操作前提

- 用户具有微信设置权限
- 已配置企业微信相关参数

### （二）操作步骤

1. 进入**微信设置**页面
2. 配置企业微信二维码：
   - 上传门店企业微信二维码
   - 配置商铺招商人员二维码
   - 配置店长二维码
3. 配置**引导关注**页面：
   - 上传引导关注图片
   - 配置关注引导文案
   - 设置关注跳转链接
4. 保存配置后，小程序端自动生效

### （三）功能规则

**二维码展示规则**：
- 管家可通过APP端上传企业微信二维码
- 小程序端展示该二维码供客户扫码添加
- 若门店未配置具体管家，展示商铺招商人员或店长二维码

**引导关注规则**：
- 新用户首次进入小程序时展示引导关注页面
- 用户可选择关注或跳过
- 关注后自动跳转至首页

### （四）注意事项

- 二维码图片需清晰可识别
- 引导关注文案需符合微信规范
- 配置修改后需测试验证效果

# 配置说明

## 金刚位配置

在PC端后台配置小程序首页金刚位：

- **配置入口**：小程序管理 → 金刚位设置
- **配置项**：
  - 图标上传（建议尺寸80×80像素，文件不超过2MB）
  - 名称设置（限10字符）
  - 排序调整（1-5）
  - 链接配置（HTTP/PAGES类型）
  - 上下架状态
- **规则**：
  - 保存时必须满足上架数量≥4个
  - 下架后的金刚位不可进行排序调整
  - 链接类型为PAGES时，APPID为必填项

## 推荐活动配置

- **配置入口**：营销活动 → 推荐活动
- **关键配置项**：
  - 活动名称、活动时间
  - 参与人资格、排除参与人资格
  - 奖励条件（入住X天后发券）
  - 赠券模式（不赠券/推荐人/被推荐人/双方/按租期）
  - 颜色自定义（按钮颜色、页面背景颜色）
  - 关联选项开关（是否开启关联管家、是否开启关联企业选项）
  - 尘锋Feedback URL

## 通知待办配置

- **配置入口**：系统设置 → 通知配置
- **关键配置项**：
  - 通知类型开关
  - 悬浮窗提示开关
  - 通知时效设置
  - 待办升级规则

# 数据初始化

## 历史数据处理

1. **存量客户数据**：自动同步至客户管理模块
2. **历史推荐记录**：保留原有数据，新功能上线后按新规则执行
3. **权限数据**：根据角色自动映射至新权限体系

## 上线前准备

1. 配置金刚位和轮播图
2. 配置推荐活动规则
3. 配置权限体系
4. 导入组织架构和成员数据
5. 测试各功能模块
6. 培训一线业务人员
