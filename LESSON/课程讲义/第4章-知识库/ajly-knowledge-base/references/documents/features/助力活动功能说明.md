# 助力活动功能说明

助力活动功能说明

# 功能概述

助力活动功能主要是通过配置活动规则、发放券、配置小程序页面等能力，实现用户在小程序上转发助力活动，由目标用户点击助力，实现助力目标，并且自动发放优惠券。

在发放优惠券的功能上，可以设置多个助力梯度，当完成当前助力梯度目标后，自动发放优惠券。

# 小程序

## 活动页

活动页的进度条根据当前有几个活动阶段，均分进度条，每当完成一个阶段则平均涨一部分进度条。例如，有3个活动阶段，用户完成了第一个活动阶段，则进度条涨1/3。

【已得x元】，即为当前活动所有阶段已经获取的优惠券金额之和。

【最高x元】，即为当前活动所有阶段的优惠券金额之和。

【还差x人，助力成功再得x元租房红包】，即为当前活动阶段的目标人数，和优惠券总额。例如，用户当前阶段的目标人数为2人，助力成功后发放面额100元的券共计两张，则显示【还差2人，助力成功再得200元租房红包】

累计邀请人数，即为该活动该用户邀请人数的综合。例如，用户阶段1邀请了2人，阶段2邀请了3人，阶段3邀请了1人，则累计邀请了6人。

## 助力页

该页面样式固定

【您的好友最高可获得租房红包x元】，即为当前活动所有阶段的优惠券金额之和。

【还差x人，助力成功再得x元租房红包】，即为当前活动阶段的目标人数，和优惠券总额。例如，用户当前阶段的目标人数为2人，助力成功后发放面额100元的券共计两张，则显示【还差2人，助力成功再得200元租房红包】

邀请人不可以为自己助力，即使符合条件。

# 尘锋数据流转

在创建活动时，在最底部的【外部Feedback URL】填入尘锋-线索对接中的Feedback UR，从而实现关键数据自动传到尘锋。


| 字段名 | 解释 |
| --- | --- |
| userPhone | 电话 |
| roleTypeName | 角色类型 |
| referrerPhone | 关联的邀请人 |
| dealStatusName | 成交状态 |
