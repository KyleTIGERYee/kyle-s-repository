# 小程序金刚位自定义功能说明

小程序金刚位自定义功能说明

# 概述

本功能支持运营人员在PC端后台自定义配置小程序首页金刚区的展示内容，包括图标、名称、排序、链接等，实现对小程序首页金刚区入口的灵活管理。通过本功能，运营人员可根据业务需求随时调整金刚区展示内容，提升小程序首页的运营灵活性。

金刚区是小程序首页的核心入口区域，展示常用功能入口，如"帮我找房"、"商铺租赁"、"大学生就业"、"企业合作"等。本功能支持最多5个金刚位配置，每个金刚位可独立设置名称、图标、链接、排序及上下架状态。

# 功能模块

## 功能一：金刚位配置管理

### （一）操作前提

- 拥有PC端【小程序首页设置】菜单权限
- 金刚位图标文件需符合格式要求（建议尺寸80×80像素，文件大小不超过2MB）

### （二）操作步骤

1. 登录PC端，进入【系统设置】-【小程序首页设置】-【首页金刚区】页面
2. 查看当前金刚位配置列表，系统默认展示5个金刚位配置项
3. 点击**设置金刚区**按钮，打开金刚位配置弹窗
4. 对每个金刚位进行以下配置：
   - **图标上传**：点击图标区域上传图片，支持鼠标移入后下载、编辑（替换）操作；点击图片可放大预览
   - **名称设置**：在名称输入框填写金刚位显示名称，限制10个字符
   - **排序设置**：通过下拉选择框设置金刚位展示顺序（1-5），数字越小排序越靠前
   - **链接配置**：
     - 选择链接类型：HTTP（外部链接）或PAGES（小程序内部页面）
     - 如选择PAGES类型，APPID为必填项
     - 填写完整的链接地址
   - **上下架状态**：点击**上架**或**下架**按钮控制金刚位展示状态
5. 配置完成后，点击**保存**按钮提交

### （三）注意事项

- 保存时系统会校验上架数量必须大于等于4个，否则无法保存
- 下架后的金刚位不可进行排序调整
- 首次上线时，系统会复用现有的图片、名称和排序作为默认值
- **上架**按钮与**下架**按钮为互斥状态，同一时间仅可选择一个

## 功能二：金刚位排序规则

### （一）排序逻辑说明

金刚位排序采用数值优先级机制，排序值越小展示位置越靠前。

**场景一：下架操作对排序的影响**

假设当前A/B/C/D/E五个金刚位均处于上架状态，排序依次为1/2/3/4/5：
- 将C下架后，排序自动调整为A(1)/B(2)/D(3)/E(4)/C(5)
- 再将B下架后，排序变更为A(1)/D(2)/E(3)/C(4)/B(5)

**场景二：上架操作对排序的影响**

假设当前A/B/C/D/E五个金刚位均处于下架状态：
- 将B上架后，排序变更为B(1)/A(2)/C(3)/D(4)/E(5)
- 再将D、E上架后，排序变更为B(1)/D(2)/E(3)/A(4)/C(5)

### （二）默认配置规则

首次上线时，系统默认配置如下：

| 金刚位 | 默认状态 | 说明 |
| --- | --- | --- |
| 帮我找房 | 上架 | 默认展示 |
| 商铺租赁 | 上架 | 默认展示 |
| 大学生就业 | 上架 | 默认展示 |
| 企业合作 | 上架 | 默认展示 |
| 自助看房 | 下架 | 默认不展示 |

### （三）重新上架规则

- 下架后重新上架的金刚位，默认排在已上架金刚位的最后位置
- 如需调整位置，需手动修改排序值

## 功能三：链接配置规则

### （一）链接类型说明

| 链接类型 | 适用场景 | APPID要求 |
| --- | --- | --- |
| HTTP | 跳转外部H5页面 | 非必填 |
| PAGES | 跳转小程序内部页面 | 必填 |

### （二）配置示例

- **HTTP类型**：`https://www.example.com/page`
- **PAGES类型**：`pages/index/index`，需同时填写APPID

### （三）注意事项

- 链接为必填项，保存时必须填写
- 选择PAGES类型时，APPID字段为必填校验项
- 建议优先使用PAGES类型跳转，以获得更好的用户体验


# 配置说明

## 偏好设置

在【小程序首页设置】中可配置以下参数：

- **金刚位最大数量**：固定为5个，不可调整
- **名称最大长度**：10个字符
- **图标文件大小限制**：2MB
- **最小上架数量**：4个（保存校验）


# 注意事项

1. **保存校验**：保存时必须满足上架数量≥4个，否则系统提示"上架数量不能少于4个"
2. **下架限制**：下架状态的金刚位不可调整排序，需先上架后才能修改排序
3. **图标规范**：建议使用PNG格式透明背景图标，确保在小程序端展示效果
4. **链接有效性**：配置HTTP链接时，请确保链接地址可正常访问
5. **缓存刷新**：配置保存后，小程序端可能存在缓存，建议提示用户下拉刷新页面
6. **互斥状态**：上架和下架按钮为互斥状态，点击其中一个后另一个自动失效
