# 自助看房系统功能说明

自助看房系统功能说明

# 概述

自助看房系统是为租客提供的一种无需管家陪同、通过人脸识别和智能门锁技术实现自主看房的解决方案。租客可通过小程序预约看房时间，到达门店后通过人脸识别验证身份，自助开启楼栋门禁和房间门锁进行看房。

本系统适用于安居乐寓旗下已配置智能门锁和门禁设备的门店，支持战区和门店在PC端进行看房单管理、偏好设置和楼栋配置，同时在小程序端提供门店浏览、心愿单管理、看房申请和自助开门等功能。

# 功能模块

## 一、PC端功能

### （一）菜单路径

【营销推广】-【自助看房】

### （二）看房单管理

#### 1. 查询功能

**操作步骤**：

1. 点击【营销推广】-【自助看房】进入看房单管理页面
2. 在查询区域选择筛选条件：
   - **战区**：下拉框选择，支持搜索
   - **门店**：下拉框选择，支持搜索
   - **物业地址**：下拉框选择，支持搜索
3. 点击**查询**按钮，系统根据条件筛选展示数据
4. 点击**重置**按钮，清空所有查询条件

#### 2. 列表展示

列表展示以下字段信息：

| 字段名 | 说明 |
| --- | --- |
| 战区 | 门店所属战区 |
| 门店 | 门店名称 |
| 物业地址 | 楼栋物业地址 |
| 房间号 | 看房单关联的房间号，多个房间以标签形式展示 |
| 看房人姓名 | 申请人姓名 |
| 看房人手机号 | 默认脱敏展示，点击**查看**图标可显示完整号码 |
| 看房时间段 | 预约看房的具体时间段 |
| 看房状态 | 待看房、已看房、已过期、已失效 |
| 看房评价 | 仅"已看房"状态显示评价（满意/不满意） |

**注意事项**：

- 只有"已看房"状态的看房单才显示评价，"待看房"状态的评价不展示
- 手机号默认脱敏展示（如：138****8000），点击**查看**图标可切换显示完整号码

#### 3. 导出功能

点击**导出**按钮，可将当前查询结果导出为Excel文件。

### （三）偏好设置

#### 1. 功能说明

配置同一楼栋多少天内不可重复预约自助看房，用于控制租客频繁预约同一楼栋。

#### 2. 操作步骤

1. 在看房单管理页面，点击**偏好设置**按钮
2. 在弹窗中设置天数：
   - 输入范围：0-365天
   - 0表示不限制
3. 点击**保存**，完成配置

#### 3. 业务规则

- **判断逻辑**：在设置的天数内，只要同一楼栋有开门记录（开房门或开门禁均算），则不可重复申请
- **天数计算**：包括今天（如设置2天，则包括今天和昨天零点以后）
- **拦截时机**：在生成申请看房单时触发校验
- **提示信息**：如不满足条件，提示"该楼栋最近已经有已看房的记录，不可重复预约"

### （四）楼栋设置

#### 1. 功能说明

配置哪些楼栋支持自助看房功能，控制楼栋在小程序自助看房列表中的展示。

#### 2. 操作步骤

1. 在看房单管理页面，点击**楼栋设置**按钮
2. 在弹窗中查询楼栋：
   - 选择**战区**、**门店**筛选条件
   - 点击**查询**按钮
3. 在列表中勾选需要设置为自助看房的楼栋
4. 点击**设置自助看房楼栋**，完成配置

#### 3. 数据来源与过滤规则

- **数据来源**：从楼栋数据读取
- **剔除规则**：没有分配房间的楼栋直接不展示
- **批量设置**：支持批量勾选多个楼栋进行设置

#### 4. 门店展示规则

| 场景 | 普通门店列表 | 自助看房门店列表 |
| --- | --- | --- |
| 门店设置了自助看房楼栋 | 展示 | 展示 |
| 门店取消所有自助看房楼栋设置 | 展示 | 不展示 |
| 门店下所有房间都被租了 | 展示 | 不展示 |

**注意事项**：

- 门店取消所有自助看房楼栋设置后，该门店在自助看房列表中不展示，但在普通门店列表正常展示
- 门店下所有房间都被租了，该门店在自助看房列表中不展示

## 二、小程序端功能

### （一）入口

#### 1. 首页图标入口

在小程序首页的功能图标区域，点击**自助看房**图标进入自助看房门店列表。

#### 2. 我的看房入口

在小程序首页左下角悬浮按钮，点击**我的看房**进入我的看房列表页面。

### （二）门店列表

#### 1. 功能说明

展示所有已设置自助看房楼栋的门店，支持距离排序和价格排序。

#### 2. 展示规则

- 但凡有自助看房楼栋的门店都在列表中展示
- 门店定位取门店位置坐标

#### 3. 操作步骤

1. 进入自助看房门店列表页面
2. 查看门店信息：
   - 门店名称、标签（如"近地铁"、"拎包入住"）
   - 地址、距离
   - 房源缩略图、价格
3. 点击门店卡片进入门店详情页

#### 4. 加入心愿单按钮

- 仅设置了自助看房的房间显示**加入心愿单**按钮
- 普通未设置自助看房的房源不显示该按钮

### （三）心愿单

#### 1. 功能说明

心愿单类似于购物车，租客可将感兴趣的自助看房房源加入心愿单，统一进行预约申请。

#### 2. 加入心愿单

**操作步骤**：

1. 在房间详情页，点击**加入心愿单**按钮
2. 系统校验：
   - 同一房间不可重复添加
   - 已租房间不可添加
3. 添加成功后，房间加入心愿单列表

#### 3. 心愿单列表

**展示规则**：

- 同一栋房间展示在一起
- 不同门店分开展示
- 不同楼栋在同一门店内分开展示

**已租判断**：

- 已登记租客或已交押金均视为已租
- 已租房间标记为"已租"且不可点击，只能删除

#### 4. 管理功能

- **删除**：点击删除图标，可将房间从心愿单移除
- **勾选预约**：勾选楼栋后，点击**申请自助看房**按钮进行预约

### （四）申请看房

#### 1. 操作前提

- 已登录小程序
- 心愿单中已勾选需要预约的楼栋
- 同一楼栋在偏好设置天数内无开门记录
- 待看房数量未达到10条上限

#### 2. 操作步骤

1. 在心愿单页面，勾选需要预约的楼栋
2. 点击**申请自助看房**按钮
3. 填写申请人信息：
   - **真实姓名**（必填）
   - **身份证号**（必填）
4. 选择预约时间：
   - 日期：当天起6天内（包括当天）
   - 时间段：2小时为一个时间段（如8:00-10:00、10:00-12:00）
5. 点击**提交申请**
6. 授权"看房提醒通知"后，完成申请

#### 3. 二要素校验

- **校验时机**：提交时进行校验
- **校验内容**：身份证合法性校验
- **注意事项**：姓名与身份证号不匹配的情况可能校验不出来

#### 4. 看房单生成规则

- 按楼栋生成看房单
- 一次申请多个楼栋则生成多条看房单
- 每条看房单独立计算状态

#### 5. 数量限制

- 同时存在的待看房最多10条
- 计算方式：已申请数量 + 本次申请数量
- 已看房的不计入限制
- 超过10条则提示"超过十条的话就不给申请了"

### （五）我的看房

#### 1. 有效列表

包含以下状态的看房单：

**【待看房】**
- 已申请成功，等待看房
- 可操作：查看详情、开启门禁/房门


#### 2. 无效列表

包含以下状态的看房单：

**【已看房】**
- 已完成看房
- 可操作：查看详情、查看评价

**【已过期】**
- 预约时间已过，未完成看房
- 由晚上9点定时任务自动变更

**【已失效】**
- 门店取消自助看房设置导致
- 仅对待看房状态的看房单生效

#### 3. 状态流转

| 原状态 | 新状态 | 触发条件 |
| --- | --- | --- |
| 待看房 | 已看房 | 开门后1小时自动变更（以最早开门时间计算） |
| 待看房 | 已过期 | 当天晚上9点定时任务，未看房则过期 |
| 待看房 | 已失效 | 门店取消自助看房设置 |

**注意事项**：

- 已看房状态变更以最早开门时间往后推1小时计算（包括开房门或开门禁）
- 晚上9点定时任务将未看房的待看房单置为已过期（原计划8点，后改为9点）

### （六）看房执行

#### 1. 人脸识别验证

**操作步骤**：

1. 在看房详情页，点击**开启楼栋门禁**或**开启房门**
2. 如未进行人脸识别，系统自动唤起人脸识别页面
3. 进行人脸识别验证
4. 验证成功后，方可开启门禁/房门

**验证规则**：

- 每一栋楼都要单独进行人脸识别
- 同一看房单内不同楼栋也需分别验证
- 人脸识别失败场景：非真实人脸、不匹配的人脸、照片人脸均会失败

#### 2. 开启楼栋门禁

**操作步骤**：

1. 人脸识别验证通过后，点击**开启楼栋门禁**
2. 系统展示该楼栋所有门禁设备
3. 选择需要开启的门禁
4. 点击开启

**时间限制**：

- 5分钟内可无限次开启
- 超过5分钟后不可开启
- 只有在预约日的8:00-20:00才可以开启


#### 3. 开启房门

**操作步骤**：

1. 人脸识别验证通过后，在看房详情页找到需要开启的房间
2. 点击**开启房门**
3. 等待门锁开启

**时间限制**：

- 5分钟内可无限次开启
- 超过5分钟后不可开启
- 门锁和门禁的5分钟限制分开计算，互不影响
- 只有在预约日的8:00-20:00才可以开启


**已租房间**：

- 已租房间显示"已租"标记
- 不可开启房门

#### 4. 尘锋线索对接（商机创建）

**触发时机**：

用户成功开启楼栋门禁或房间门后，系统自动触发尘锋线索对接，创建联系人信息。

**传输字段**：

| 字段名 | 说明 | 来源 |
| --- | --- | --- |
| 姓名 | 看房人真实姓名 | 看房申请时填写的姓名 |
| 手机号 | 看房人手机号 | 小程序登录用户手机号 |

**对接方式**：

- 通过尘锋SCRM的线索对接功能进行传输
- 数据实时同步至尘锋系统
- 尘锋系统根据接收到的信息自动创建联系人/线索

**业务价值**：

- 自助看房用户自动进入尘锋线索池
- 便于后续管家跟进转化
- 实现自助看房到签约的全链路追踪

#### 5. 看房评价

**操作步骤**：

1. 开完房门后，退出并重新进入看房详情页
2. 页面展示评价卡片"本次看房体验如何？"
3. 选择**满意**或**不满意**
4. 评价后推送管家联系卡片

**注意事项**：

- 评价卡片默认不展示，需重新进入页面后才显示
- 不点击评价，卡片会一直存在
- 评价完成后，无论满意或不满意，均推送管家卡片

#### 6. 管家联系

在管家卡片中，可进行以下操作：

- **添加微信**：点击后展示管家微信二维码
- **联系电话**：点击后展示管家电话，支持一键拨打

## 三、定时任务

### （一）早上8点提醒任务

**执行时间**：每天早上8点

**任务内容**：

- 对当天8点之前申请的看房单发送短信和推送提醒
- 超过8点申请的，不再发送提醒（定时任务已执行完毕）

**推送规则**：

- 如有多栋楼，推送只展示其中一栋
- 点击推送后跳到看房单列表页面

### （二）晚上9点过期处理任务

**执行时间**：每天晚上9点

**任务内容**：

- 将当天未看房的待看房单置为已过期
- 原计划8点执行，后改为9点执行

**注意事项**：

- 7点半开门但未满1小时（未变成已看房）的看房单，在9点会被置为已过期
- 已看房的不受该任务影响

# 配置说明

## 一、偏好设置

**配置路径**：PC端【营销推广】-【自助看房】-**偏好设置**

**配置项**：

| 配置项 | 说明 | 默认值 | 取值范围 |
| --- | --- | --- | --- |
| 预约限制天数 | 同一楼栋多少天内不可重复预约自助看房 | 7天 | 0-365天 |

**配置影响**：

- 0表示不限制
- 设置后，在生成申请看房单时触发校验
- 判断同一楼栋在设置天数内是否有开门记录（开门禁或开房门均算）

## 二、楼栋设置

**配置路径**：PC端【营销推广】-【自助看房】-**楼栋设置**

**配置项**：

- 勾选需要设置为自助看房的楼栋

**配置影响**：

- 已设置的楼栋在小程序自助看房列表中展示
- 未设置的楼栋在小程序自助看房列表中不展示
- 门店取消所有自助看房楼栋设置后，该门店在自助看房列表中不展示

# 业务规则汇总

## 一、看房单状态流转规则

| 原状态 | 新状态 | 触发条件 | 说明 |
| --- | --- | --- | --- |
| 待看房 | 已看房 | 开门后1小时自动变更 | 以最早开门时间计算（包括开房门或开门禁） |
| 待看房 | 已过期 | 晚上9点定时任务 | 当天未看房的待看房单自动过期 |
| 待看房 | 已失效 | 门店取消自助看房设置 | 仅对待看房状态的看房单生效 |

## 二、重复预约限制规则

1. 根据PC端偏好设置的天数进行限制
2. 开门记录包括：开房门、开门禁
3. 天数计算包括今天（如2天包括今天和昨天零点以后）
4. 拦截时机：生成申请看房单时
5. 不满足条件时提示"最近已经有已看房的"

## 三、门禁/房门开启规则

1. 5分钟内可无限次开启
2. 超过5分钟后不可开启
3. 门锁和门禁的5分钟限制分开计算，互不影响
4. 门禁是跟楼栋绑定的，不是跟看房单绑定的

## 四、看房单数量限制规则

1. 同时存在的待看房最多10条
2. 计算方式：已申请数量 + 本次申请数量
3. 已看房的不计入限制
4. 超过10条则不给申请

## 五、数据展示规则

1. 仅"已看房"状态显示评价，待看房状态的评价不展示
2. 楼栋展示过滤：仅展示设置了自助看房的楼栋
3. 已租判断：已登记租客或已交订金均视为已租
4. 已租房间在心愿单中标记且不可点击，只能删除
