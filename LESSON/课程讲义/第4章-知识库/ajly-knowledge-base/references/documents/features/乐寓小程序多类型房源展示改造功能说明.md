# 乐寓小程序多类型房源展示改造功能说明

乐寓小程序多类型房源展示改造功能说明

# 概述

本功能针对乐寓小程序房源展示体系进行升级改造，从原有仅展示公司自营收储房源的模式，扩展为支持同时展示三类房源：公司自营收储房源、个人房东房源、其他公司主体房源。通过在门店层级增加"自营"标签进行区分，实现多类型房源的统一展示与管理，同时保持户型、房间描述、价格展示等核心体系的一致性。

本功能适用于小程序端（C端用户）和PC端后台管理系统，涉及房源展示、门店管理、搜索筛选等核心场景。

# 功能模块

## 一、房源类型定义与管理

### （一）房源类型定义

系统支持以下三种房源类型：

| 房源类型 | 标签展示 | 房源来源说明 |
|---------|---------|-------------|
| 自营房源 | 展示"自营"标签 | 公司自行收储并统一运营的房源 |
| 个人房东房源 | 不展示标签 | 个人房东委托运营的房源 |
| 其他公司主体房源 | 不展示标签 | 其他企业或机构委托运营的房源 |

### （二）后台门店类型配置

#### 1. 操作前提

- 用户具有门店管理权限
- 用户为店长、战区负责人或运营中台角色

#### 2. 操作步骤

1. 登录PC端后台，进入【门店管理】-【门店列表】
2. 点击**新增门店**或选择已有门店点击**编辑**
3. 在门店基础信息中，选择`门店类型`字段：
   - 选择**自营**：该门店下所有房源展示"自营"标签
   - 选择**个人房东**：该门店下房源不展示标签
   - 选择**其他公司主体**：该门店下房源不展示标签
4. 填写其他门店信息（门店名称、地址、店长等）
5. 点击**保存**，完成门店类型配置

#### 3. 功能规则

**门店类型与标签展示规则**：
- 门店类型为"自营"时，小程序端该门店及下属房源展示"自营"标签
- 门店类型为"个人房东"或"其他公司主体"时，小程序端不展示特殊标签
- 门店类型一旦设置，该门店下所有房源统一遵循该类型的展示规则

**历史门店处理**：
- 存量门店默认设置为"自营"类型
- 运营人员可根据实际情况修改存量门店类型

### （三）楼栋与房源管理

#### 1. 操作前提

- 已完成门店创建并配置门店类型
- 用户具有楼栋管理权限

#### 2. 操作步骤

1. 进入【门店管理】-【楼栋管理】
2. 点击**新增楼栋**，选择所属门店
3. 填写楼栋信息（楼栋名称、楼层数等）
4. 点击**保存**，完成楼栋创建
5. 进入【房源管理】，点击**新增房源**或**批量导入**
6. 选择所属门店和楼栋，录入房源信息
7. 房源的展示类型自动继承所属门店的类型配置

#### 3. 功能规则

**房源类型继承规则**：
- 房源类型自动继承所属门店的类型设置
- 不支持单独修改单个房源的类型
- 如需调整房源类型，需调整所属门店的类型

## 二、小程序首页房源列表改造

### （一）操作前提

- 用户进入小程序首页
- 系统已完成门店类型配置

### （二）功能展示

#### 1. 房源卡片展示

**改造前**：房源卡片仅展示房源基础信息

**改造后**：房源卡片增加门店类型标签展示

**展示规则**：
- 房源所属门店类型为"自营"时，卡片左上角展示"自营"标签
- 房源所属门店类型为"个人房东"或"其他公司主体"时，不展示标签

**房源卡片信息结构**：
```
┌─────────────────────────┐
│ [自营标签]               │  ← 仅自营房源展示
│                         │
│    [房源封面图]          │
│                         │
│ 门店名称·距离            │
│ 户型 | 面积              │
│ ¥XXXX/月 起            │
└─────────────────────────┘
```

#### 2. 列表排序规则

**排序逻辑保持不变**：
- 已授权定位：按用户定位由近到远排序
- 未授权定位：优先展示"热门门店"房源；若无热门门店，则按后台"店面排序"展示

**标签不影响排序**：房源类型标签不参与排序逻辑，仅作为展示标识

### （三）注意事项

- "自营"标签样式需与UI设计稿保持一致
- 标签展示位置固定于房源卡片左上角
- 标签展示不影响房源其他信息的展示布局

## 三、门店详情页改造

### （一）操作前提

- 用户在小程序首页点击门店卡片进入门店详情页

### （二）功能展示

#### 1. 门店头部信息展示

**改造前**：展示门店名称、地址、评分、距离等基础信息

**改造后**：在门店名称旁增加门店类型标签展示

**展示规则**：
- 门店类型为"自营"时，门店名称右侧展示"自营"标签
- 门店类型为"个人房东"或"其他公司主体"时，不展示标签

**门店头部信息结构**：
```
┌─────────────────────────┐
│                         │
│ 门店名称 [自营]          │  ← 仅自营门店展示标签
│ ⭐4.8 · 距您1.2km        │
│ 深圳市宝安区福永街道...  │
│                         │
└─────────────────────────┘
```

#### 2. 门店房源列表展示

- 门店下所有房源卡片统一遵循门店类型标签展示规则
- 自营门店下所有房源卡片均展示"自营"标签
- 非自营门店下房源卡片不展示标签

### （三）注意事项

- 门店详情页的标签样式需与首页列表保持一致
- 标签展示不影响门店其他信息的展示布局

## 四、房间详情页改造

### （一）操作前提

- 用户在房源列表或门店详情页点击房源卡片进入房间详情页

### （二）功能展示

#### 1. 房间头部信息展示

**改造前**：展示房间号、户型、面积等基础信息

**改造后**：在房间信息区域增加所属门店类型标签展示

**展示规则**：
- 房间所属门店类型为"自营"时，房间号旁展示"自营"标签
- 房间所属门店类型为"个人房东"或"其他公司主体"时，不展示标签

**房间头部信息结构**：
```
┌─────────────────────────┐
│                         │
│ A栋301 [自营]            │  ← 仅自营房源展示标签
│ 单间 | 25㎡ | 朝南       │
│                         │
│ ¥1,500/月              │
│                         │
└─────────────────────────┘
```

#### 2. 房间信息展示统一性

**以下信息展示保持统一，不受房源类型影响**：
- 户型展示：统一展示户型名称和布局图
- 房间描述：统一展示房间配置、设施信息
- 价格展示：统一展示挂牌价、优惠价、押金标准
- 服务费用：统一展示服务费、水电费等费用标准
- 房间图片：统一展示封面图、卧室图、客厅图等

### （三）注意事项

- 房间详情页的标签样式需与列表页保持一致
- 标签仅作为房源来源标识，不影响价格、服务等核心信息的展示

## 五、搜索与筛选改造

### （一）操作前提

- 用户使用小程序搜索功能或筛选功能

### （二）功能展示

#### 1. 搜索结果列表展示

- 搜索结果中的房源卡片统一遵循房源类型标签展示规则
- 自营房源展示"自营"标签
- 非自营房源不展示标签

#### 2. 地图找房页面展示

- 地图上的房源标记点展示规则保持不变
- 点击标记点弹出的房源信息卡片遵循标签展示规则

#### 3. 筛选功能

**本期暂不增加按房源类型筛选功能**

- 保留现有筛选条件：通勤、距离、租金、房型
- 房源类型不作为筛选条件

### （三）注意事项

- 搜索结果和地图找房的标签展示需与首页列表保持一致
- 筛选功能暂不支持按房源类型筛选

# 权限说明

| 角色 | 权限范围 |
|-----|---------|
| 管家 | 可查看所有类型门店和房源信息，可录入房源信息 |
| 店长 | 管家所有权限 + 可修改本门店类型配置 |
| 战区负责人 | 店长所有权限 + 可管理战区内所有门店类型配置 |
| 运营中台 | 全量门店类型配置权限、数据查看、导出权限 |

# 配置说明

## 门店类型配置

**配置入口**：【系统设置】-【字典设置】-【门店类型】

**配置项**：
- **自营**：公司自行收储并统一运营的房源
- **个人房东**：个人房东委托运营的房源
- **其他公司主体**：其他企业或机构委托运营的房源

**配置规则**：
- 门店类型字典项支持启用/禁用
- 禁用后，该类型门店无法新增，但存量门店不受影响
- 类型名称支持修改，但类型编码固定

## 标签样式配置

**配置入口**：【小程序管理】-【UI配置】-【标签样式】

**配置项**：
- **标签文字**：默认为"自营"
- **标签背景色**：默认为品牌主色
- **标签文字色**：默认为白色
- **标签圆角**：默认为4px
- **标签字号**：默认为12px

# 数据初始化

## 存量数据处理

1. **存量门店**：
   - 所有存量门店默认设置为"自营"类型
   - 运营人员可根据实际情况在后台修改门店类型

2. **存量房源**：
   - 存量房源自动继承所属门店的类型设置
   - 无需单独处理

3. **历史数据展示**：
   - 小程序端历史展示记录不受改造影响
   - 改造后新产生的展示记录按新规则执行

## 上线前准备

1. 配置门店类型字典项
2. 设置"自营"标签样式
3. 确认存量门店类型（默认为自营）
4. 测试各类型门店和房源的展示效果
5. 培训运营人员门店类型配置流程

# 注意事项

- **标签唯一性**：目前仅"自营"房源展示标签，个人房东和其他公司主体房源不展示标签，保持页面简洁
- **类型继承性**：房源类型完全继承所属门店类型，不支持单独设置房源类型
- **排序不受影响**：房源类型不参与任何排序逻辑，仅作为展示标识
- **筛选暂不开放**：本期暂不支持按房源类型筛选，如需该功能请在后续版本迭代
- **统一展示体系**：户型、房间描述、价格展示等核心信息展示不受房源类型影响，保持统一
- **数据一致性**：门店类型修改后，该门店下所有房源的展示类型实时同步更新
