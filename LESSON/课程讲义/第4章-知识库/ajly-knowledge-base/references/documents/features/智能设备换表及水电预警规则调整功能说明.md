# 智能设备换表及水电预警规则调整功能说明

智能设备换表及水电预警规则调整功能说明

# 概述

本功能针对智能水电表设备管理进行优化升级，包含三大核心模块：智能设备换表功能、智能设备列表字段扩展、水电预警规则调整。通过新增换表操作能力、完善设备信息展示、优化异常用量预警机制，提升设备管理效率和问题处理速度。

本功能适用于 PC 端租务管理系统和 APP 端乐寓管理应用，主要面向管家、店长等业务人员。

# 功能模块

## 一、智能设备换表功能

### （一）操作前提

- 房间已绑定智能电表或智能水表
- 管家/店长具有设备管理权限
- 新表已完成物理安装并具备入网条件

### （二）PC 端换表操作

#### 1. 入口位置

- **电表换表**：【智能硬件】→【智能电表】→ 房间详情页 → **换表**按钮（位于删除电表按钮前）
- **水表换表**：【智能硬件】→【智能冷水表/热水表】→ 房间详情页 → **换表**按钮（位于抄表按钮前）

#### 2. 电表换表信息录入

点击**换表**按钮后，弹出电表换表信息填写页面，需录入以下字段：

| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| `电表品牌` | 是 | 下拉选择，不能为空 |
| `电表编号` | 是 | 文本输入，唯一性校验 |
| `网关编号` | 是 | 文本输入，不能为空 |
| `网关端口号` | 是 | 文本输入，不能为空 |
| `旧表读数` | 是 | 数字，≥0，小数点后两位 |


#### 3. 水表换表信息录入

点击**换表**按钮后，弹出水表换表信息填写页面，需录入以下字段：

| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| `水表品牌` | 是 | 下拉选择，不能为空 |
| `水表编号` | 是 | 文本输入，唯一性校验 |
| `水表类型` | 是 | 系统默认选择：冷水表/热水表 |
| `网关编号` | 是 | 文本输入，不能为空 |
| `网关端口号` | 是 | 文本输入，不能为空 |
| `旧表读数` | 是 | 数字，≥0，小数点后两位 |


#### 4. 保存与确认

- 点击**保存**按钮，系统校验字段格式
- 校验通过后弹出确认提示框："换表后，将生成待结算电/水量的账单，并自动解绑原表，已录入的当前读数会生成一条新的抄表记录，请确认录入信息无误。"
- 点击**确认**按钮，完成换表操作
- 校验失败时，页面顶部显示弱提示报错信息

### （三）APP 端换表操作

#### 1. 入口位置

- **电表换表**：【智能电表】→ 房间详情页 → **换表**按钮
- **水表换表**：【智能水表】→ 房间详情页 → **换表**按钮

#### 2. 操作流程

1. 点击**换表**按钮，进入换表页面
2. 填写新表基本信息（字段同 PC 端）
3. 点击**保存**，弹出确认提示框
4. 点击**确认**，完成换表操作
5. 提示"已完成换表操作！"，返回智能电表/水表房间页面

### （四）换表后触发任务

换表操作完成后，系统自动执行以下任务：

#### 1. 生成待结算账单

- 对旧表进行抄表，计算待结算电量/水量
- 生成一条旧表的后付费账单记录
- 账单金额 =（换表时原表读数 - 上次止数）× 单价

#### 2. 更新设备详情

设备详情页更新为新表信息：
- 电表/水表编号
- 电表/水表品牌
- 网关编号
- 网关品牌
- 待结算电量/水量（换表当日为 0，随后按用量正常更新）
- 当前读数（录入的当前读数，随后按用量正常更新）

#### 3. 新增用能记录

- **用水记录**（水表换表）：抄表时间为换表当日，水表编号为新表编号，水表读数为录入的当前读数，用水量默认为 0
- **用电记录**（电表换表）：抄表时间为换表当日，电表编号为新表编号，电表读数为录入的当前读数，用电量默认为 0

#### 4. 新增操作记录

- 操作时间：换表时保存后点击**确认**的时间
- 操作人：当前操作用户
- 操作类型：**换表**（新增类型）
- 水表/电表编号：新表编号
- 备注：固定文案【PC 端换电表】/【PC 端换冷水表】/【PC 端换热水表】/【APP 端换电表】/【APP 端换水表】

| 新增字段 | 字段说明 | 数据来源 |

| `网关编号` | 水表所属网关的编号 | 绑定水电表时录入的数据 |

#### 3. 智能热水表列表

新增字段位置：位于水表编号后，付费模式前

| 新增字段 | 字段说明 | 数据来源 |

## 三、水电预警规则调整

### （一）业主自留房剔除预警(功能未上线)

#### 1. 背景说明

业主自留房在系统中无统一命名规则且无房态标签，水电预警规则全局应用，导致部分公寓业主房源用水用电量超过预警值被拦截，仅生成用电记录，不生成后付费账单记录。

#### 2. 调整方案

待"房态需求"上线后，支持自定义房态，新增业主房标签，将此部分房源从水电预警/拦截的数据范围内剔除。

### （二）预警逻辑调整

#### 1. 计算方式变更

水电预计/拦截逻辑恢复为**按日用量平均值**计算。

#### 2. 预警值标准

根据系统已有租客数据摸底和 AI 估算，单人单房夏季月电量均值在 300 度、水量均值在 5 吨，考虑到实际场景存在单日峰值，暂拟预警值如下：

| 类型 | 日均值预警标准 | 参考场景 |
|-----|---------------|---------|
| 水 | 1 吨/天 | 成年人洗澡 15 分钟的用水量通常在 0.09 到 0.225 吨之间 |
| 电 | 50 度/天 | 2 匹空调 24 小时耗电量约 11.28 度；1.5 匹空调 24 小时耗电量约 8 度 |

#### 3. 离线后重新上线计算规则

当某水表或电表离线后重新上线，根据离线日期和用量计算平均值：

**计算公式**：
```
日均用量 =（恢复在线时读数 - 离线时读数）÷ 离线天数
```

**示例**：
- 9 月 1 日电表抄表 100 度，随后电表离线
- 9 月 16 日恢复在线，抄表 250 度
- 差值 = 250 - 100 = 150 度
- 离线天数 = 15 天
- 日均值 = 150 ÷ 15 = 10 度（在异常值范围内，不触发预警）

### （三）预警处理流程

1. 系统每日计算各房间水电用量日均值
2. 超过预警值时，标记为异常用量
3. 异常用量仅生成用电/用水记录，不生成后付费账单记录
4. 业务人员可在设备详情页查看异常标记并人工核实

# 权限说明

| 角色 | 权限范围 |
|-----|---------|
| 管家 | 可查看智能设备列表、执行换表操作、查看设备详情 |
| 店长 | 可查看智能设备列表、执行换表操作、查看设备详情、查看操作记录 |
| 运维人员 | 可查看所有设备信息、处理异常预警、导出设备数据 |

# 注意事项

- **换表前确认**：换表前请确认新表已完成物理安装并具备入网条件，避免换表后设备离线
- **当前读数录入**：换表时录入的当前读数将作为新表的初始读数，请确保读数准确
- **旧表账单结算**：换表后会自动生成旧表的待结算账单，请及时处理
- **预警值调整**：水电预警值根据季节和实际情况可能动态调整，请关注系统公告
- **业主房标签**：业主自留房剔除预警功能需等待"房态需求"上线后生效
- **离线计算**：设备离线期间如发生实际用量，恢复上线后按平均值计算，可能存在偏差
