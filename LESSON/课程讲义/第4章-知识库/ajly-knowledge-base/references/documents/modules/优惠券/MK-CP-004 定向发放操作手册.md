# MK-CP-004 定向发放操作手册

> **功能编号**: MK-CP-004  
> **功能名称**: 定向发放  
> **所属模块**: 优惠券管理  
> **操作频率**: 中频  
> **最后更新**: 2026-03-23  
> **执行人**: 紫灵

---

## 一、功能概述

- **功能目的**: 向指定用户定向发放优惠券，支持精准营销
- **使用场景**: VIP 客户专属优惠、投诉补偿、活动奖励、定向促销
- **业务价值**: 实现精准用户触达，提升用户满意度和转化率

---

## 二、入口方式

### 2.1 导航路径

菜单路径：租赁管理 → 优惠券 → 手动发券（标签页）

### 2.2 按钮位置（ASCII 结构图）

```
| [搜索] [重置] [发券] ← 定向发放入口                      |
+----------------------------------------------------------+
| 数据列表：                                                |
| 序号 | 用途 | 发券场景 | 发券原因 | 发券明细 |           |
| 状态 | 收券人数 | 发券时间 | 创建人 | 操作 |              |
| 操作：[详情] [明细]                                       |
+----------------------------------------------------------+
| 分页：共 3001 条 | 10 条/页 | 1 2 3 4 5 6 ... 301 前往 [页]  |
+----------------------------------------------------------+
```

### 2.3 前置条件

- [ ] 已创建有效的优惠券模板（MK-CP-001）
- [ ] 优惠券状态为"已发布"
- [ ] 拥有优惠券发放权限
- [ ] 目标用户信息（手机号/用户 ID）

## 四、操作流程

### 4.1 正常流程

```
步骤 1: 导航至租赁管理 → 优惠券
步骤 2: 点击"手动发券"标签页
步骤 3: （可选）使用筛选条件查找特定发券记录
步骤 4: 点击"发券"按钮发起定向发券
步骤 5: 选择优惠券模板
步骤 6: 输入目标用户手机号（支持单个/批量）
步骤 7: 填写发券场景和发券原因
步骤 8: 提交发券申请
步骤 9: 系统向目标用户发放优惠券
步骤 10: 查看发券结果（收券人数更新）
```

### 4.2 流程图

```

│  进入手动   │ →  │  点击发券   │ →  │  选择优惠   │
│  发券页面   │    │  按钮       │    │  券模板     │
└─────────────┘    └─────────────┘    └─────────────┘
                                              ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  查看发券   │ ←  │  系统发放   │ ←  │  输入用户   │
│  结果       │    │  优惠券     │    │  手机号     │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 五、校验规则

### 5.1 必填校验

| 字段 | 错误提示 | 触发时机 |
|------|---------|---------|
| 优惠券模板 | "请选择优惠券" | 提交时 |
| 目标用户手机号 | "请输入用户手机号" | 提交时 |
| 发券场景 | "请输入发券场景" | 提交时 |
| 发券原因 | "请输入发券原因" | 提交时 |

### 5.2 格式校验

| 字段 | 格式要求 | 错误提示 | 示例 |
|------|---------|---------|------|
| 手机号 | 11 位数字 | "手机号格式不正确" | 13800138000 |
| 发券场景 | 文本，≤50 字 | "发券场景不能超过 50 字" | VIP 客户专属优惠 |
| 发券原因 | 文本，≤200 字 | "发券原因不能超过 200 字" | 客户投诉补偿 |

### 5.3 业务校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 优惠券有效性 | 优惠券必须为已发布状态 | "优惠券未发布，无法发放" |
| 库存校验 | 发放数量 ≤ 优惠券剩余库存 | "库存不足，无法发放" |
| 用户有效性 | 手机号对应用户存在 | "用户不存在" |
| 重复发放 | 同一用户不能重复获得同种券 | "该用户已获得此券" |

## 六、异常场景

| 异常编号 | 异常场景 | 系统提示 | 处理方式 |
|---------|---------|---------|---------|
| EX-001 | 优惠券已下架 | "优惠券已下架，无法发放" | 重新发布优惠券或选择其他券 |
| EX-002 | 库存不足 | "库存不足，当前剩余 X 张" | 减少发放数量或补充库存 |
| EX-003 | 手机号格式错误 | "手机号格式不正确" | 检查手机号格式后重试 |
| EX-004 | 用户不存在 | "用户不存在" | 确认手机号是否正确 |
| EX-005 | 网络超时 | "网络繁忙，请稍后再试" | 检查网络连接后重试 |
| EX-006 | 无发放权限 | "您没有发券权限" | 联系管理员授权 |
