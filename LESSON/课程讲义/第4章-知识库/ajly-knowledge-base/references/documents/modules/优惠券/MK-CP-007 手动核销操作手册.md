# MK-CP-007 手动核销操作手册

> **功能编号**: MK-CP-007  
> **功能名称**: 手动核销  
> **所属模块**: 优惠券管理  
> **操作频率**: 低频  
> **最后更新**: 2026-03-23  
> **执行人**: 紫灵

---

## 一、功能概述

- **功能目的**: 管理员手动为用户核销优惠券，处理特殊情况
- **使用场景**: 系统异常补核销、线下活动核销、客服协助核销
- **业务价值**: 提供核销兜底方案，处理自动化流程无法覆盖的场景

---

## 二、入口方式

### 2.1 导航路径

菜单路径：租赁管理 → 优惠券 → 券列表 → 选择券 → 详情 → 手动核销

### 2.2 按钮位置

```
+----------------------------------------------------------+
| 优惠券详情                                               |
+----------------------------------------------------------+
| ...                                                      |
+----------------------------------------------------------+
| 核销管理                                                  |
| [手动核销] [核销记录]                                     |
+----------------------------------------------------------+
```

### 2.3 前置条件

- [ ] 已创建有效的优惠券模板
- [ ] 用户持有该优惠券且状态为"未使用"
- [ ] 拥有优惠券核销权限
- [ ] 核销原因合理

---

## 三、页面结构

### 3.1 页面布局

```
+----------------------------------------------------------+
| 手动核销                                                  |
+----------------------------------------------------------+
| 优惠券信息：[券名称] [券类型] [优惠额度]                  |
+----------------------------------------------------------+
| 用户信息：                                                |
| - 用户手机号 [输入框，必填]                               |
| - 用户姓名 [自动填充，只读]                               |
+----------------------------------------------------------+
| 核销信息：                                                |
| - 核销原因 [下拉选择 + 文本输入，必填]                    |
| - 核销备注 [文本输入，可选]                               |
+----------------------------------------------------------+
| [确认核销] [取消]                                         |
+----------------------------------------------------------+
```

### 3.2 字段说明

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 用户手机号 | 文本输入 | 是 | 11 位手机号 | 持有券的用户手机号 |
| 核销原因 | 下拉选择 | 是 | 系统异常/线下活动/客服协助/其他 | 选择核销原因 |
| 核销备注 | 文本输入 | 否 | ≤200 字 | 补充说明 |

---

## 四、操作流程

### 4.1 正常流程

```
步骤 1: 导航至租赁管理 → 优惠券
步骤 2: 选择目标优惠券，点击"详情"
步骤 3: 点击"手动核销"按钮
步骤 4: 输入用户手机号
步骤 5: 系统查询该用户持有的优惠券
步骤 6: 选择核销原因
步骤 7: （可选）填写核销备注
步骤 8: 点击"确认核销"
步骤 9: 系统验证并执行核销
步骤 10: 核销成功，更新券状态为"已使用"
```

---

## 五、校验规则

### 5.1 必填校验

| 字段 | 错误提示 | 触发时机 |
|------|---------|---------|
| 用户手机号 | "请输入用户手机号" | 提交时 |
| 核销原因 | "请选择核销原因" | 提交时 |

### 5.2 业务校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 用户存在性 | 手机号对应用户存在 | "用户不存在" |
| 持券校验 | 用户持有该优惠券且未使用 | "用户未持有此券" |
| 券状态 | 优惠券状态正常 | "优惠券状态异常" |
| 权限校验 | 操作人有核销权限 | "无核销权限" |

## 六、异常场景

| 异常编号 | 异常场景 | 系统提示 | 处理方式 |
|---------|---------|---------|---------|
| EX-001 | 用户不存在 | "用户不存在" | 确认手机号是否正确 |
| EX-002 | 用户未持券 | "用户未持有此优惠券" | 确认用户和券信息 |
| EX-003 | 券已使用 | "该优惠券已被使用" | 无需重复核销 |
| EX-004 | 无权限 | "您没有核销权限" | 联系管理员授权 |
| EX-005 | 网络超时 | "网络繁忙，请稍后再试" | 检查网络后重试 |
