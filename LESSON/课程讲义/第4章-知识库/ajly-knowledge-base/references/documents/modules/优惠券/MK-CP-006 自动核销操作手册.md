> **操作频率**: 系统自动

---

## 一、功能概述

- **功能目的**: 用户在使用优惠券支付时，系统自动完成核销操作
- **使用场景**: 用户在线支付房租/服务费/押金时使用优惠券抵扣
- **业务价值**: 简化用户操作流程，提升支付体验，确保核销准确性

---

## 二、入口方式

### 2.1 触发方式

自动核销为系统自动流程，无需手动操作入口。

**触发条件**:
- 用户在小程序端选择使用优惠券
- 用户完成支付操作
- 系统验证优惠券有效性后自动核销

### 2.2 查看核销记录

菜单路径：租赁管理 → 优惠券 → 券列表 → 选择券 → 详情 → 核销记录

---

## 三、核销流程

### 3.1 自动核销流程

```
步骤 1: 用户在小程序端选择房源/服务
步骤 2: 进入支付页面，选择可用优惠券
步骤 3: 系统验证优惠券有效性（状态/有效期/适用条件）
步骤 4: 用户确认支付
步骤 5: 系统扣减优惠券状态为"已使用"
步骤 6: 系统记录核销时间、核销金额、核销订单
步骤 7: 更新优惠券核销统计数据
```

### 3.2 流程图

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  用户选择   │ →  │  系统验证   │ →  │  验证通过   │
│  优惠券     │    │  优惠券     │    │             │
└─────────────┘    └─────────────┘    └─────────────┘
                                              ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  更新统计   │ ←  │  记录核销   │ ←  │  完成支付   │
│  数据       │    │  信息       │    │  核销券     │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 四、校验规则

### 4.1 核销前置校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 券状态 | 优惠券状态必须为"已发布/核销中" | "优惠券不可用" |
| 有效期 | 当前时间在有效期内 | "优惠券已过期" |
| 适用条件 | 订单满足券使用条件（业态/金额/房源） | "不满足使用条件" |
| 库存 | 券未被使用过 | "优惠券已使用" |
| 互斥 | 未与其他互斥券同时使用 | "不能与其他券同用" |

## 五、异常场景

| 异常编号 | 异常场景 | 系统提示 | 处理方式 |
|---------|---------|---------|---------|
| EX-001 | 优惠券已过期 | "优惠券已过期" | 选择其他有效券 |
| EX-002 | 不满足使用条件 | "不满足优惠券使用条件" | 检查订单是否符合要求 |
| EX-003 | 优惠券已使用 | "优惠券已被使用" | 选择其他券 |
| EX-004 | 支付失败 | "支付失败，优惠券未核销" | 重新支付或联系客服 |
| EX-005 | 网络超时 | "网络繁忙，请稍后再试" | 检查网络后重试 |
