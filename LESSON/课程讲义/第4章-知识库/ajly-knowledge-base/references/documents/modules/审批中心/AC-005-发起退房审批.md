## 一、功能概述

***请注意：公寓管理和企业合同的退房审批流程是在企业微信中进行，通过乐寓租务系统的操作退房发起，在企业微信中进行审批，请查看ajly-knowledge-base/references/documents/features/原路退款SOP（含退订、退换转租-持续更新）.md***

- **功能目的**: 在商业管理中执行退房操作时，根据系统审批配置自动发起退房审批流程，将退房申请提交给指定审批人审核
- **使用场景**: 当系统启用了退房审批流程时，业务员/管家办理租客退房后，退房申请需经过审批人（店长/经理/管理员）审核通过后方可生效
- **业务价值**: 建立退房业务的风险管控机制，防止违规退房操作，确保退房结算金额准确，保护公司和租客双方权益

## 二、入口方式

### 2.1 导航路径

**发起退房审批不是独立功能，而是退房操作的后续流程：**

```
 → 找到目标房间 → 操作 → 退房 → 填写退房信息 → 提交 → 自动生成审批单
         或
商业管理 → 找到目标房间 → 操作 → 退房 → 填写退房信息 → 提交 → 自动生成审批单
```

### 2.2 触发条件

发起退房审批自动触发的条件：

1. **系统配置**: 审批中心→偏好设置中已配置退房审批流程
2. **审批人设置**: 已设置审批人（店长/经理/管理员）和可审核门店
3. **退房操作完成**: 在商业管理中完成退房信息填写并提交
4. **符合审批规则**: 退房类型/金额等符合配置的审批触发规则

### 2.3 前置条件

- [ ] 系统中已存在待退房的租客/房间
- [ ] 当前用户具有退房操作权限（业务员/管家/店长/经理/管理员）
- [ ] 审批中心已配置退房审批流程（审批人/可审核门店）
- [ ] 当前用户已登录系统

## 四、操作流程

### 4.1 正常流程

**步骤 1: 进入商业管理页面**
- 导航到租赁管理→（或商业管理）
- 找到目标房间所在的楼栋/项目

**步骤 2: 打开退房操作弹窗**
- 点击目标房间的"操作"按钮
- 选择"退房"选项

**步骤 3: 填写退房信息**
- 实际退房时间：选择日期（必填）
- 退房性质：选择"正常退房"或"违约退房"（必填）
- 退房类型：从下拉列表选择（如"工作变动"、"合同到期"等）（必填）
- 退房原因：填写具体原因（必填，500 字以内）
- 退房评价：选择"好评"、"中评"或"差评"（选填）

**步骤 4: 确认退房结算金额**
- 系统自动计算应退还/应扣除金额
- 查看明细（违约金、超期房租费等）
- 确认无误

**步骤 5: 上传附件（可选）**
- 点击"+"上传退房照片
- 最多 9 张，支持 png/jpg/GIF 格式
- 每张不超过 2M

**步骤 6: 填写备注（可选）**
- 填写需要说明的其他信息
- 不超过 500 字

**步骤 7: 提交退房申请**
- 点击"提交"按钮
- 系统校验必填字段
- 如启用审批：显示审批提示"您的退房申请将提交给 [审批人] 审批"
- 提交成功，生成审批单

**步骤 8: 查看审批进度**
- 导航到审批中心→业务审批
- 在列表中找到刚提交的退房审批
- 审批状态显示"待审批"

### 4.2 异常流程

| 异常场景 | 触发条件 | 系统表现 | 处理方式 |
|----------|----------|----------|----------|
| 未配置审批流程 | 审批中心未设置退房审批 | 退房提交后直接生效，无审批单 | 联系管理员配置审批流程 |
| 实际退房时间为空 | 未选择实际退房时间 | 提示"请选择实际退房时间" | 选择日期后重新提交 |
| 退房性质未选择 | 未选择正常/违约退房 | 提示"请选择退房性质" | 选择退房性质后重新提交 |
| 退房类型未选择 | 未选择退房类型 | 提示"请选择退房类型" | 选择退房类型后重新提交 |
| 退房原因未填写 | 退房原因输入框为空 | 提示"请填写退房原因" | 填写原因后重新提交 |
| 退房原因超长 | 超过 500 字 | 提示"退房原因不能超过 500 字" | 精简内容后重新提交 |
| 图片格式错误 | 上传非 png/jpg/GIF 格式 | 提示"图片格式不支持" | 转换格式后重新上传 |
| 图片大小超限 | 单张图片>2M | 提示"图片大小不能超过 2M" | 压缩图片后重新上传 |
| 图片数量超限 | 上传超过 9 张 | 提示"最多上传 9 张图片" | 删除多余图片 |
| 备注超长 | 超过 500 字 | 提示"备注不能超过 500 字" | 精简内容后重新提交 |
| 无退房权限 | 当前用户无退房操作权限 | "操作"菜单无"退房"选项或按钮禁用 | 联系管理员分配权限 |
| 房间状态异常 | 房间非"已租"状态（如已空置/已退租） | 无"退房"选项或提示"房间状态不允许退房" | 确认房间状态正确 |
| 有未结清账单 | 房间存在未结清的应收/应付账单 | 提示"存在未结清账单，请先处理" | 前往财务管理结清账单 |
| 审批人未配置 | 审批流程已启用但未设置审批人 | 提示"审批人未配置，请联系管理员" | 联系管理员配置审批人 |
| 网络异常 | 提交时网络中断 | 提示"网络异常，提交失败" | 检查网络后重新提交 |

```
                     └─[审批驳回]──> 已租（恢复原状态）
```

- **影响模块**: 
  - 商业管理（房间状态变更）
  - 财务管理（退房结算金额）
  - 审批中心（审批单生成）
  
- **影响数据**: 
  - 退房申请记录（提交人/提交时间/退房信息）
  - 审批单记录（审批状态/审批人/审批时间）
  - 房间状态（已租→待退房→空置）
  - 财务账单（应收/应付款项）

- **后续操作**: 
  - AC-002 处理审批（通过）：审批人审核通过退房申请
  - AC-003 处理审批（驳回）：审批人驳回退房申请
  - AC-004 查看已办审批：查看已审批完成的退房记录
  - AM-004/CM-009 退房操作：重新发起退房（如被驳回）

## 八、常见问题

**Q1: 退房提交后没有生成审批单？**  
A1: 可能原因：①审批中心未配置退房审批流程；②当前门店不在审批范围内。请检查审批中心→偏好设置中的审批人配置和可审核门店设置。

**Q2: 如何知道审批人是谁？**  
A2: 提交退房时，弹窗底部会显示审批提示"您的退房申请将提交给 [审批人姓名] 审批"。也可在审批中心→偏好设置中查看配置的审批人。

**Q3: 审批被驳回了怎么办？**  
A3: ①进入审批中心→业务审批，点击"不通过"链接查看驳回原因；②根据驳回原因修改退房信息；③重新发起退房操作。

**Q4: 审批通过后房间状态何时更新？**  
A4: 审批通过后，房间状态自动从"待退房"更新为"空置"，可立即重新出租。

**Q5: 能否撤回已提交的审批？**  
A5: 待审批状态的审批单不支持撤回。如需取消，请联系审批人驳回，或联系管理员处理。

**Q6: 退房结算金额如何计算？**  
A6: 系统根据合同到期时间、实际退房时间、租金标准自动计算：①正常退房且未超期：无费用；②违约退房或超期：收取违约金 + 超期房租费（超期天数×日租金）。
