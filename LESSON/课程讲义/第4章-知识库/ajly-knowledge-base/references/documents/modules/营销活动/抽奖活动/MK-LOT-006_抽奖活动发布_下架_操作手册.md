> **所属模块**: 营销推广 → 营销活动 → 抽奖活动  
> **操作频率**: 中频

> **执行人**: 紫灵

## 一、功能概述

- **功能目的**: 控制抽奖活动的上线和下线状态，管理活动的可见性和可参与性
- **使用场景**: 
  - 活动配置完成后，需要发布让用户参与
  - 活动需要提前下线或暂停
  - 活动配置有误需要撤回修改
- **业务价值**: 确保活动运营的可控性，避免配置错误的活动对用户造成影响，支持灵活的活动上下架管理

## 四、操作流程

### 4.1 发布操作流程

```
步骤 1: 登录系统，进入营销推广 → 营销活动
步骤 2: 点击"抽奖活动"标签页，切换到抽奖活动列表
步骤 3: 找到状态为"未发布"的目标活动
步骤 4: 点击操作列的"发布"按钮
步骤 5: 系统直接执行发布（无需确认）
步骤 6: 活动状态变为"已发布"，操作按钮更新
步骤 7: 活动链接列显示"复制"按钮，可复制链接分享
```

### 4.2 取消发布操作流程

```
步骤 1: 登录系统，进入营销推广 → 营销活动
步骤 2: 点击"抽奖活动"标签页，切换到抽奖活动列表
步骤 3: 找到状态为"已发布"的目标活动
步骤 4: 点击操作列的"取消发布"按钮
步骤 5: 系统弹出确认对话框"确定取消发布吗"
步骤 6: 点击"确定"按钮确认操作
步骤 7: 活动状态变为"已取消"，操作按钮更新
步骤 8: 活动链接列"复制"按钮消失
```

### 4.3 流程图

```

│ 未发布状态   │
│ (配置完成)   │
└──────┬──────┘
       │ 点击"发布"

       ▼
┌─────────────┐
│ 已发布状态   │──────────────┐

       │ 点击"取消发布"       │

       ▼                     │

│ 已取消状态   │──────────────┘
│ (手动下线)   │ 可重新发布
└─────────────┘
```

## 五、校验规则

### 5.1 发布前校验

| 校验项 | 校验规则 | 错误提示 | 触发时机 |
|--------|---------|---------|---------|
| 活动配置完整性 | 活动基本信息、奖池配置必须完成 | 请先完成活动配置 | 点击发布时 |
| 活动时间校验 | 活动结束时间必须晚于开始时间 | 活动时间设置错误 | 创建活动时 |
| 奖池配置校验 | 至少配置一个奖品 | 请配置奖池 | 点击发布时 |
| 关联券校验 | 如关联优惠券，券必须有效 | 关联的优惠券已失效 | 点击发布时 |

### 5.2 取消发布校验

| 校验项 | 校验规则 | 错误提示 | 触发时机 |
|--------|---------|---------|---------|
| 二次确认 | 必须点击确认对话框的"确定"按钮 | - | 点击取消发布时 |
| 权限校验 | 用户必须有营销活动管理权限 | 无权限操作 | 点击取消发布时 |

### 5.3 业务校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 状态流转 | 只有"未发布"状态可发布，只有"已发布"状态可取消发布 | 当前状态不允许此操作 |
| 时间校验 | 已过期的活动不能发布 | 活动已过期，无法发布 |
| 奖池校验 | 奖池为空的活動不能发布 | 请先配置奖池 |

## 七、关联影响

### 7.1 影响模块

- **小程序前端**: 发布后用户可在小程序看到并参与活动
- **优惠券管理**: 如活动关联优惠券，发布后券可被用户抽取
- **数据统计**: 发布后开始统计参与数据
- **短信通知**: 取消发布后，用户访问活动链接会收到活动已结束提示

### 7.2 影响数据

- **活动状态表**: 更新活动状态字段
- **活动可见性**: 控制小程序端活动展示
- **参与记录**: 发布后用户可产生新的参与记录
- **统计数据**: 发布后开始累计参与人数、中奖人数等

### 7.3 后续操作

- **发布后**:
  - 监控活动参与数据（MK-LOT-003）
  - 查看奖品发放情况（MK-LOT-004）
  - 分析活动效果（MK-LOT-005）
  
- **取消发布后**:
  - 已参与用户数据保留
  - 已中奖用户奖品正常发放
  - 可重新编辑配置后再次发布

## 九、注意事项

### 9.1 发布注意事项

1. **发布前检查**: 确保活动配置完整，奖池已配置
2. **时间校验**: 确认活动时间范围正确
3. **关联券检查**: 如关联优惠券，确认券在有效期内
4. **发布时机**: 选择合适的时间发布，避免配置错误影响用户

### 9.2 取消发布注意事项

1. **二次确认**: 取消发布需要二次确认，防止误操作
2. **数据保留**: 取消发布后，已参与用户数据和奖品发放不受影响
3. **用户影响**: 取消发布后，用户访问活动链接会看到活动已结束
4. **重新发布**: 取消发布的活动可重新编辑配置后再次发布

### 9.3 状态管理最佳实践

1. **先测试后发布**: 新活动建议先在小范围测试后再正式发布
2. **及时监控**: 发布后及时监控参与数据和异常情况
3. **灵活调整**: 发现问题及时取消发布并修正
4. **记录操作**: 重要活动的发布/取消操作建议记录操作人和时间
