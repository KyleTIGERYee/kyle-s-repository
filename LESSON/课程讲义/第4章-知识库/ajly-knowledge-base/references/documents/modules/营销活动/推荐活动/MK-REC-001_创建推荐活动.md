> **功能编号**: MK-REC-001  
> **功能名称**: 创建推荐活动  
> **所属模块**: 营销推广 → 营销活动 → 推荐活动  
> **操作频率**: 中频  
> **最后更新**: 2026-03-23  
> **执行人**: 紫灵

---

## 一、功能概述

- **功能目的**: 创建新的房源推荐类营销活动，通过推荐吸引客户看房成交
- **使用场景**: 
  - 运营人员需要发起新的推荐活动
  - 设置推荐人和被推荐人的资格条件
  - 配置推荐奖励规则
  - 定制活动落地页样式
- **业务价值**: 通过社交推荐获取新客源，降低获客成本，提高成交转化

---

## 二、入口方式

### 2.1 导航路径

菜单路径：营销推广 → 营销活动 → 推荐活动 → 新建活动

### 2.2 按钮位置（ASCII 结构图）

```
+----------------------------------------------------------+
| 营销活动 → 推荐活动                                       |
+----------------------------------------------------------+
| 筛选区：                                                  |
| 活动名称 | 活动时间 (起 - 止) | 活动状态 |                   |
| [搜索] [重置] [新建活动] [导出]                           |
|                    ↑                                      |
|              点击此处进入创建页面                          |
+----------------------------------------------------------+
```

### 2.3 前置条件

- [ ] 已登录管理后台，具有营销活动管理权限
- [ ] 已准备好活动基本信息（名称、时间）
- [ ] 已准备好活动图片素材（背景图、卡片图）
- [ ] 已确定推荐规则（推荐人/被推荐人资格、奖励条件）

### 3.2 字段说明

#### 基本信息

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 活动名称 | 文本 | 是 | 不能为空，建议 20 字以内 | 活动的名称，用于列表展示 |
| 活动时间 | 日期范围 | 是 | 开始日期 ≤ 结束日期，不能为空 | 活动的有效时间范围 |

#### 活动规则

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 推荐人资格 | 下拉选择 | 是 | 必须选择 | 定义哪些用户可以作为推荐人 |
| 被推荐人资格 | 下拉选择 | 是 | 必须选择 | 定义哪些用户可以被推荐 |
| 奖励条件 | 数字 | 是 | ≥0 的整数 | 被推荐人入住超过多少天后触发奖励 |

#### 活动奖励

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |

## 四、操作流程

### 4.1 正常流程

```
步骤 1: 登录管理后台，进入 营销推广 → 营销活动 → 推荐活动
步骤 2: 点击 [新建活动] 按钮
步骤 3: 填写基本信息（活动名称、活动时间）
步骤 4: 配置活动规则（推荐人资格、被推荐人资格、奖励条件）
步骤 5: 配置活动奖励（赠券模式、是否收集银行卡信息）
步骤 6: 配置推荐人转发页（规则文本、背景图、卡片图、颜色）
步骤 7: 配置被推荐人信息收集页（规则文本、背景图）
步骤 8: （可选）填写外部 Feedback URL
步骤 9: 点击 [提交] 按钮保存
步骤 10: 系统验证通过后，活动创建成功，返回列表页
```

### 5.2 格式校验

| 字段 | 格式要求 | 错误提示 | 示例 |
|------|---------|---------|------|
| 活动时间 | 开始日期 ≤ 结束日期 | "结束日期不能早于开始日期" | 2026-03-23 至 2026-12-31 |
| 奖励条件 | ≥0 的整数 | "请输入有效的天数" | 0, 1, 7, 30 |
| 外部 Feedback URL | 有效的 HTTP/HTTPS URL | "请输入有效的 URL 地址" | https://example.com/callback |
| 活动背景图 | 图片格式（JPG/PNG），推荐 375x812 | "请上传有效的图片文件" | 375x812 PNG |
| 转发卡片图 | 图片格式（JPG/PNG），推荐 5:4 | "请上传有效的图片文件" | 500x400 JPG |

### 5.3 业务校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 活动时间范围 | 活动结束日期不能早于当前日期 | "活动结束日期不能早于今天" |
| 排除用户导入 | 导入的用户手机号格式必须正确 | "第 X 行手机号格式错误" |
| 图片大小限制 | 上传图片不能超过 5MB | "图片大小不能超过 5MB" |

## 六、异常场景

| 异常编号 | 异常场景 | 系统提示 | 处理方式 |
