# MK-REC-002 配置推荐规则 操作手册

> **功能编号**: MK-REC-002  
> **功能名称**: 配置推荐规则  
> **所属模块**: 营销活动 - 推荐活动  
> **操作频率**: 中频  
> **最后更新**: 2026-03-23  
> **执行人**: 紫灵

---

## 一、功能概述

- **功能目的**: 配置推荐活动的参与资格、奖励条件和奖励规则
- **使用场景**: 创建推荐活动后，设置推荐人和被推荐人的资格要求、奖励触发条件及优惠券发放规则
- **业务价值**: 通过灵活的规则配置，实现精准营销和有效激励，提升房源推荐转化率

### 3.2 字段说明

#### 基本信息

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 活动名称 | 文本 | 是 | 不可编辑，继承自定义 | 显示活动名称 |
| 活动时间 | 日期范围 | 是 | 结束日期≥开始日期 | 活动有效期 |

#### 活动规则

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 推荐人资格 | 下拉选择 | 是 | 必选 | 可选：全部用户、指定用户群体 |
| 被推荐人资格 | 下拉选择 | 是 | 必选 | 可选：非在租且不在商机库、全部用户等 |
| 奖励条件 | 数字 | 是 | ≥0 的整数 | 被推荐人需入住的最少天数 |

#### 活动奖励

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 赠券模式 | 下拉选择 | 是 | 必选 | 可选：赠送推荐人、赠送被推荐人、双方都赠送 |
| 适用租期 | 多选标签 | 是 | 至少选择 1 个 | 可选：1 个月~12 个月及以上 |
| 选择券 | 下拉选择 | 是 | 必选 | 从已有优惠券中选择 |
| 赠送量 | 数字 | 是 | ≥1 的整数 | 发放优惠券数量 |

#### 落地页配置

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 活动规则文本 | 富文本 | 是 | 不能为空 | 推荐人转发页显示的活动规则 |
| 活动背景图 | 图片上传 | 是 | 推荐 375x812 像素 | 推荐人转发页背景 |
| 转发卡片图 | 图片上传 | 是 | 推荐 5:4 比例 | 分享卡片封面图 |
| 按钮颜色 | 颜色选择器 | 是 | 必选 | 页面按钮颜色 |
| 页面背景色 | 颜色选择器 | 是 | 必选 | 页面背景颜色 |
| 关联管家选项 | 复选框 | 否 | - | 是否开启管家关联功能 |
| 被推荐人规则文本 | 富文本 | 是 | 不能为空 | 被推荐人信息收集页规则 |
| 被推荐人背景图 | 图片上传 | 是 | - | 被推荐人页面背景 |
| 关联企业选项 | 复选框 | 否 | - | 是否开启企业关联功能 |
| 外部 Feedback URL | 文本 | 否 | URL 格式 | 外部回调地址 |

## 四、操作流程

### 4.1 正常流程

```
步骤 1: 进入营销活动 → 推荐活动列表
步骤 2: 点击目标活动的"详情"按钮
步骤 3: 配置活动规则：
        - 选择推荐人资格（如：全部用户）
        - 选择被推荐人资格（如：非在租且不在商机库）
        - 设置奖励条件（如：入住超过 0 天）
步骤 4: 配置活动奖励：
        - 选择赠券模式（如：赠送推荐人）
        - 选择适用租期（可多选：1 个月、2 个月等）
        - 选择优惠券（从已有券中选择）
        - 设置赠送量
        - 点击"继续添加"可增加多组规则
步骤 5: 配置落地页：
        - 编辑推荐人转发页规则文本
        - 上传活动背景图和转发卡片图
        - 选取按钮颜色和页面背景色
        - 勾选是否开启关联管家选项
步骤 6: 配置被推荐人信息收集页：
        - 编辑活动规则文本
        - 上传活动背景图
        - 勾选是否开启关联企业选项
        - （可选）填写外部 Feedback URL
步骤 7: 点击"保存"按钮
步骤 8: 系统验证通过后保存成功
```

### 4.2 流程图

```
开始
  ↓
进入推荐活动详情页
  ↓
配置活动规则（推荐人/被推荐人资格、奖励条件）
  ↓
配置活动奖励（赠券模式、租期、优惠券、数量）
  ↓
配置推荐人转发页（规则文本、图片、颜色）
  ↓
配置被推荐人信息收集页
  ↓
点击保存
  ↓
系统验证 → 验证失败 → 显示错误提示 → 修改后重新保存
  ↓
验证成功
  ↓
保存成功，返回列表
  ↓
结束
```

|------|---------|---------|------|
| 奖励条件 | ≥0 的整数 | 请输入有效的天数 | 0、7、30 |
| 赠送量 | ≥1 的整数 | 请输入有效的赠送数量 | 1、5、10 |
| 活动背景图 | 图片格式 (JPG/PNG)，推荐 375x812 | 请上传符合要求的图片 | 375x812.png |
| 转发卡片图 | 图片格式 (JPG/PNG)，推荐 5:4 比例 | 请上传符合要求的图片 | 500x400.jpg |
| 外部 Feedback URL | 有效的 HTTP/HTTPS URL | 请输入有效的 URL 地址 | https://example.com/callback |

### 5.3 业务校验

| 校验项 | 校验规则 | 错误提示 |
|--------|---------|---------|
| 活动时间 | 结束日期必须大于等于开始日期 | 结束日期不能早于开始日期 |
| 优惠券选择 | 必须选择已存在的优惠券 | 请选择有效的优惠券 |

| EX-007 | 活动规则文本为空 | 请输入活动规则文本 | 填写规则文本后重新提交 |
