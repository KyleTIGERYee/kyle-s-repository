## 一、功能概述

新建结算单用于汇总渠道在指定周期内的成交记录，生成佣金结算单并提交审批，是渠道佣金支付的前置环节。

**前置条件**:
- 已登录安居乐寓管理系统
- 拥有渠道管理模块操作权限
- 系统中已存在待结算的成交记录（CM-CH-004）

## 三、操作步骤

### 3.1 进入佣金结算页面

1. 点击左侧菜单【租赁管理】展开
2. 点击【渠道管理】
3. 点击【佣金结算】标签页
4. 查看结算单列表

### 3.2 新建结算单

1. 点击页面右上角「新建结算单」按钮
2. 弹出新建结算单表单对话框

### 3.3 选择渠道和周期

1. 点击「选择渠道」下拉框
2. 选择目标渠道
3. 选择结算周期（月份）
4. 系统自动带出结算类型

### 3.4 查看成交明细

1. 系统自动列出该渠道在结算周期内的所有待结算成交记录
2. 核对成交明细是否正确
3. 确认佣金总额

### 3.5 填写结算信息

| 字段 | 类型 | 必填 | 说明 | 校验规则 |
|------|------|------|------|----------|
| 佣金总额 | 数字输入 | ✅ | 汇总佣金金额 | 自动计算，可调整 |
| 扣减金额 | 数字输入 | ❌ | 扣款金额 | 0-佣金总额之间 |
| 实付金额 | 数字输入 | ✅ | 实际支付金额 | = 佣金总额 - 扣减金额 |
| 收款账户 | 文本输入 | ❌ | 渠道收款账户 | 自动带出 |
| 备注说明 | 多行文本 | ❌ | 补充说明 | 最多 500 字 |

### 3.6 上传附件（可选）

1. 点击「选择文件」按钮
2. 选择结算相关文件（如发票扫描件）
3. 支持 JPG、PNG、PDF 格式
4. 文件大小不超过 10MB

### 3.7 保存并提交

1. 检查所有必填字段已填写
2. 点击「保存并提交」按钮
3. 系统校验数据合法性
4. 校验通过则保存成功，提示"结算单创建成功，已提交审批"
5. 自动关闭对话框，返回列表页
6. 新结算单显示在列表第一行，状态为"待审批"

### 4.1 必填项校验

| 字段 | 错误提示 |
|------|----------|
| 选择渠道 | "请选择渠道" |
| 结算周期 | "请选择结算周期" |
| 佣金总额 | "请输入佣金总额" |
| 实付金额 | "请输入实付金额" |

### 4.2 格式校验

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 佣金总额 | 正数 | "佣金总额必须为正数" |
| 扣减金额 | 0-佣金总额之间 | "扣减金额不能大于佣金总额" |
| 实付金额 | 正数 | "实付金额必须为正数" |
| 结算周期 | 有效月份 | "请选择有效的结算周期" |

### 4.3 业务校验

| 规则 | 错误提示 |
|------|----------|
| 实付金额 = 佣金总额 - 扣减金额 | "实付金额计算错误" |
| 渠道在结算周期内无待结算成交 | "该渠道在结算周期内无待结算成交" |
| 结算单已存在 | "该渠道该周期的结算单已存在" |

## 六、结算单状态

| 状态 | 说明 | 后续操作 |
|------|------|----------|
| 待审批 | 已提交，等待审批 | 审批通过后进入待付款 |
| 已驳回 | 审批未通过 | 修改后重新提交 |
| 待付款 | 审批通过，等待付款 | 执行佣金结算 |
| 已付款 | 已完成付款 | 结算完成 |

## 七、常见问题

### Q1: 结算单可以修改吗？
**A**: 待审批状态的结算单可以修改或撤销。已审批通过的结算单不能修改，需联系审批人驳回后修改。

### Q2: 结算单可以删除吗？
**A**: 待审批状态的结算单可以删除。已提交审批的结算单不能删除，需先撤销审批。

### Q3: 一个渠道一个月可以创建多个结算单吗？
**A**: 不可以。一个渠道一个结算周期只能创建一个结算单，避免重复结算。

### Q4: 扣减金额如何填写？
**A**: 如渠道有违规行为需要扣款，在扣减金额中填写扣款数额。实付金额会自动计算。

### Q5: 结算单审批流程是什么？
**A**: 结算单提交后 → 渠道负责人审批 → 财务负责人审批 → 审批通过后进入待付款状态。

## 八、注意事项

1. **周期唯一**: 一个渠道一个结算周期只能创建一个结算单
2. **明细准确**: 仔细核对成交明细，确保无遗漏无重复
3. **金额正确**: 实付金额必须等于佣金总额减扣减金额
4. **附件完整**: 建议上传发票扫描件，便于财务审核
5. **及时提交**: 结算周期结束后及时创建结算单，避免延误付款
