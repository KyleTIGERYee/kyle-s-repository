## 一、功能概述

- **功能目的**: 查询和管理客户自助预约看房记录，支持按战区、门店、物业地址筛选，支持数据导出
- **使用场景**: 
  - 门店管理员查看本店看房预约记录
  - 战区管理员查看辖区内所有看房预约
  - 导出看房数据用于统计分析
- **业务价值**: 帮助运营人员高效管理客户看房预约，提升看房转化率，优化房源带看安排

### 3.2 字段说明

| 字段名称 | 字段类型 | 必填 | 校验规则 | 说明 |
|---------|---------|------|---------|------|
| 战区 | 下拉选择 | 否 | 无 | 支持按战区筛选，选项包含第一战区、第二战区、虚拟战区等 |
| 门店 | 下拉选择 | 否 | 无 | 支持按门店筛选，根据所选战区动态加载 |
| 物业地址 | 下拉选择 | 否 | 无 | 支持按物业地址筛选，根据所选门店动态加载 |
| 房间号 | 文本 | 只读 | 无 | 显示预约看房的房间号，支持多房间（如"101 102"） |
| 看房人姓名 | 文本 | 只读 | 无 | 预约看房的客户姓名 |
| 看房人手机号 | 文本 | 只读 | 脱敏显示 | 手机号中间 4 位隐藏（如 131****5824） |
| 看房时间段 | 日期时间 | 只读 | 无 | 预约的看房时间段，格式：YYYY-MM-DD HH:mm-HH:mm |
| 看房状态 | 文本 | 只读 | 无 | 待看房/已看房/已过期/已失效 |
| 看房评价 | 文本 | 只读 | 无 | 客户看房后的评价，无评价显示"-" |

### 3.3 按钮说明

| 按钮名称 | 位置 | 功能说明 | 权限要求 |
|---------|------|---------|---------|
| 偏好配置 | 页面顶部右侧 | 配置自助看房偏好设置 | 管理员权限 |
| 楼栋设置 | 页面顶部右侧 | 设置支持自助看房的楼栋 | 管理员权限 |
| 导出 | 筛选区右侧 | 导出当前筛选条件下的看房记录为 Excel | 导出权限 |
| 搜索 | 筛选区右侧 | 根据筛选条件查询数据 | 查看权限 |
| 重置 | 筛选区右侧 | 清空所有筛选条件 | 查看权限 |

### 3.4 看房状态说明

| 状态 | 说明 | 显示条件 |
|------|------|---------|
| 待看房 | 预约时间未到或已到但未确认看房 | 预约时间 > 当前时间 或 预约时间已到但未操作 |
| 已看房 | 客户已完成看房 | 管理员确认客户已看房 |
| 已过期 | 预约时间已过 | 预约结束时间 < 当前时间 |
| 已失效 | 预约被取消或房源不可用 | 房源下架/预约取消 |

## 四、操作流程

### 4.1 正常流程

```
步骤 1: 登录管理后台，点击"营销推广"菜单
步骤 2: 在展开的子菜单中点击"自助看房"
步骤 3: 系统加载自助看房列表页面，默认显示所有记录
步骤 4: （可选）在筛选区选择战区/门店/物业地址
步骤 5: 点击"搜索"按钮执行筛选
步骤 6: 查看数据列表中的看房记录
步骤 7: （可选）点击"导出"按钮导出当前数据
步骤 8: 在导出确认对话框中点击"确定"完成导出
```

## 六、异常场景

| 异常编号 | 异常场景 | 系统提示 | 处理方式 |
|---------|---------|---------|---------|
| EX-001 | 无筛选条件下数据量过大 | 无提示，正常加载分页 | 建议添加筛选条件缩小范围 |
| EX-002 | 筛选条件无匹配数据 | 列表显示为空 | 调整筛选条件或点击"重置" |
| EX-003 | 导出时网络超时 | 可能无提示或显示"网络繁忙" | 检查网络连接后重试 |
| EX-004 | 无导出权限 | 导出按钮可能不显示或点击无响应 | 联系管理员开通权限 |
| EX-005 | 数据加载失败 | 可能显示"数据加载失败" | 刷新页面或检查网络 |
| EX-006 | 导出文件过大 | 浏览器可能阻止下载 | 添加筛选条件减少数据量 |
