## 一、文档结构模板

### 1.1 标准文档结构

```markdown
# 文档标题

文档标题（重复，纯文本）

# 概述/功能概述

简述功能目的、适用范围和主要特性（2-3段）。

# 主要功能模块/操作指引

## 功能一/（一）操作前提

说明使用本功能的前提条件。

## 功能二/（二）核心操作步骤

1. 步骤一说明
   - 子步骤说明
   - 注意事项

2. 步骤二说明

## 功能三/（三）注意事项

- 注意点1
- 注意点2

# 权限说明（如适用）

| 角色 | 权限范围 |
| --- | --- |
| 角色A | 权限说明 |

# 配置说明/偏好设置（如适用）

说明配置项和参数设置。

# 功能细节说明（如适用）

详细的功能逻辑和规则说明。

# 数据初始化/历史数据处理（如适用）

存量数据的处理方式。

# 相关系统对接（如适用）

| 字段名 | 解释 |
| --- | --- |
| 字段A | 说明 |
```

### 1.2 章节顺序规范

| 顺序 | 章节名称 | 是否必须 |
|------|----------|----------|
| 1 | 概述/功能概述 | ✅ 必须 |
| 2 | 操作指引/功能模块 | ✅ 必须 |
| 3 | 权限说明 | 可选 |
| 4 | 配置说明 | 可选 |
| 5 | 功能细节说明 | 可选 |
| 6 | 数据初始化 | 可选 |
| 7 | 注意事项 | 可选 |
| 8 | 相关系统对接 | 可选 |

## 二、标题格式规范

### 2.1 一级标题

- **格式**：`# 文档标题`
- **内容**：与文件名一致或高度相似
- **重复**：文档开头重复一次标题内容（纯文本，无符号）

**示例**：
```markdown
# 家具家电新业务相关需求的功能说明

家具家电新业务相关需求的功能说明
```

### 2.2 二级标题

- **格式**：`## 章节名称` 或 `# 章节名称`
- **用途**：主要章节划分
- **命名**：简短、概括性强，名词短语

**示例**：
```markdown
# 供应商管理

## 供应商列表页
```

### 2.3 三级标题

- **格式**：`### 子章节名称` 或 `### （一）操作前提`
- **用途**：具体操作步骤、子功能说明
- **特殊格式**：可使用带括号的中文序号

**示例**：
```markdown
### （一）操作前提
### （二）核心操作步骤
### （三）注意事项
```

## 四、列表和表格规范

### 4.1 无序列表

- **符号**：使用 `-` 或 `·`（中间点）
- **场景**：功能点列举、注意事项
- **缩进**：子项使用缩进

**示例**：
```markdown
- 功能点1
- 功能点2
  - 子功能点
  - 子功能点
```

### 4.2 有序列表

- **符号**：使用 `1.` `2.` `3.` 或 `①` `②` `③`
- **场景**：操作步骤、流程说明

**示例**：
```markdown
1. 步骤一
2. 步骤二
3. 步骤三

① 发起修改
② 信息填写
③ 提交审核
```

### 4.3 表格规范

**标准格式**：
```markdown
| 字段名 | 解释 |
| --- | --- |
| 字段A | 说明 |
| 字段B | 说明 |
```

**使用场景**：
1. 字段映射说明
2. 业务规则对比
3. 权限矩阵
4. 配置参数说明

## 五、格式标记规范

### 5.1 粗体使用

| 使用场景 | 格式 | 示例 |
|----------|------|------|
| 功能按钮名称 | `**按钮名称**` | **开发票**、**提交审核** |
| 关键术语首次出现 | `**术语**` | **购方审核权限** |
| 重要提醒 | `**注意内容**` | **请注意：** |
| 配置项名称 | `**配置项**` | **入住日期最早可选N天后** |

### 5.2 特殊符号

| 符号 | 使用场景 | 示例 |
|------|----------|------|
| 【】 | 系统菜单路径 | 【字典设置】-【租客字典】 |
| "" | 界面显示文字 | "商户流水无法开具发票" |
| `` | 字段名、代码 | `userPhone`、`-` |
| · | 列表符号 | · 功能点 |

## 六、数字和单位规范

### 6.1 数字表示

| 类型 | 格式 | 示例 |
|------|------|------|
| 整数 | 阿拉伯数字 | 5天、30天、100元 |
| 百分比 | 数字+% | 50%、100% |
| 金额 | 数字+元 | 300元、100元 |
| 日期 | YYYY-MM-DD | 2024.1.1-2026.12.31 |
| 时间 | YYYY-MM-DD hh:mm:ss | 精确到秒 |
| 范围 | N-M | 1-30、0-100 |

### 6.2 单位使用

| 单位 | 使用场景 |
|------|----------|
| 天/日 | 时间周期 |
| 月 | 租期单位 |
| 元 | 货币单位 |
| % | 比例、折扣 |
| 间 | 房间数量 |
| 人 | 人数 |
| 张 | 优惠券数量 |

### 6.3 变量表示

使用大写字母表示可配置变量：
- N、M、X、Y 等
- "假设填入此项设置的值为N"
- "M>=N"

## 七、语言风格规范

### 7.1 语态使用

| 语态 | 使用场景 | 示例 |
|------|----------|------|
| 主动语态 | 描述系统行为 | 系统会自动增加... |
| 被动语态 | 描述用户被限制的操作 | 该流水被触发... |
| 祈使句 | 操作步骤说明 | 点击"开发票"按钮... |

### 7.2 句式特征

1. **条件句**："当...时"、"如果...则"、"若..."
2. **并列说明**：使用分号";"连接
3. **举例说明**："例如..."、"如..."
4. **强调句式**："请注意..."、"特别说明..."

### 7.3 语言风格

- **正式书面语**：整体风格正式、专业
- **技术化表达**：使用专业术语
- **口语化提示**：注意事项中可适当口语化

## 八、流程描述规范

### 8.1 操作流程

**标准结构**：
```markdown
1. 步骤名称
   具体操作内容...
   
2. 步骤名称
   具体操作内容...
```

### 8.2 条件流程

**使用格式**：
```markdown
场景1：...
场景2：...
场景3：...

或

如果...则...
否则...
```

### 8.3 状态流转

**使用格式**：
```markdown
【状态名称】：状态说明
   - 子状态1：说明
   - 子状态2：说明
```

**示例**：
```markdown
【入住确认状态】分为：

"未确认"：此时用户还不可在小程序中点击确认...
"已确认"：此时用户已在小程序中点击了确认...
"已取消"：此时用户拒绝了确认入住...
```

## 九、文档类型特定规范

### 9.1 功能说明文档

- 必须包含"功能概述"
- 详细描述功能逻辑
- 包含操作步骤
- 说明权限要求

### 9.2 SOP操作文档

- 必须包含"操作前提"
- 详细的分步骤说明
- 包含异常处理
- 明确的注意事项

### 9.3 配置说明文档

- 列出所有配置项
- 说明配置默认值
- 描述配置影响范围
- 提供配置示例
