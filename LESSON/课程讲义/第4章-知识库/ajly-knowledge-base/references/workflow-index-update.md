# 知识库入库/洗库运维指引 (Librarian 工作流)

本文档专供大模型在被分配“清洗历史文档”或“新写了文章需要录入大图谱”的任务时，用来警醒自己的准则。

**⚠️ 请时刻牢记你的双层身份：你既是业务精算师，又是系统安全维护员。**

---

## 一、双轨制物理存放规则

当你新写或迁移了 Markdown (`.md`) 文件后，须遵守以下二选一的双轨落盘规则物理存放文件：

### 轨道 A：业务大事件与功能说明区 (`features/`)
**适用场景**：一整个产品迭代和业务功能说明。
**存放路径**：`references/documents/features/你的文件.md`

### 轨道 B：原子级按钮操作手册 (`modules/`)
**适用场景**：细粒度极高的系统“单一界面”操作图谱。
**存放路径**：`references/documents/modules/[对应的系统大类名称]/你的文件.md`

---

## 二、大脑提炼阶段 (提取业务属性)

1. **绝对通读**
   调用工具深入阅读该文件的所有字符。不要泛泛而过。
2. **生成高精摘要 (`summary_preview`)**
   将通读后的精华压缩为 50-100 字以内的话术，以便雷达搜索时提供最佳语境。
3. **萃取关键名录 (`keywords`)**
   提炼 3-5 个能够作为检索爆破点的强特征名词。
4. **反思行业黑话 (`synonyms`)**
   思考这些词汇在一线应用中可能存在的口语化叫法（如“置换补偿”->“换房补贴”）。

---

## 三、利用网关落盘阶段 (绝对禁止肉搏改 JSON)

当你在脑海中组装好上面的数据后，**严禁使用文本编辑工具直接修改 `system_map.json`！** 那会导致 JSON 层级错乱崩溃。

你必须通过以下两个 Python 小网关脚本来通知底层写盘模块！

### 挂载文件进图谱雷达中枢
使用沙盒终端工具执行（注意：必须使用技能根目录的绝对路径，即 SKILL.md 中定义的 `SKILL_ROOT`）：
```bash
python3 <SKILL_ROOT>/scripts/update_node.py --file "references/documents/.../你的文件.md" --summary "你萃取的精华摘要" --keys "词1,词2,词3"
```

### 将行业黑话连通系统雷达
如果你找到了同义词映射：
```bash
python3 <SKILL_ROOT>/scripts/update_synonyms.py --term "主关键词(如:退房)" --synonyms "俗称黑话1,俗称黑话2"
```
