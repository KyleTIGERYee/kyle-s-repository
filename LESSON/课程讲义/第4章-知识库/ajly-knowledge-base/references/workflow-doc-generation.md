# 文档生成工作流程

本文档定义基于输入内容生成功能说明文档的标准流程。

---

## 支持输入类型

| 输入类型 | 处理方式 | 输出格式 |
|----------|----------|----------|
| 会议纪要 | 提取功能需求、业务流程 | 功能说明文档 |
| HTML原型 | 分析页面结构、交互逻辑 | 功能说明文档 |
| PRD文档 | 提炼功能点、业务规则 | 功能说明文档 |
| 文字描述 | 结构化整理 | 功能说明文档 |
| 需求清单 | 归纳整理 | 功能说明文档 |

---

## 文档生成五步法

### Step 1: 分析输入内容

- 理解用户提供的原始内容
- 提取关键信息：功能需求、业务流程、用户角色、系统交互
- 识别功能模块和子功能

### Step 2: 参考规范对照

- 阅读 [writing-standards.md](writing-standards.md) 了解文档书写规范
- 根据功能类型，参考对应的参考文档
- 确定文档结构和术语使用

### Step 3: 生成标准文档

- 按照标准文档结构生成内容
- 使用规范的术语（PC端/小程序/租客/管家等）
- 遵循格式标记规范（粗体、菜单路径等）
- 包含必要的章节（概述、操作指引等）

### Step 4: 系统挂载点落盘（必做）

**最终落盘动作**：
1. 请将你生成的纯原味 Markdown 物理文件存放进目标专区中：
   - 若属于某次业务升级/新功能发布：存在 `references/documents/features/`
   - 若纯粹某个页面上的按钮/列表页帮助手册：存在 `references/documents/modules/[分类]/`
2. **务必调用你的沙盒终端环境运行入库脚本**（注意：使用技能根目录的绝对路径，即 SKILL.md 中定义的 `SKILL_ROOT`）：
   运行 `python3 <SKILL_ROOT>/scripts/update_node.py --file "你的文件相对路径" --summary "精华摘要" --keys "关键词1,关键词2"`
   只有脚本跑完，安居乐寓的中枢雷达 `system_map.json` 才会接纳这个新节点，大模型检索才能看见它。
   如果总结出了同义词/黑话，也请运行 `python3 <SKILL_ROOT>/scripts/update_synonyms.py --term "主词" --synonyms "黑话1,黑话2"`。

### Step 5: 质量自省机制

- 强制读取 `references/quality-checklist.md` 进行双重验证。
- 确保术语使用一致，菜单、按钮、字段等 Markdown 黑体块表现完好。
- 验证流程是否形成了闭环。

---

## 强制参考规范

在生成新文档时，**必须**：

✅ 阅读书写规范：[writing-standards.md](writing-standards.md)  
✅ 参考同类文档：根据功能类型参考对应的参考文档  
✅ 使用标准术语：PC端、小程序、租客、管家等  
✅ 遵循格式规范：按钮用粗体、菜单路径用【】、字段名用代码块  
✅ 包含必要章节：必须包含"概述"和"操作指引"  

---

## 禁止行为

❌ 严重禁止：不使用标准术语（如用"电脑端"代替"PC端"）  
❌ 严重禁止：不遵循格式标记规范（如按钮不用粗体）  
❌ 严重禁止：遗漏必要章节（如缺少"概述"）  
❌ 严重禁止：随意创造术语而不参考现有文档  

---

## 旧有对标实境参考

当你处于迷惑期，不知道要写出什么样的文风时：
强烈建议你先通过运行 `python3 <SKILL_ROOT>/scripts/search_map.py "相近的业务关键词"` 来找到同类文档路径。
然后找到 `features/` 或 `modules/` 节点下类似业务的一个物理文件，深读它。以此来对齐你的文字排列规格与语气。

## 标准文档结构

```markdown
# 文档标题

文档标题（重复，纯文本）

# 概述

简述功能目的、适用范围和主要特性。

# 功能模块

## 功能一

### （一）操作前提

- 前提条件1
- 前提条件2

### （二）操作步骤

1. 点击【菜单】-【子菜单】进入页面
2. 点击**按钮**，填写表单
3. 点击**提交**，完成操作

### （三）注意事项

- 注意点1
- 注意点2

# 权限说明（如适用）

| 角色 | 权限范围 |
| --- | --- |
| 管家 | 可查看、可编辑 |
| 店长 | 可查看、可编辑、可删除 |

# 配置说明（如适用）

- **参数A**：说明，默认值为N
- **参数B**：说明，默认值为M

# 相关系统对接（如适用）

| 字段名 | 解释 |
| --- | --- |
| fieldA | 说明 |
| fieldB | 说明 |
```
