# 转租功能

## 乐寓系统对转租功能进行了更新，在APP和PC端均可进行操作

点击“转租”后，会先进入原租客合同的结算页面，该页面计算逻辑与退房结算一致，区别为：1.“转租时间”默认为操作的当天，不可修改；2.会生成一条300元的转租手续费。

点击下一步后即可登记新租客，其流程与普通的登记租客一致。

增加了转租原因的选择，这些原因可以在【字典设置】-【租客字典】-【租客转租原因】中自定义配置

# 续租功能相关调整

在定价中增加“续约涨幅”的功能，可按门店配置不同的续约涨幅。

# 新签功能相关调整

新登记的合同，若在当天（自然日）未缴纳押金和首期租金，则在0点将该合同做退房处理，退房类型为“没有签成”。

系统自动关闭合同后，自动发送待办给到该合同的录入人，待办内容为：“xxx房源的xxx租客的合同因为超时未支付房租，已自动关闭该合同”

# 换房功能相关调整

换房新合同的结束时间不能早于上一份合同结束时间

增加了换房原因的选择，这些原因可以在【字典设置】-【租客字典】-【租客换房原因】中自定义配置

# 退房功能相关调整

增加了退房原因的选择，这些原因可以在【字典设置】-【租客字典】-【退房原因】中自定义配置

完成退房后，租客可以在小程序-我的-我的租约中，点击“退房账单”。若需要退款给该租客，该租客可以查看退款的明细；若该租客需要额外缴纳账单，则可以在此处进行支付。

历史未付的，可以在流水账中红冲删除以前的流水，客户可再次付款

# 价格计算方式的更新

字段概念：

成交价：系统上叫做出房价格

定价：又称挂牌价格

房间优惠：即房间的特价，又称特价折扣等。

身份优惠：系统上叫做企业框架合同，实际上是深梦启航、深梦扬帆这种身份优惠

续租系数：系统上叫续租涨幅，新概念

价格的计算统一为：成交价=定价*房间优惠*租期优惠*企业优惠*续租系数

# 数据初始化

由于计算方式的调整，需要对存量数据进行处理，处理历史数据的处理方式为：

个人（包括框架）

1）定价：当时签约定价

2）特价：当时签约特价

3）企业租金折扣：当前企业设置的租金折扣，没有就默认为1

4）企业押金折扣：当前企业设置的押金折扣，没有就默认为1

5）叠乘涨幅：1

6）租期优惠：出房价格/（定价*特价*企业优惠）

特别说明

由于租期优惠是通过计算得出，并且存在续约合同调整企业框架的情况，所以会存在一定的初始化数据偏差，现对这种情况做出说明：

特价优惠=定价/特价，但是这个算出来的值因为四舍五入导致落库数据不准确，又因为租期优惠=出房价格/（定价*特价*企业优惠），这样算出来的租期优惠就会有1.01等大于1的数据；

续租的合同，新老合同可能将框架企业进行了调整，导致算出来的租期折扣不对，有一部分租期折扣的数据大于了1，初始化时将这部分错误数据都处理为1了。

有几条非续租的数据，计算出来的租期优惠是大于1的，未做处理。

如果某房间在最近一次的特价被取消了，而且签约前没有设置最新特价（也就是说，签约前，特价的记录为“-”），那么房间特价取的是最近一次有数据的特价。本来特价应该是1，但实际取到了0.9。这种情况非常少见，如果遇见请找开发（黄伟）修改数据。

5.新的特价数据，如果是通过价格批量导入的，特价优惠会是计算出来的=特价/定价，这里也会有四舍五入
