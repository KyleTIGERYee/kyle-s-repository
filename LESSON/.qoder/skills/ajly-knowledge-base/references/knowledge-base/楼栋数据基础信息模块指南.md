# 楼栋数据基础信息模块指南

楼栋数据基础信息模块指南

本指南主要阐述如何使用乐寓租务管理系统管理业主合同、生成账单、调整账单等操作。

# 录入业主合同

登录乐寓租务管理系统，点击【公寓管理】-【楼栋信息】-操作-【项目管理】-【修改项目】

将右上角的“自持物业”改为“租赁他人物业”

其中“收储面积”填在“建筑面积”

在本弹窗中填写业主的相关信息

产权信息上传

递增可设置百分比递增或者固定金额递增

如需每3年涨5%，则每隔3年填写5%即可，其他年份均填0

完成以上信息的填写后，点击保存即可。

# 多业主合同的录入

对于同一栋楼有多个业主的情况，例如，某楼栋甲有3名业主，在系统中楼栋名字为【甲】，则在楼栋甲的基础上录入第一名业主的信息，收款名字、银行账号等关键信息都使用业主一的资料。

对于业主二和业主三，点击【登记楼栋】-【租赁他人物业】，按照楼栋甲的合同信息再录入一份同样的数据，区别点在于，需要填入【座栋】字段，其中业主相关的资料按业主二的相关资料填写，合同周期、租金、押金等，都录入业主二合同中的保底租金部分。填写完成之后点击保存结束即可。请注意，不要为新增加的楼栋分配房间。

# 业主账单的调整

选择要处理的楼栋，点击【查看】-【项目账单】

## 调整首期账单和尾期账单

因为系统是按照整月计算房租（例如4.15~5.14），而非自然月（例如4.1~4.30）

所以首期账单和尾期账单的金额是错误的，故需要调整首期账单和尾期账单金额。

点击【操作】-【修改本条】，将应付房租调整为正确的房租即可。

## 调整涨幅节点账单

如果存在设置了涨幅的楼栋，需要在涨幅的节点月份，修改账单。

例如图中合同开始日期为12号，2026年12月的账单，已经是涨幅5%后的价格，按照正常的计算逻辑应该是1号到11号按照涨价前的价格计算，12号到31号按涨价后的价格计算，计算后重新调整账单。（每次涨幅节点，都应该做此调整）

## 标记付款

对于已经支付的账单，点击【未付】，即可进行标记付款的操作

## 降租

如果在某个时间点完成了降租谈判，需要对后续租金降租，则在需要开始降租的那期账单，点击【操作】-【房租降价】，选择【以后全部都降】，并且填写降租的金额（=降租前租金-降租后租金）

（这里的意思是降价3000元）

如果过去已经付款了，系统也已经操作了付款，现在或者未来已经谈好了降租，需要调整过去的账单的话，点击【财务记录】。找到你需要调整的账单流水，点击操作-【冲红删除】，该账单将自动变为未收。

随后对房租进行降租操作，再在点击“未付”操作标记付款前，点击【其他付款】，然后填写字段【预付收益】，即可在预付房租的基础上增加这一部分的资金。然后点击保存即可。请不要点击批量保存。

预付收益的部分，需要在现在或者未来的账单进行抵扣，则在需要抵扣的账单中，点击【其他扣款】，然后填写字段【预付收益扣回】，即可在预付房租的基础上扣除掉这一部分的资金。然后点击保存即可。

## 押金抵扣

如果存在押金抵扣租金的情况 ，需要在点击“未付”操作标记付款前，点击【其他扣款】，然后填写字段【押金抵扣】，即可在预付房租的基础上扣除掉这一部分的资金。然后点击保存即可。请不要点击批量保存，否则会在所有账单都添加上这笔抵扣的。

完成以上的账单调整操作后，均可进行标记付款的操作。
