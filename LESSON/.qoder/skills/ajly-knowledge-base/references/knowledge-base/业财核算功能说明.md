# 业财核算功能说明

业财核算功能说明

# 前言

本功能强依赖于业务系统中【流水账】功能模块，所有财务科目相关的收入和支出数据的产生，都需要通过流水账进行生成。

基本原理：

设置【费项】在不同场景下对应的【种类/原因】（此为一次性配置工作）

与业务系统有关的账单（如租客账单收入、业主账单支出）产生的流水账或手动添加的流水账，会关联到对应的【种类/原因】；

同时，创建自定义的【财务科目】（此为一次性配置工作）

将【财务科目】与流水账中的【种类/原因】建立关联关系（此为一次性配置工作）

按【种类/原因】设置分摊规则（此为一次性配置工作）

当产生流水账时，会生成对应的【种类/原因】，根据此【种类/原因】，会查询关联的【财务科目】，在该【财务科目】下产生对应的收/支数据，并且根据分摊规则进行记账。

简单来说：在完成全部配置后，与业务系统有关的账单（如租客账单收入、业主账单支出），只需要租客正常支付账单，或者对业主账单进行“收款”，自动产生对应的流水账即可完成记账；与业务系统无关的账单，则需要手动在【流水账】进行操作记录，生成对应的流水账即可完成记账。

# 配置方法

## 核心计算逻辑

## 财务字典配置

入口【字典设置】-【财务字典】-【业财核算】

财务字典按照收入和支出分别进行配置，财务一级科目的录入在【添加收入科目】或【添加支出科目】

二级科目在录入时，需先选择好一级科目，随后在右侧点击【添加】

## 种类/原因的配置

入口【流水账】-【流水账】-【偏好设置】-【种类原因设置】，对于系统自动产生的费项，建议结合最近几个月的流水账记录，对所有会产生收入支出的费项进行种类/原因的设置。

对于非系统自动产生的费项，需要 手动记录流水账的，建议在【其他收支项目】中进行记账，【其他收支项目】可以添加自定义项目，具体方法：

在【字典设置】-【财务字典】-【其他收支项目】中，可以手动创建自定义项目，创建的其他收支项目，是费项，在手动记账时可以进行选择。最后，再用同样的方法，设置其他收支项目对应的【种类原因】

# 业财核算-偏好设置

## 关联科目

在设置好上述内容后，可以在【业财核算】-【偏好设置】中进行最终设置。

在【关联科目】中，是根据【财务科目】关联需要的流水原因，可以按照收入科目和支出科目分别进行设置。如，收入科目-价外收入-赔偿款，点击“+”，设置其对应的流水种类原因为房租或押金类收入-租客赔偿款。那么后续产生了该种类原因的流水，会将其收入数据记录到价外收入-赔偿款的收入科目中。

## 设置分摊规则

在上一步中，产生了房租或押金类收入-租客赔偿款种类原因的流水，会将其收入数据记录到价外收入-赔偿款的收入科目中，但是该流水是否需要分摊，如何分摊，仍需要配置。

点击【设置分摊规则】，在这里，是按照种类原因设置是否需要分摊和分摊周期的，需要按照收入原因和支出原因进行分别设置。点击【添加种类原因】，选择房租或押金类收入-租客赔偿款，【是否分摊】根据需要选择为“否”。

如果是其他的需要按照权责口径统计的种类原因，可根据需要选择为“是”，分摊周期可以选择“账单有效期”或者“自定义”。

“账单有效期”：如果是业务系统关联的账单（如租客账单），则自动按照账单的周期进行分摊；如果是业务系统无关的账单，即手动录入的流水，则按照时手动填入的账单周期进行分摊。

“自定义”：会根据自定义的时长进行分摊，如选择分摊周期为1年，则会从操作日开始往后计算一年，作为分摊周期。

# 数据来源

## 自动生成

业务系统相关的账单，如果产生的对应的流水账，则会自动进行入账的操作。（前提是需要在配置阶段完成所有相关配置）

特别说明：如果需要记录向业主支付的房租数据，请在房东账单处，对房东账单进行手动收款，并且生成流水账。

## 手动录入

所有非业务系统相关的账单，均需要手动录入流水账。

入口【流水账】-【流水账】-【记账】

按照记账需求填入楼栋的物业地址，并且在下拉选项中选择具体的房间号。入账门店会自动带出，按照实际情况填写账单周期。

按照记账需求，选择收支类型为【收】或【支】，点击“明细”，选择具体的账单费项明细，种类/原因会自动带出，填写应收付金额后点击旁边空白处即可。

输入应收付金额后点击旁边空白处后，下方会弹出【业财核算】模块，可以手动操作是否计入核算，是否分摊，以及选择分摊周期。系统会读取后台的相关配置，此处无需进行修改，直接点击保存即可。

也可以按照楼栋或者门店进行记账，按楼栋进行记账时，输入物业地址后选择第一个不带房间号的物业地址即可。

按门店进行记账时，需要将【业务类型】字段改为“其他”，即可不用输入物业地址或房间号，账会记在门店下。

# 业财核算报表

## 主要账目表

除财务科目为【押金应收款】、【押金应付款】、【订金预收账款】、【订金预付账款】外的所有科目，产生账目均存入到此表中。

这里的入账数据，会根据分摊规则，分摊到每个月中。例如11月1号向某租客趸付了2个月房租共计6100元，则在查看11月数据时，该房间的收入3000元，查看12月数据时，该房间收入为3100元。

如果不做分摊，则会统一记录在账单的付款日（或者手动调整后的日期）。

## 订金收支表和押金收支表

只记录【押金应收款】、【押金应付款】、【订金预收账款】、【订金预付账款】等四个科目对应的账。即押金的收/支和订金的收/支。

业财核算总表

可以按照一级科目进行统计，收入类型的科目-支出类型的科目，可计算出利润总额。收入和支出类型的科目，可以手动添加和删除科目，进行不同维度的统计。

根据查询条件，可以对不同月份，不同门店、楼栋进行利润的统计。如果是提前支付、趸付的账单，并且进行了分摊费用的，可以查询未来的月份，并且查看未来的利润数据。
