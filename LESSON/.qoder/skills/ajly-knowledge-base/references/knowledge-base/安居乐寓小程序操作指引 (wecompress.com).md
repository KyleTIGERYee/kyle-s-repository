
# 安居乐寓小程序系统操作与培训手册

**适用对象：** 运营专员、门店管家、系统管理员


**版本依据：** 2024.07.17 操作指引 

---

## 第一部分：全平台功能结构详解

本部分还原了系统的小程序端、电脑后台端及管家端的完整功能架构，帮助新用户建立全局认知。

1. C端小程序功能架构 

**1.1 首页板块**

* 
**基础服务**：获取定位、获取手机号授权、展示当前定位信息 


* 
**营销展示**：轮播图展示（点击进详情）、首页弹屏 


* **核心功能入口**：
* 
**搜索**：支持按地铁站、门店搜索 


* 
**找房**：地图找房（按区域查询）、帮我找房（填写需求单） 


* 
**业务**：商铺租赁、晨阳计划（查看）、企业合作（活动查看）、今日特惠 




* **房源列表展示逻辑**：
* 
**筛选**：按通勤、距离、租金、房型筛选 


* **排序逻辑**：
* 
*已授权定位*：按用户定位由近到远排序 


* 
*未授权定位*：优先展示“热门门店”房源；若无热门门店，则按后台“店面排序”展示 




* 
**特惠房源**：按价格由低到高展示 





**1.2 “我的”板块（分角色展示）** 

* 
**场景A：未注册/未签约用户** 


* 
**记录**：浏览记录（近7天）、我的收藏（房型/门店/商铺） 


* 
**权益**：我的卡券（待使用/已使用/已失效） 


* 
**帮助中心**：在线客服、客户热线（400电话）、关注公众号（二维码）、关于乐寓、用户协议、隐私政策 


* 
**广告位**：平台活动功能展示 




* 
**场景B：已签约租客** 


* 
**基础信息**：租客姓名、手机号、注销功能 


* **租务管理**：
* 我的租约（点击切换查看其他租约） 


* 账单支付：支付房租、交其他费、其他账单 


* 在线服务：我的合同、我的维修、我的投诉（提交及记录）、支付记录、申请退房、我的评价 




* **智能管理**：
* 房间门锁（开门页面）、公区门禁 


* 智能电表/水表（查看读数） 




* 
**其他**：我的卡券、帮助中心同上 





2. 管家端（移动端）功能架构 

* 
**我的**：个人信息、上传企业微信二维码（客户扫码添加） 


* 
**租赁管理** ：


* 优惠券：查看优惠券列表、发放数量/已发放数量 


* 配额发券：操作发券 


* 发放记录：按名称/手机号查询 





### 3. 电脑端（后台）功能架构

* 
**运营配置**：设置热门门店、店面设置（视频/图片/简介）、系统设置

## 第二部分：运营侧后台配置指南

本章节主要面向运营人员，讲解如何在后台配置小程序的前端展示内容。

1. 营销活动配置 

* **首页弹屏/轮播图**：
* 在后台【小程序管理】中配置。
* 
**首页弹屏**：用户打开小程序时弹出，可点击查看详情或关闭。若未配置则不显示 。


* 
**轮播图**：首页顶部滑动图片，需配置跳转链接 。




* **广告位配置**：
* 包括“晨阳计划”、“企业合作”以及“我的”页面下方的广告位 。


* 
**注意**：部分跳转页面是由代码写死的（如晨阳计划详情页）。如需更改这部分内容，必须将新链接提供给研发人员进行代码替换，并通过小程序发版更新 。





### 2. 门店与房源管理

2.1 门店设置（重难点） 

* 
**基础信息**：需在后台编辑门店的所在城市、区域、店名（如：安居乐寓福永二店）、经纬度等 。


* 
**位置定位（关键操作）**：在设置地址时，**必须选中下拉框中自动联想出的具体地址**。直接手动输入的文本无法作为定位依据，会导致距离计算失效 。


* 
**店长绑定**：必须准确选择该门店的所属店长，这关系到业务归属 。


* 
**热门门店规则（推荐逻辑）** ：


* **作用**：设置为“热门门店”后，将在小程序首页优先展示和推荐。
* **排序规则**：
1. 
**手动置顶优先**：后台手动勾选“置顶”的门店最优先展示，最多展示3个 。


2. 
**系统自动补位**：若未手动置顶，或置顶不足3个，系统会根据**门店收藏量**排名，自动抓取前三名展示 。


3. 
**例外情况**：若手动置顶门店已满3个，则不再按收藏量抓取其他门店 。







2.2 特惠房/商铺设置 

* 
**操作路径**：在房源/商铺列表页，点击“编辑定价”或“录入特价” 。


* **设置内容**：
* 开启“特惠特价”开关。
* 设置折扣力度（如八折、九折）或具体特价金额。
* 设置**特价生效时间段**（开始日期至结束日期） 。




* 
**前端展示**：设置后，小程序端会显示特价标识，并同时展示原价与特价 。

## 第三部分：房源发布与图片管理（管家/店长必读）

1. 公寓发房流程 

* 
**录入方式**：支持电脑端单个录入和批量导入（下载Excel模板） 。


* **必填信息**：物业地址、楼栋、房间号、定价等。

### 2. 图片上传规范（易错点）

此操作在**乐寓管理APP**中进行，非电脑端 。

* 
**操作步骤**：进入APP -> 找到对应房源 -> 点击添加照片/相机按钮 。


* 
**图片分类**：需分别上传封面图片、卧室、客厅、卫生间、阳台图片 。


* **重要逻辑（特别注意）**：
* 
**封面图片**：仅用于小程序首页的列表展示，**不会自动展示在房间详情页内** 。


* 
**详情页图片**：如果希望用户点进房间详情也能看到那张好看的封面图，必须在“卧室”或“客厅”等对应模块下，**再次上传同一张图片** 。





3. 商铺发房 

* 除了基础信息（面积、价格）外，支持添加“入场须知”和“视频” 。


* 
**入场须知**：若未添加内容，小程序端将自动隐藏此版块 。

## 第四部分：用户端逻辑与管家分配机制

### 1. 租客登录与授权

* 
**授权**：首次进入引导获取地理位置（用于推荐最近房源）和手机号。用户可以拒绝授权，不影响进入小程序浏览 。


* **默认页面**：
* 
**无签约房源账号**：默认进入“房源”首页 。


* 
**有签约房源账号**：默认进入“我的”页面，方便交租和开门 。





2. 管家分配逻辑（私海保护） 

当租客在小程序上浏览房源或发起咨询时，系统按以下规则分配管家：

1. 
**分配前提**：管家必须在后台配置了手机号，且在APP端上传了**企业微信二维码** 。


2. 
**随机分配**：若租客首次浏览，系统将在该门店配置了信息的管家中随机分配 。


3. **锁定保护（30天规则）**：
* 一旦租客联系过某位管家（拨打电话或扫码加微），系统会锁定该关系 。


* 
**在接下来的30天内**，无论该租客浏览哪个房源，他在小程序里看到的联系电话和二维码，**都将显示为之前联系过的那位管家**，避免撞单 。





3. 多租约处理 

* 若一个租客账号下有2份及以上有效租约，小程序会**默认展示上一次选中**的那个租约。
* 租客需点击“我的租约”手动切换，才能查看另一套房的账单或进行开门操作 。
