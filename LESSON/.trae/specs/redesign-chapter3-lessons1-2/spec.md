**不适用场景**: 知识库检索（需要使用知识检索节点，无法使用自定义工具）

* 配置API端点:

  * URL: `https://uat-weapp.szajly.com/api/wechat/ai/searchHouse`

  * 方法: POST

  * 鉴权: API Key（Header: applet-token, wxMiniAppId）

**步骤2: 定义参数和返回值（10分钟）**

* 配置输入参数（storeNames, area, layoutName, minPrice, maxPrice等）

**步骤3: 将自定义工具集成到Agent（5分钟）**

* 在Agent节点中添加"自定义工具"

* 配置参数映射

* 测试端到端流程

### 两种模式对比

| 对比项   | 工作流模式           | 自定义工具模式         |
| ----- | --------------- | --------------- |
| 适用场景  | 复杂逻辑、需要数据处理     | 简单API调用         |
| 灵活性   | 高（可使用代码节点）      | 中（参数映射）         |
| 知识库支持 | ✅ 支持            | ❌ 不支持           |
| 开发难度  | 中               | 低               |
| 复用性   | 高（可在多个Agent中使用） | 高（可在多个Agent中使用） |

**五、工具类型与应用场景（15分钟）- How-to**

**常见工具类型速查表**:

| 工具类型  | 工作流模式      | 自定义工具模式 | 应用场景      |
| ----- | ---------- | ------- | --------- |
| 搜索工具  | ✅ 知识库检索节点  | ❌ 不支持   | 政策查询、FAQ  |
| API工具 | ✅ HTTP请求节点 | ✅ 支持    | 房源查询、管家联系 |
| 计算工具  | ✅ 代码执行节点   | ❌ 不支持   | 数据处理、格式转换 |
| 数据库工具 | ✅ 数据库节点    | ❌ 不支持   | 用户信息查询    |

**模式选择建议**:

* 需要知识库检索 → 必须使用工作流模式

* 需要数据处理/代码逻辑 → 使用工作流模式

* 简单API调用 → 可使用自定义工具模式（更快捷）

**工具选择策略**:

* 必要性原则

* 原子性原则

* 明确性原则

* 安全性原则

### 常见问题

**Q1：导入YAML文件失败怎么办？**

* 检查YAML文件格式是否正确

* 确认Dify版本支持该配置

* 尝试手动创建工作流

**Q2：Agent节点不调用工具怎么办？**

* 检查系统提示词中是否明确定义了工具使用场景

* 确认工具参数配置正确

* 查看Agent节点的工具选择策略

**Q3：自定义工具模式和工作流模式怎么选？**

* 需要知识库检索 → 必须使用工作流模式

* 需要数据处理/代码逻辑 → 使用工作流模式

* 简单API调用 → 使用自定义工具模式（更快捷）

**Q4：工具调用后没有返回结果？**

* 检查API是否正常工作

* 查看HTTP请求节点的返回状态

* 检查代码执行节点的错误日志
