## 问题分析

背景色仍然不正确，排查发现可能的原因：

1. **Tailwind CSS 覆盖** - Tailwind 的默认样式可能覆盖了自定义 CSS
2. **Canvas 背景透明** - 粒子画布如果没有填充，会显示浏览器默认白色背景
3. **CSS 加载顺序** - styles.css 在 Tailwind 之后加载，但某些样式仍可能被覆盖

## 修复方案

### 1. 在 HTML 中添加内联样式确保背景色
在 `<head>` 中添加 `<style>` 标签直接设置背景色，确保最先加载

### 2. 修复 Canvas 背景
给 canvas 添加背景色或确保它正确覆盖整个视口

### 3. 增强 CSS 选择器优先级
使用更具体的选择器和 `!important` 确保背景色生效

## 具体修改

**文件：index.html**
- 在 `<head>` 中添加内联样式：
  ```html
  <style>
    html, body {
      background-color: #0F172A !important;
    }
    #slideContainer {
      background-color: #0F172A;
    }
  </style>
  ```

**文件：css/styles.css**
- 给 `#slideContainer` 添加背景色
- 给 `main` 标签添加背景色
- 确保 canvas 有正确的 z-index 和背景
