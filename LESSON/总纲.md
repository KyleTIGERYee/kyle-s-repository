### 第二课：提示词工程
* **提示词方法论深化**
    * 结构化提示词框架：**BROKE** (Background, Role, Objective, Key Results, Evolve) 或 **CO-STAR**
    * **Coze特有技巧**：
        * 利用 Coze 的 `Optimize` 按钮逆向学习优秀 Prompt
        * 变量引用：在 Prompt 中使用 `{{variable}}` 动态引用用户输入
* **进阶技巧**
    * **Few-Shot Prompting (少样本提示)**：给模型“喂”几个安居乐寓的标准回答范例
    * **分隔符的使用**：使用 `###` 或 `"""` 隔离指令与数据，防止指令注入
    * **Markdown 格式控制**：强制模型输出表格、加粗重点，适配前端展示
* **实战 (Coze)**
    * **案例**：编写“安居乐寓金牌销售”的 System Prompt
    * **练习**：通过自然语言描述需求，使用 Coze 的自动优化功能生成第一版 Prompt，并进行人工微调

* **Coze中的记忆机制**

    * **变量 (Variables)**：全局变量与会话变量的区别
* **实战 (Coze)**

## 第二章：工作流 (Workflow)

### 第一课：工作流I：基础工作流

* **工作流概念**
    * 什么是工作流：将大模型调用封装成可复用的“流水线”
    * 为什么需要工作流：解决 Prompt 无法处理复杂逻辑（如多步计算、条件判断）的问题
* **节点功能详解**
    * **输入 (Input)**：接收用户消息
    * **大模型 (LLM)**：核心推理节点
    * **变量 (Variable)**：存储中间结果
    * **条件判断 (Condition)**：实现分支逻辑（如判断价格是否超预算）
    * **输出 (Output)**：返回结果
* **演示**：搭建一个基础工作流
    * 场景：简单的房源信息查询
    * 流程：接收用户需求 -> 调用 LLM 格式化查询 -> 返回结果
* **实战**
    * 学员动手搭建基础工作流
    * 目标：能够独立创建并运行一个包含 3-5 个节点的工作流

### 第二课：工作流II：进阶工作流与RAG基础


* **RAG（检索增强生成）基础**
* 概念：为什么大模型需要外挂知识库
* 向量数据化：文本如何变成计算机理解的数字


* **知识库搭建**
* 数据清洗：如何处理安居乐寓的非结构化数据
* 简单搭建知识库：Q&A类型


* 知识库如何嵌入工作流
* 意图识别
* **实战**
* 搭建一个具备意图识别、能调用不同知识库的工作流



### 第三课：工作流III：复杂工作流
* **逻辑控制节点**
    * **选择器 (Condition)**：If-Else 逻辑。例如：预算>5000 推荐豪宅，<2000 推荐合租
    * **循环 (Loop)**：批量处理数据（例如：一次性处理 10 条房源数据的清洗）
* **数据处理与连接**
    * **Code 节点**：使用 Python/JavaScript 处理复杂数据（如计算房租押一付三的具体金额）
    * **HTTP 请求节点**：在 Coze 中配置 API（模拟查询安居乐寓内部库存系统）
* **复杂场景设计**
    * 错误处理：如果 HTTP 请求失败，如何走“兜底分支”？
* **演示**
    * **AI找房 2.0 (Pro版)**：用户输入需求 -> 判断意图 -> (分支A: 查库存API  -> 输出) / (分支B: 闲聊)。详情请查看C:\Users\37927\Desktop\VIBECODING\LESSON_TRAE\AI项目规划.md
* **实战**
    * 搭建“智能比价助手”：输入两个房源 ID，工作流并发查询详细信息，利用 LLM 节点生成对比表格

## 第四章：搭建自己的项目

### 第一课：产品思维

    * **MVP (Minimum Viable Product)**：不要想造火箭，先造滑板。用 Coze 10分钟搭出的 Bot 就是最好的 MVP

* **工具箱：从想法到原型**
    * **核心工具：Coze** -> 验证业务逻辑、Prompt 效果和数据流通畅度
