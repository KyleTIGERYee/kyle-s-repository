# 抽奖活动功能说明

抽奖活动功能说明

# 功能概述

抽奖活动功能主要是通过配置活动规则、发放券、配置小程序页面等能力，实现用户在新签或者续约后在小程序上可进行抽奖活动，可抽到优惠券、实体奖品两种类型的奖励并且可以设置不中奖概率。若抽到的是优惠券，则自动发放对应优惠券。

# 新建活动

基本信息

活动名称将会展示在小程序页面，活动时间影响活动自动结束

活动规则

关于用户资格方面，请参考下表

关于用户资格的说明：

可参与的用户=参与人资格中的用户-排除参与人中的用户

也就是说，只要用户在【排除参与人】的用户池里，那么不管他符不符合参与人资格，那他都不能参与活动。

关联业务的说明：

关联业务有三种：续约、新签、不关联。

当选择为不关联时，只要符合资格的用户，进入小程序页面便直接获取抽奖次数

当选择为续约时，只要符合资格的用户，完成续约动作即可获取抽奖次数（不考虑是否缴纳续约首期租金）

当选择为新签时，只要符合资格的用户，完成新签动作即可获取抽奖次数（需缴纳首期租金）

总结：新签和续约时，看用户新签和续约前是什么身份，看这个身份是否符合用户资格，如符合则新签和续约后会发放抽奖次数，否则不发放。

例如：参与人资格为在租同住人，关联业务为新签，则新用户在新签时不会获得抽奖次数，在住承租人新签也不会获得抽奖次数，只有在租同住人新签才会获得抽奖次数

抽奖次数

即为符合条件时一次发放的抽奖次数，可填入1-30，发放抽奖次数时，如果关联的业务是新签和续约，则发放到新签和续约合同的承租人手机号上。

实体奖中奖短信通知手机号

如果有用户抽中实体奖品，则给这个手机号发送短信提醒。

关联奖品

概述：新签和续约时，根据新签或者续约的合同期限，来决定不同的奖池，即下面的“适用租期”；每个租期的奖池中，最多可以设置6个奖品，每个奖品都有对应的概率，根据概率进行抽奖。

如果关联业务选择为不关联，则无需设置不同租期，仅需设置奖品。

可设立多条规则，最多设立12条适用租期，点击下方按钮“添加适用租期”即可添加租期。

每个租期，最多可以设置6个奖品，最少需要设置2个奖品，点击右侧的加号和减号即可添加和删除。

【奖品类型】可选不发奖品、自定义实体奖品、优惠券等三种类型。

当选择为“不发奖品”时，仅需填写概率即可；

当选择为“自定义实体奖品”时，需填入奖品名称、库存和概率，奖品名称最多填入7个汉字，库存最多填入99，当自定义奖品被抽完时，小程序转盘中显示为“已抽完”；

当选择为“优惠券”时，需选择优惠券、赠券量和概率，优惠券的逻辑和助力活动、推荐活动一致：读取“活动专用”券，每个活动专用券只能被同一个活动选择，同一活动的不同租期可以选择同一个券。

每个租期中奖池奖品的概率之和必须等于100%。

关于概率的详细说明：

本活动的中奖概率为古典概率模型，每次抽奖之间的概率独立计算，互不干扰，在“运气”好的情况下，甚至可以多次抽中低概率奖品。

当实体奖品的库存被抽空，则该实体奖品的中奖概率被平分到该规则下的其他奖品中

如：适用租期为1个月/2个月/3个月/时，【不发奖品】概率为10%，【券1】为30%，【券2】为30%，【券3】为20%，【海景房一套】为10%，当“海景房一套”的库存被抽空时，对应的10%概率，平分到其他四个奖品中，则平摊后的概率为【不发奖品】概率为12.5%，【券1】为32.5%，【券2】为32.5%，【券3】为22.5%

【活动页配置】

只可以配置活动背景图和活动规则。按钮的样式是设计过的，所以无法变更颜色，可以自定义底图

# 小程序

活动发布后，同其他营销活动一样，是复制该活动的链接，然后添加到banner或者小程序弹窗中，用户可以点击banner或小程序弹窗进入到活动中。

不同的租期进入到小程序页面中，都会看到对应的奖池，转盘最少2格，最多6格；

当奖品类型为优惠券时，转盘中展示为具体的x元租房红包，x为优惠券的总额（比如50元的优惠券发2张，则展示为100元租房红包，图标固定为“券”的样式；

当奖品类型为实体奖品时，转盘中展示为实体奖品的名称，如“烤箱一台”，图标固定为“礼物”的样式；

当奖品类型为不发奖品时，转盘中固定为感谢参与的样式。

点击抽奖，转盘会加速旋转，最终根据概率进行抽奖，指针落在抽到的奖品上。

含底图的样式（全房通设计了一版底图）

不含底图的样式，黑底部分会变成上传的背景图。

# 尘锋数据流转

在创建活动时，在最底部的【外部Feedback URL】填入尘锋-线索对接中的Feedback UR，从而实现关键数据自动传到尘锋。
