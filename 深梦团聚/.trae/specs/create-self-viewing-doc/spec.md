# 自助看房系统功能说明文档规格

## Why

基于自助看房系统讨论会议纪要（以会议纪要内容为准），生成符合安居乐寓标准的**自助看房系统功能说明文档**。该文档将用于：
1. 记录自助看房系统的完整功能设计
2. 作为开发和测试的参考依据
3. 作为业务培训的知识库文档

**重要说明**：本文档以 `自助看房系统讨论.txt` 会议纪要内容为准，其他原型文件仅作参考。如有描述冲突，以会议纪要为准。

## What Changes

- 创建一份完整的《自助看房系统功能说明文档》
- 文档需遵循安居乐寓书写规范（writing-standards.md）
- 文档需包含PC端和小程序端的功能说明
- 使用标准术语：PC端、小程序、租客、管家、店长

## Impact

- **Affected specs**: 安居乐寓知识库文档体系
- **Affected code**: references/knowledge-base/ 目录
- **Output location**: references/knowledge-base/自助看房系统.md

## ADDED Requirements

### Requirement: 文档结构规范

文档必须遵循以下标准结构：
```

### （二）操作步骤
### （三）注意事项

```

### Requirement: 术语规范

必须使用以下标准术语：
- **系统端**：PC端、小程序、APP端、乐寓系统
- **角色**：租客、承租人、同住人、管家、店长
- **业务**：新签、续约、换房、转租、退房、费项、流水账
- **格式**：按钮用**粗体**、菜单路径用【】、字段名用`代码块`、提示用"双引号"

### Requirement: PC端功能覆盖（以会议纪要为准）

文档必须包含PC端以下功能模块：

#### 1. 看房单管理
- **查询功能**：战区、门店、物业地址筛选，支持下拉框搜索
- **列表展示**：房间号、看房人姓名、手机号（脱敏/查看）、看房时间段、看房状态、看房评价
- **导出功能**：看房单数据导出
- **评价展示规则**：只有"已看房"状态才显示评价，待看房状态的评价不展示

#### 2. 偏好设置
- 配置同一楼栋多少天内不可重复预约自助看房
- 天数配置范围：0-365天（0表示不限制）
- 判断逻辑：多少天以内有开过房间（开门禁或开房门都算）
- 拦截时机：在生成申请看房单时触发校验

#### 3. 楼栋设置
- 从楼栋数据读取可设置自助看房的楼栋
- **剔除规则**：没有分配房间的楼栋直接不展示
- 控制楼栋在小程序自助看房列表中的展示
- 支持批量勾选设置
- 门店取消所有自助看房楼栋设置后，该门店在自助看房列表中不展示（但在普通门店列表正常展示）
- 门店下所有房间都被租了，该门店在自助看房列表中不展示

### Requirement: 小程序端功能覆盖（以会议纪要为准）

文档必须包含小程序端以下功能模块：

#### 1. 自助看房入口
- 首页图标入口
- 我的看房入口

#### 2. 门店列表
- 展示已设置自助看房的门店（但凡有自助看房楼栋的门店都展示）
- 定位功能：门店定位取门店位置，房间定位取楼栋位置
- 仅设置了自助看房的房间显示"加入心愿单"按钮

#### 3. 心愿单
- 加入心愿单功能（仅自助看房房源显示按钮）
- 心愿单列表展示：同一栋房间展示在一起，不同门店分开展示，不同楼栋在同一门店内分开展示
- **已租判断**：已登记租客 或 已交押金 均视为已租，已租房间标记且不可点击，只能删除
- 重复添加校验：同一个房间已在心愿单中则无法重复添加
- 删除功能
- 勾选预约功能

#### 4. 申请看房
- 申请人信息：姓名、身份证（二要素校验）
- **二要素校验**：提交时校验，身份证合法性可校验，但姓名与身份证号不匹配的情况可能校验不出来
- 预约时间选择：当天起6天内（包括当天），2小时为一个时间段
- 看房单生成逻辑：按楼栋生成，一次申请多个楼栋则生成多条看房单
- **数量限制**：同时存在的待看房最多10条（已申请数量 + 本次申请数量），已看房的不计入限制
- **重复预约拦截**：根据PC端偏好设置的天数，判断同一楼栋是否有开门记录（开门禁或开房门都算），如有则提示"最近已经有已看房的"

#### 5. 我的看房
- **有效列表**：待看房、已看房
- **无效列表**：已过期、已失效
- **状态流转**：
  - 待看房 → 已看房：开门后1小时自动变更（以最早开门时间往后推1小时）
  - 待看房 → 已过期：当天晚上9点定时任务（原计划8点，后改为9点）
  - 待看房 → 已失效：门店取消自助看房设置（仅对待看房生效）

#### 6. 看房执行
- **人脸识别验证**：每一栋楼都要单独进行人脸识别，同一看房单内不同楼栋也需分别验证
- **人脸识别失败场景**：非真实人脸、不匹配的人脸、照片人脸均会失败
- **开启楼栋门禁**：5分钟内可无限次开启，超过5分钟需重新人脸识别
- **开启房门**：5分钟内可无限次开启，超过5分钟需重新人脸识别
- **时间计算规则**：门锁和门禁的5分钟限制分开计算，互不影响
- **看房评价**：开完房门后再次进入页面展示评价（满意/不满意），评价后推送管家卡片
- **管家联系**：支持添加微信、拨打电话

#### 7. 定时任务
- **早上8点**：发送当天看房提醒短信和推送（仅8点之前申请的会提醒）
- **推送规则**：多栋楼只展示一栋，点击后跳到看房单列表
- **晚上9点**：将未看房的待看房单置为已过期（原计划8点，后改为9点）

### Requirement: 业务规则覆盖（以会议纪要为准）

#### 1. 看房单状态流转
- 待看房 → 已看房：开门后1小时自动变更（以最早开门时间计算，包括开房门或开门禁）
- 待看房 → 已过期：当天晚上9点定时任务
- 待看房 → 已失效：门店取消自助看房设置

#### 2. 重复预约限制
- 偏好设置天数内，同一楼栋有开门记录则不可重复申请
- 开门记录包括：开房门、开门禁
- 天数计算包括今天（如2天包括今天和昨天零点以后）
- 拦截时机：生成申请看房单时

#### 3. 门禁/房门开启规则
- 5分钟内可无限次开启
- 超过5分钟需重新人脸识别
- 门锁和门禁的5分钟限制分开计算
- 门禁是跟楼栋绑定的，不是跟看房单绑定的

#### 4. 看房单数量限制
- 同时存在的待看房最多10条
- 计算方式：已申请数量 + 本次申请数量
- 已看房的不计入限制
- 超过10条则不给申请

#### 5. 数据展示规则
- 仅"已看房"状态显示评价，待看房状态的评价不展示
- 楼栋展示过滤：仅展示设置了自助看房的楼栋
- 已租判断：已登记租客 或 已交押金 均视为已租
