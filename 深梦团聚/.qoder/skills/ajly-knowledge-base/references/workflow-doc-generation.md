# 文档生成工作流程

本文档定义基于输入内容生成功能说明文档的标准流程。

---

## 支持输入类型

| 输入类型 | 处理方式 | 输出格式 |
|----------|----------|----------|
| 会议纪要 | 提取功能需求、业务流程 | 功能说明文档 |
| HTML原型 | 分析页面结构、交互逻辑 | 功能说明文档 |
| PRD文档 | 提炼功能点、业务规则 | 功能说明文档 |
| 文字描述 | 结构化整理 | 功能说明文档 |
| 需求清单 | 归纳整理 | 功能说明文档 |

---

## 文档生成五步法

### Step 1: 分析输入内容

- 理解用户提供的原始内容
- 提取关键信息：功能需求、业务流程、用户角色、系统交互
- 识别功能模块和子功能

### Step 2: 参考规范对照

- 阅读 [writing-standards.md](writing-standards.md) 了解文档书写规范
- 根据功能类型，参考对应的参考文档
- 确定文档结构和术语使用

### Step 3: 生成标准文档

- 按照标准文档结构生成内容
- 使用规范的术语（PC端/小程序/租客/管家等）
- 遵循格式标记规范（粗体、菜单路径等）
- 包含必要的章节（概述、操作指引等）

### Step 4: 生成知识索引（必须）

**⚠️ 重要：必须同时更新 knowledge-base.json**

从生成的文档中提取关键信息，创建 JSON 知识索引条目：

```json
{
  "keys": ["关键词1", "关键词2", "关键词3"],
  "content": "【核心功能：XXX】\n\n**1. 主要功能**：\n功能描述...\n\n**2. 关键规则**：\n- 规则1\n- 规则2\n\n**3. 注意事项**：\n- 注意点1\n- 注意点2",
  "strategy": "keyword",
  "enabled": true,
  "file_path": "knowledge-base/文档名称.md"
}
```

**字段说明**：
- `keys`: 关键词列表，用于检索（功能名称、业务模块、操作类型、角色类型）
- `content`: 知识库提炼精华（核心功能、关键规则3-5条、注意事项2-3条）
- `file_path`: 对应的 Markdown 文档路径，必须以 `knowledge-base/` 开头

**更新步骤**：
1. 打开 `knowledge-base/knowledge-base.json`
2. 在数组末尾添加新的知识索引条目
3. 确保 JSON 格式正确（逗号、引号、括号匹配）

### Step 5: 质量检查

- 对照检查清单验证 Markdown 文档
- 验证 JSON 知识索引格式正确
- 确保术语使用一致
- 检查格式标记正确性
- 验证流程描述清晰

---

## 强制参考规范

在生成新文档时，**必须**：

✅ 阅读书写规范：[writing-standards.md](writing-standards.md)  
✅ 参考同类文档：根据功能类型参考对应的参考文档  
✅ 使用标准术语：PC端、小程序、租客、管家等  
✅ 遵循格式规范：按钮用粗体、菜单路径用【】、字段名用代码块  
✅ 包含必要章节：必须包含"概述"和"操作指引"  

---

## 禁止行为

❌ 严重禁止：不使用标准术语（如用"电脑端"代替"PC端"）  
❌ 严重禁止：不遵循格式标记规范（如按钮不用粗体）  
❌ 严重禁止：遗漏必要章节（如缺少"概述"）  
❌ 严重禁止：随意创造术语而不参考现有文档

## 标准文档结构

```markdown
# 文档标题

文档标题（重复，纯文本）

# 概述

简述功能目的、适用范围和主要特性。

# 功能模块

## 功能一

### （一）操作前提

- 前提条件1
- 前提条件2

### （二）操作步骤

1. 点击【菜单】-【子菜单】进入页面
2. 点击**按钮**，填写表单
3. 点击**提交**，完成操作

### （三）注意事项

- 注意点1
- 注意点2

# 权限说明（如适用）

| 角色 | 权限范围 |
| --- | --- |
| 管家 | 可查看、可编辑 |
| 店长 | 可查看、可编辑、可删除 |

# 配置说明（如适用）

- **参数A**：说明，默认值为N
- **参数B**：说明，默认值为M

# 相关系统对接（如适用）

| 字段名 | 解释 |
| --- | --- |
| fieldA | 说明 |
| fieldB | 说明 |
```
