# SCRM2.0功能说明

## SCRM2.0功能说明

### 乐寓系统手机号同步到尘锋系统

乐寓系统在创建员工的时候，会根据员工姓名同步到尘锋系统同名员工的手机号；如果有同名员工，将会覆盖尘锋系统同名员工的手机号，如果出现这种情况，请手动在尘锋修改手机号。

### 客户缴纳首期租金后将会传数到联系人自定义字段

组合字段：承租人/同住人【区域】-【店面】-【物业地址】-【门牌号】。多份合同则拼接展示、只展示在租的

租约信息：承租人/同住人/已退租-【签约信息】（签约合同姓名-签约合同手机号-户型-合同开始日-合同结束日）。多份合同、只展示在租的。

如果是续约了的话，该字段的合同开始和结束时间会被覆盖为续约合同的开始和结束时间。

如果是退房的话，只显示最后一间房的租约信息。

该字段中，如果联系人为同住人，则【签约信息】中的合同姓名和手机号为承租人的。

### 企微备注名

默认展示他在乐寓系统的名字，不同的管家添加同一个客户，企微备注名都会展示为他的乐寓系统名字。

展示的是承租人的最新合同的姓名，如果联系人存在承租人和同住人的身份，则只展示承租人合同的姓名。如果全部合同退租了，包括同住人，则备注名变成已退租。

### 企微好友标签

根据联系人在乐寓系统的身份进行展示，如果既有承租人身份也有同住人身份，则会默认显示承租人的好友标签。如果全部合同退租了，包括同住人，则好友标签变成已退租。

### 退租时

清掉区域、门店名称、物业地址、房间号、组合字段。

保留字段租约信息、退租日期。

企微备注名和好友标签变成已退租

如果客户再次签约，则退租日期清掉，其他字段根据新合同数据进行展示。
