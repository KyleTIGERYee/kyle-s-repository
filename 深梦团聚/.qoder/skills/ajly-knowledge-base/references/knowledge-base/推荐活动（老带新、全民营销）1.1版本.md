# 推荐活动（老带新、全民营销）1.1版本

推荐活动（老带新/全民营销）1.1功能说明

# 概述：

在原有的推荐活动的基础上，增加了自动发券的功能，可以根据新客户的成交租期长短，配置各种活动专用券，自动发放不同任意数量的券；同时券可以配置每次只能使用一张，可给用户发多张券，限制其分多次使用。另外，现在可以在新建活动时，填写尘锋“线索对接”的接口URL，活动所收集的线索，将自动传到尘锋系统中进行分配。

# 更新说明：

活动规则增加【推荐人资格】枚举：“非在租用户”

活动规则增加条件规则：“奖励条件”

活动奖励增加赠券模式，可按照不同租期自动赠送不同种类的券

推荐人转发页可以自定义按钮颜色和页面颜色

## 可以填入尘锋“线索对接”所配置的URL，实现自动将线索传到尘锋

优惠券可以配置“活动专用”券，用于营销活动发放

推荐活动1.0需求回顾，请查看原型链接：Lanhu_Share

# 推荐活动功能调整

## 新建活动

1.活动规则-推荐人资格中，增加“非在租用户”：排除当前在租租客和同住人

## 奖励条件变更为”被推荐人入住超过X天“，例如，当填入值为“3”时，用户5月1号入住（并缴纳租金），则5月4号时用户会收到奖励。

如果用户换房了，则会从第一份合同的开始日期开始算起，例如，当填入值为“3”时，用户5月1号入住（并缴纳租金），5月2号换房，在5月4号时用户同样会收到奖励。

如果用户是转租签约的，视作新签用户，符合规则即会发放优惠券。

【活动奖励】-【赠券模式】，支持按照不同模式进行自动赠券

·不赠券：同原有功能

·赠送推荐人：当满足奖励条件后，发送优惠券给活动中对应的推荐人

·赠送被推荐人：当满足奖励条件后，发送优惠券给活动中对应的被推荐人（即新客户）

·赠送双方：即为以上两种情况之和

以下以赠券-推荐人为例子，讲解自动赠券逻辑：

【适用租期】：即新客户成交合同签的租期月份，若在发券之前产生了换房、续约等业务行为都只取成交合同的租期。租期可选择1~12个月以上，每个租期只能被选择一次，且所有租期都必须被选择。

【选择券】：读取“活动专用”类型的有效优惠券（细节请查看优惠券功能调整），每个券只能被一个活动选择，可以在同一个活动下选择多次，但不能被其他活动选择。

【赠送量】：只能输入0-100的整数

综合下来即为：当新客户签约x个月时，发放对应的券，发N张。例如规则配置用户签约12个月，发放面额为100的券，赠送量为6，当满足赠券条件后，一次性发放6张面额为100的券。（优惠券中可配置每次只可用一张，这样可以限制用户必须分6次才可以用完）

落地页配置，支持配置按钮颜色、页面背景颜色

颜色点击后可以打开颜色选取器，可用吸管工具吸取颜色

5.【是否开启关联管家】可以在小程序推荐人页面中打开或关闭关联管家选项。

【是否开启关联企业选项】可以在小程序被推荐人页面中打开或关闭企业名称字段

## 表单统计

字段调整：

·【用户电话】改为【被推荐人电话】

·增加【成交时间】，置于【被推荐人电话】后。统计表单中成交客户的缴纳第一期租金的时间，精确到具体的时间点 YYYY-MM-DD hh:mm:ss

·增加【管家名称】、【管家电话】，置于表单最后一列

·增加【推荐人领券】，置于【推荐人电话】后，展示领券的券名称，如果没有就是“-”。（只有实际发放了才显示该名称）

·增加【被推荐人领券】，置于【被推荐人电话】后，展示领券的券名称，如果没有就是“-”。（只有实际发放了才显示该名称）

·增加增加【成交数】，统计该被推荐人成交了几间房（主要是用于统计企业客户）

·增加增加【企业名称】，如果没有就是“-”。

表单统计导出时，除了以上数据，还增加银行收款信息的导出（收款人、银行账户、银行名称、开户行

## 其他

若同一个新用户同一天成交了多间房，则不会自动发放优惠券

# 优惠券功能调整

增加券用途：【活动专用】，只有该类型的券，才可以在新建活动时，被选择到赠券规则中。

当券用途为【活动专用】时，没有【领券方式】【次数】【发行量】【适用签约时长】【适用房源】【是否对外展示】等配置项，因为无需配置。

当勾选【支付时单次用券上限为1次】时，用户收到多张同类型的券后，每次只能使用一张，也就是说，用户需要分多次才可以用完。

# 尘锋系统

现在可以在创建活动时，在最底部的【外部Feedback URL】填入尘锋-线索对接中的Feedback UR，从而实现关键数据自动传到尘锋。

相关字段和解释：

可以在尘锋的线索对接中建立上表中字段和尘锋字段的关联关系，建立关联关系后，用户数据将会自动传到对应的字段，例如：


| 字段名 | 解释 |
| --- | --- |
| recomedPhone | 被推荐人电话 |
| recomedPhone | 推荐人电话 |
| housePrice | 租金预算 |
| checkInTime | 意向入住时间 |
| butlerName | 管家名称 |
| butlerPhone | 管家电话 |
| houseType | 意向房型 |
| region | 意向地区 |
| activityName | 活动名称 |
