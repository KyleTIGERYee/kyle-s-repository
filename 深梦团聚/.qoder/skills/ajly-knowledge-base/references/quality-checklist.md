# 文档生成质量检查清单

本文档定义生成功能说明文档后的质量检查标准。

---

## 双重产出检查

生成新文档时，**必须同时产出**：

- [ ] Markdown 文档（`knowledge-base/文档名称.md`）
- [ ] JSON 知识索引（更新到 `knowledge-base/knowledge-base.json`）

**⚠️ 重要：缺少任一产出，视为未完成**

---

## 结构检查

- [ ] 包含"概述"章节
- [ ] 包含"操作指引"或"功能模块"章节
- [ ] 章节顺序符合规范（概述→功能→权限→配置→对接）
- [ ] 标题层级正确（一级#、二级##、三级###）
- [ ] 文档开头重复标题（纯文本）

---

## 术语检查

- [ ] 使用"PC端"而非"电脑端/web端"
- [ ] 使用"小程序"而非"微信端/移动端"
- [ ] 使用"租客"指代租房客户
- [ ] 使用"管家"指代一线业务人员
- [ ] 使用"店长"指代门店负责人
- [ ] 术语使用一致（不混用"租客"和"客户"）
- [ ] 首次出现的专业术语适当解释

---

## 格式检查

- [ ] 按钮名称使用粗体（**提交**、**保存**）
- [ ] 菜单路径使用【】（【字典设置】-【租客字典】）
- [ ] 界面提示文字使用""（"操作成功"）
- [ ] 字段名使用代码块（`userPhone`）
- [ ] 配置项名称使用粗体（**参数名称**）
- [ ] 列表符号统一（无序用-，有序用数字）
- [ ] 表格格式正确（|分隔，表头分隔线）

---

## 内容检查

- [ ] 语言风格正式、专业（不用口语化表达）
- [ ] 数字使用阿拉伯数字（5天、30天）
- [ ] 金额格式统一（300元、100元）
- [ ] 日期格式统一（YYYY-MM-DD）
- [ ] 流程描述清晰（步骤明确、逻辑连贯）
- [ ] 包含必要的注意事项
- [ ] 权限说明完整（如适用）
- [ ] 配置说明清晰（如适用）

---

## JSON 知识索引检查

- [ ] 已添加到 `knowledge-base/knowledge-base.json`
- [ ] JSON 格式正确（逗号、引号、括号匹配）
- [ ] `keys` 字段包含 3-5 个关键词
- [ ] `content` 包含核心功能、关键规则、注意事项
- [ ] `file_path` 以 `knowledge-base/` 开头
- [ ] `file_path` 与实际 Markdown 文件名一致
- [ ] `strategy` 设置为 "keyword"
- [ ] `enabled` 设置为 true

---

## 完整性检查

- [ ] 概述章节说明功能目的和范围
- [ ] 操作步骤包含前置条件
- [ ] 异常情况有处理说明
- [ ] 相关系统对接有字段映射（如适用）

---

## 常见错误

| 错误类型 | 错误示例 | 正确示例 |
|----------|----------|----------|
| 术语不规范 | 电脑端 | PC端 |
| 格式错误 | 点击提交按钮 | 点击**提交**按钮 |
| 菜单路径错误 | 字典设置-租客字典 | 【字典设置】-【租客字典】 |
| 缺少章节 | 直接写功能点 | 先写#概述 |
| 口语化 | 用户可以点这里 | 点击**按钮** |

---

## 检查通过标准

**必须同时满足**：
1. Markdown 文档检查通过
2. JSON 知识索引检查通过

所有检查项通过，方可输出最终文档。

如有未通过项，需返回修改后再检查。

---

## 输出要求

完成质量检查后，必须同时输出：

1. **Markdown 文档内容** - 完整的 `.md` 文件内容
2. **JSON 知识索引** - 新增的 JSON 条目
3. **更新后的 knowledge-base.json** - 包含新条目的完整 JSON 文件
4. **质量检查报告** - 各检查项的通过情况
