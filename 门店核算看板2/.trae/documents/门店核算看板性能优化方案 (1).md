## 优化目标
解决项目进入时因大量数据接口请求导致的卡顿问题

## 问题诊断
1. **串行请求瀑布** - ensureResourcesInitialized 中串行调用两个大接口
2. **缺乏数据缓存** - 每次切换月份都重新请求所有数据
3. **大数据处理阻塞主线程** - 数据适配器遍历大量数据无时间切片
4. **组件重复渲染** - 缺乏有效的 memo 优化

## 优化方案

### Phase 1: 消除请求瀑布 (CRITICAL)
- [ ] 将 `ensureResourcesInitialized` 中的串行请求改为 `Promise.all` 并行
- [ ] 使用 `Promise.allSettled` 处理部分失败场景

### Phase 2: 实现数据缓存策略 (CRITICAL)
- [ ] 添加内存缓存层，缓存已请求的月份数据
- [ ] 实现 stale-while-revalidate 策略，优先展示缓存数据
- [ ] 添加缓存过期机制

### Phase 3: 大数据处理优化 (HIGH)
- [ ] 使用 `requestIdleCallback`/`setTimeout` 切片大数据处理
- [ ] 将数据转换逻辑移至 Web Worker (可选)
- [ ] 实现虚拟滚动优化长列表渲染

### Phase 4: 组件渲染优化 (MEDIUM)
- [ ] 使用 `React.memo` 包裹纯展示组件
- [ ] 使用 `useMemo` 缓存计算结果
- [ ] 使用 `useCallback` 稳定回调函数引用
- [ ] 实现骨架屏替代 loading spinner

### Phase 5: 代码分割与懒加载 (MEDIUM)
- [ ] 使用 `React.lazy` 和 `Suspense` 分割路由组件
- [ ] 延迟加载地图组件（首屏非必需）

## 预期效果
- 首屏加载时间减少 40-60%
- 月份切换响应时间减少 70%
- 主线程阻塞时间减少 80%
- 内存占用优化，避免重复数据存储
