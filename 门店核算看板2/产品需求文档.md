## 1. 项目概览 (Project Overview)

### **项目简介**
安居乐寓门店核算看板是一款面向长租公寓运营管理企业的 **B端财务分析可视化平台**，用于实时展示和分析各门店的收入、支出、利润等核心财务指标，支持多维度对标分析和 AI 智能问答。

### **核心价值**
- **财务透明化**：将分散在各门店的财务数据集中可视化，帮助管理层快速掌握全公司及各战区的经营状况
- **决策支持**：通过环比增长率、排行榜、趋势分析等功能，辅助管理者识别问题门店和优秀门店
- **降本增效**：通过 AI 问数功能，让非财务人员也能快速获取财务洞察，减少人工报表制作成本

### **目标用户**
1. **高管/决策层**：快速查看全公司财务健康状况，把控整体经营方向
2. **战区负责人**：监控管辖区域内各门店的业绩表现，进行横向对比
3. **门店经理**：查看本店详细收支结构，了解经营细项

## 3. 核心功能模块详解 (Core Features Breakdown)

### 3.1 总览仪表盘 (`Dashboard.tsx`)
**路由**: `/`

**功能描述**：
全公司级财务健康状况总览，是管理层快速了解整体经营状况的入口页面。

**用户操作**：
- 📅 **月份选择**：点击右上角日期选择器，切换查看历史月份数据
- 📊 **查看 KPI 指标**：净利润、总收入、总支出三大核心指标卡片
- 🗺️ **地图交互**：点击地图上的门店标记点跳转至门店详情页
- 📈 **趋势追踪**：点击 KPI 卡片跳转至对应科目的 12 月趋势分析
- 📋 **战区明细**：展开/收起科目分类查看各战区财务数据矩阵
- 🏆 **排行榜**：查看门店收入/支出/利润 Top 10，点击门店跳转详情

**关键组件**：
- `MonthPicker`：自定义月份选择弹窗
- `RankingList`：门店排行榜列表组件
- `BMapContainer`：百度地图门店分布可视化

**业务逻辑**：
- 并行请求当月和上月数据，计算真实环比增长率
- 支持"扭亏"（上月亏损→本月盈利）和"转亏"（上月盈利→本月亏损）特殊标签
- 战区矩阵数据通过科目展开/收起控制子级科目显示

---

### 3.2 门店概览 (`StoreOverview.tsx`)
**路由**: `/store/:storeId`

**功能描述**：
单个门店的详细财务报表，包含收支结构分析和人均效能指标。

**用户操作**：
- 🏪 **切换门店**：点击门店名称，弹出战区分组的门店选择器
- 📅 **月份选择**：切换查看历史月份数据
- 🍩 **收支结构分析**：交互式饼图展示收入/支出各科目占比
- ➕ **加入对标**：点击右下角 FAB 按钮将当前门店加入对标分析
- 📈 **趋势追踪**：点击 KPI 卡片跳转趋势分析页

**关键组件**：
- `StructureSection`：带交互效果的环形饼图 + 科目列表
- `MonthPicker`：复用的月份选择组件

**业务逻辑**：
- 显示门店基础信息（编制人数、在管面积、楼栋数量、地址）
- 计算人均指标（人均净利润、人均收入、人均支出）
- 环比增长率真实计算（对比上月同期数据）

---

### 3.3 对标分析 (`Benchmark.tsx`)
**路由**: `/benchmark`

**功能描述**：
多门店横向对比分析工具，支持最多 5 家门店同时对比。

**用户操作**：
- ➕ **添加门店**：弹出战区分组的门店选择器添加对比门店
- ❌ **移除门店**：点击门店标签上的关闭图标移除
- 📊 **查看对比表格**：全科目×多门店矩阵式对比
- 📈 **趋势分析**：点击任意科目行的趋势图标，跳转多店趋势对比图

**关键组件**：
- 复用 `App.tsx` 中的 `BenchmarkContext` 全局状态

**业务逻辑**：
- 第一个添加的门店作为"基准门店"标识
- 最多支持 5 个门店同时对比
- 利润科目颜色区分（正利润红色、负利润绿色）

---

### 3.4 趋势分析 (`TrendAnalysis.tsx`)
**路由**: `/trend/:subjectId`

**功能描述**：
查看特定科目过去 12 个月的趋势变化，支持全公司模式和多门店对比模式。

**用户操作**：
- 📈 **查看趋势图**：12 个月折线图，鼠标悬停显示具体数值
- 📋 **查看近期数据**：表格展示最近 4 个月的数值

**关键组件**：
- recharts `LineChart`：支持多数据系列的折线图

**业务逻辑**：
- **全公司模式**：`?mode=company`，显示该科目全公司汇总趋势
- **门店对比模式**：`?stores=id1,id2`，多门店趋势叠加对比

---

### 3.5 AI 问数 (`AIChat.tsx`)
**路由**: `/ai-chat`

**功能描述**：
基于 Dify 平台的智能财务问答助手，支持自然语言查询经营数据。

**用户操作**：
- ⌨️ **文字输入**：桌面端默认文字输入模式
- 🎤 **语音输入**：移动端默认语音模式，长按说话，上滑取消
- 💡 **快捷提问**：点击预设问题快速发起对话
- 🔄 **清空对话**：点击刷新按钮重置对话历史

**关键组件**：
- `ThinkingBubble`：AI 思考过程可折叠展示
- `Suggestions`：AI 返回的追问建议列表
- `MessageItem`：消息气泡（支持 Markdown 渲染）

**业务逻辑**：
- 集成 Dify API 流式响应 (SSE)
- 支持对话上下文 `conversation_id` 维持多轮对话
- 支持特殊标签解析：`<thinking>`、`<suggestions>`

---

### 3.6 门店地图 (`BMapContainer.tsx`)
**嵌入位置**: Dashboard 页面（桌面端可见）

**功能描述**：
通过百度地图展示深圳市内所有门店的地理分布。

**用户操作**：
- 🗺️ **缩放拖拽**：标准地图操作
- 📍 **悬停标记**：显示门店名称和所属战区
- 👆 **点击跳转**：跳转至门店详情页

**业务逻辑**：
- 深色主题定制样式（通过 `setMapStyleV2` 配置）
- 自动视野调整（`setViewport` 包含所有标记点）
- 战区颜色区分（不同战区使用不同标记颜色）

## 4. 数据模型与实体 (Data Models & Entities)

### **关键实体**

```typescript
// 战区/区域
interface Region {
  id: string;           // 如 "第一战区"
  name: string;         // 战区名称
}

// 门店
interface Store {
  id: string;           // 门店 ID
  name: string;         // 门店名称
  regionId: string;     // 所属战区 ID
  address: string;      // 门店地址
  lat: number;          // 纬度（百度坐标）
  lng: number;          // 经度（百度坐标）
  headcount?: number;   // 编制人数
  managedArea?: number; // 在管面积（㎡）
  buildingCount?: number; // 楼栋数量
}

// 科目类型
enum SubjectType {
  REVENUE = 'REVENUE',  // 收入类
  EXPENSE = 'EXPENSE',  // 支出类
  PROFIT = 'PROFIT',    // 利润类
}

// 科目定义
interface Subject {
  id: string;           // 如 "rev_1", "exp_1", "net_profit"
  name: string;         // 科目名称
  type: SubjectType;    // 科目类型
  isParent?: boolean;   // 是否为父级汇总科目
}

// 仪表盘数据
interface DashboardData {
  totalProfit: number;
  totalRevenue: number;
  totalExpense: number;
  profitGrowth: number;    // 利润环比增长率
  revenueGrowth: number;   // 收入环比增长率
  expenseGrowth: number;   // 支出环比增长率
  prevProfit?: number;     // 上月利润（用于扭亏/转亏判断）
  matrix: RegionFinancialSummary[];  // 战区财务矩阵
  rankings: { revenue, expense, profit: RankItem[] };
}

// 门店财务汇总
interface StoreFinancialSummary {
  storeId: string;
  storeName: string;
  revenue: number;
  expense: number;
  netProfit: number;
  details: Record<string, number>;  // 各科目明细
  // 环比增长率
  revenueGrowth?: number;
  expenseGrowth?: number;
  profitGrowth?: number;
}
```

### **预定义科目清单**
系统预设 **34 个财务科目**：
- **收入类 (6 个)**：公寓及商铺收入、增值服务收入、价外收入、换房手续费、转租手续费、清洁费
- **支出类 (27 个)**：收楼成本、退款、押金、人工成本、物业费、保洁费、维护费、水电费、营销费等
- **利润类 (1 个)**：净利润

### **数据流向**
```
后端 API → apiService (加密、请求) → dataAdapter (格式转换) 
→ mockData (缓存管理、业务聚合) → React 组件 (UI 渲染)
```

## 6. API 与集成 (API & Integration)

### **后端交互模式**
| 特性 | 说明 |
|------|------|
| 协议 | RESTful API (POST) |
| 认证 | 自定义加密参数 `reportDisplayKey` |
| 跨域 | 通过 Vite Proxy 代理 |
| 基地址 | `http://uat-mp.szajly.com` |

### **核心 API 端点**
```
POST /api/store/getStatisticsStore
请求参数：{ reportDisplayKey, companyId, storeIds?, storeName? }
响应：门店基础信息（员工数、面积、经纬度）

POST /api/v3/profits/report/display
请求参数：{ reportDisplayKey, companyId, storeId?, warZone?, statisticsMouth? }
响应：战区→门店→科目三级财务数据
```

## 7. 改进建议与潜在风险 (Recommendations & Risks)

### **代码质量**
| 问题 | 建议 |
|------|------|
| ⚠️ 组件过于庞大 | `Dashboard.tsx` (513行)、`StoreOverview.tsx` (760行) 应拆分为更小的子组件 |
| ⚠️ MonthPicker 重复定义 | Dashboard 和 StoreOverview 各自定义了一份，应抽取为共享组件 |
| ⚠️ 缺少错误边界 | 建议添加 `ErrorBoundary` 组件捕获运行时错误 |
| ⚠️ 状态管理分散 | BenchmarkContext 仅管理对标门店，可考虑统一使用 Zustand/Redux 管理全局状态 |

### **性能瓶颈**
| 问题 | 影响 | 建议 |
|------|------|------|
| 🐢 趋势分析 12 次并发请求 | 页面加载慢 | 后端提供批量查询接口 |
| 🐢 百度地图 SDK 体积大 | 首屏加载延迟 | 懒加载地图组件 |
| 🐢 每次切换门店重新请求 | 用户体验差 | 增加本地缓存 |

### **安全风险**
| 问题 | 建议 |
|------|------|
| 🔐 API Key 硬编码 | `ai_config.ts` 中的 Dify API Key 应通过环境变量注入 |
| 🔐 加密算法简单 | `reportDisplayKey` 使用简单 Base64，容易被破解 |

### **缺失功能（MVP 完善建议）**
| 功能 | 说明 |
|------|------|
| 🚫 用户认证 | 目前无登录系统，应接入企业 SSO |
| 🚫 权限控制 | 不同角色应看到不同范围的数据 |
| 🚫 数据导出 | 应支持导出 Excel/PDF 报表 |
| 🚫 通知提醒 | 异常指标预警推送 |
| 🚫 多语言 | 目前仅支持中文 |
